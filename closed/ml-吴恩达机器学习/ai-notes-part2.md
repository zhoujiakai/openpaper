## 43 — 1.2 神经元和大脑（10:52）

简单介绍了生物大脑中的神经元结构：树突（dendrite）接收输入信号，轴突（axon）发送输出信号，神经元之间通过突触连接。人工神经网络的灵感来自于生物神经元，但如今的神经网络几乎没有刻意模仿大脑了，只是保留了这个名字。

## 44 — 1.3 需求预测（16:23）

以电商需求预测为例：根据衣服的价格、运费、营销预算和面料类型，预测这件衣服是否会成为畅销品（top seller）。

将神经元组织成层：输入层（input layer）接收特征输入 → 隐藏层（hidden layer）→ 输出层（output layer）输出预测结果。每个神经元计算一个激活值 a（activation），激活值由前一层输入加权求和再通过激活函数得到。

神经网络中每一层的计算类似于逻辑回归。该例子中输入层有 4 个特征，隐藏层有 3 个神经元，输出层有 1 个神经元，输出是畅销品的概率。

这种网络也称作多层感知器（Multi-layer Perceptron）。

## 45 — 1.4 例子：图像识别（06:36）

以手写数字识别为例：输入是一张 28×28 的灰度图像（共 784 个像素），每个像素值作为输入层的特征。隐藏层可以学习从边缘到局部再到整体的层次化特征：
- 第一层神经元识别边缘和简单特征；
- 第二层将边缘组合成人脸器官；
- 第三层将器官组合成人脸。

这就是神经网络自动从数据中学习特征层次的能力。

## 46 — 2.1 神经网络中的层（09:50）

每个层的计算过程：接收上一层所有激活值作为输入，计算加权和加偏置（z = W·a + b），再经过激活函数 g(z) 得到输出激活值。

符号约定：
- $a_j^{[l]}$ 表示第 l 层的第 j 个神经元的激活值
- $w_j^{[l]}$ 和 $b_j^{[l]}$ 是第 l 层第 j 个神经元的参数
- 输入层记为第 0 层，即 $a^{[0]} = x$

每一层的计算实际上像逻辑回归或线性回归：先做线性变换 z = w·a + b，再经过激活函数。

## 47 — 2.2 更复杂的神经网络（07:19）

展示了多个隐藏层的神经网络架构。每个隐藏层可以有不同的神经元数量。层与层之间是全连接的（dense），即每层的每个神经元都接收上一层的所有激活值作为输入。

## 48 — 2.3 推理：做出预测（前向传播）（05:24）

前向传播（forward propagation）的定义：从输入层开始，逐层计算，直到输出层得到最终预测值。

过程：
1. 输入 x 作为第 0 层的激活值 $a^{[0]}$
2. 对每一层 l：计算 $z^{[l]} = W^{[l]}a^{[l-1]} + b^{[l]}$，然后 $a^{[l]} = g(z^{[l]})$
3. 最终 $a^{[L]}$ 是输出预测值

这与从左到右传播信息的过程对应，所以叫做前向传播。推理阶段就是做前向传播。

## 49 — 3.1 代码中的推理（07:13）

使用 Python 和 NumPy 实现前向传播的代码。TensorFlow 中可以用 `Sequential` 模型逐层定义网络结构：

```python

    Dense(units=25, activation='sigmoid'),
```

## 52 — 4.1 在一个单层中的前向传播（05:07）

详细拆解单层的前向传播计算。输入 a_in 是一个向量（含上一层的所有激活值），权重 W 是一个矩阵（每列对应一个神经元），偏置 b 是一个向量。

用 for 循环可以实现：对每个神经元，计算加权和 z，再应用激活函数 g(z) 得到输出。

但 for 循环效率低，接下来的视频会讲如何向量化。

## 53 — 4.2 前向传播的一般实现（07:53）

构建通用的 `dense` 函数实现单层前向传播：

```python
def dense(a_in, W, b, g):
    units = W.shape[1]
    a_out = np.zeros(units)
    for j in range(units):
        z = np.dot(W[:, j], a_in) + b[j]
        a_out[j] = g(z)
    return a_out
```

配合 `sequential` 函数可以将多层串联起来：

```python
def sequential(x):
    a1 = dense(x, W1, b1, sigmoid)
    a2 = dense(a1, W2, b2, sigmoid)
    a3 = dense(a2, W3, b3, sigmoid)
    return a3
```

## 55 — 6.1 神经网络如何高效实现（04:23）

深度学习能扩展到大型神经网络的原因之一是向量化：利用矩阵乘法在 GPU 和 CPU 上高效实现。

向量化实现将 for 循环替换为 `np.matmul` 矩阵乘法：

```python
Z = np.matmul(A_in, W) + B
A_out = g(Z)
```

A_in、W、B、Z、A_out 全部是矩阵（2D 数组）。这个视频是概念介绍，接下来的两个可选视频会详细讲矩阵乘法。

## 56 — 6.2 矩阵乘法（09:28）

从向量点积开始讲起：
- 向量 a = [1, 2]，向量 w = [3, 4]
- 点积：z = a·w = 1×3 + 2×4 = 11
- 等价于：a^T × w（a 的转置乘以 w）

向量-矩阵乘法：a^T 是 1×2，W 是 2×2，结果 Z 是 1×2。计算第一个元素时用 a^T 点积 W 的第一列，计算第二个元素时用 a^T 点积 W 的第二列。

矩阵-矩阵乘法：多个向量-矩阵乘法的组合。

## 57 — 6.3 矩阵乘法的规则（09:33）

矩阵乘法的一般形式：
- A 是 m×n 矩阵，W 是 n×p 矩阵
- A^T × W 的结果 Z 是 m×p 矩阵
- A^T 的第 i 行与 W 的第 j 列做点积，得到 Z 的第 i 行第 j 列元素

关键规则：
- A 的列数必须等于 W 的行数（即点积要求向量长度相同）
- 输出矩阵 Z 的行数等于 A^T 的行数，列数等于 W 的列数

## 58 — 6.4 矩阵乘法代码（04:23）

使用 NumPy 实现矩阵乘法和向量化前向传播：

```python
AT = A.T  # 转置
Z = np.matmul(AT, W)  # 矩阵乘法，也可以用 AT @ W
```

向量化前向传播的完整实现：
```python
def dense(A_in, W, B, g):
    Z = np.matmul(A_in, W) + B
    A_out = g(Z)
    return A_out
```

TensorFlow 中的约定：单个样本排列在行（而非列），所以实现中直接用 A_in 而非 A_in^T。矩阵乘法在现代计算机上执行非常快，这是神经网络高效的关键。

## 59 — 7.1 TensorFlow 实现（03:37）

回顾 TensorFlow 中训练神经网络的步骤：
1. 定义模型：`Sequential` + `Dense` 层
2. 编译模型：指定损失函数 `BinaryCrossentropy`
3. 训练模型：`model.fit(X, Y, epochs=100)`

训练过程就是重复：前向传播计算预测值 → 计算损失 → 反向传播计算梯度 → 更新参数。

## 60 — 7.2 训练细节（13:41）

详细讲解 TensorFlow 训练神经网络的三个步骤：

**第一步：定义模型**

```python
model = Sequential([
    Dense(units=25, activation='sigmoid'),
    Dense(units=15, activation='sigmoid'),
    Dense(units=1, activation='sigmoid')
])
```

**第二步：编译模型**（指定损失函数）
```python
model.compile(loss=BinaryCrossentropy())
```

逻辑回归的损失函数（二元交叉熵）：
$$L = -y \log(f(x)) - (1-y)\log(1-f(x))$$

这也是神经网络二分类常用的损失函数。

**第三步：训练模型**
```python
model.fit(X, Y, epochs=100)
```

梯度下降的反向传播算法自动计算每一层参数的梯度并更新。训练完成后参数 w 和 b 被更新为优化后的值。

## 61 — 8.1 sigmoid 的替代品（05:30）

除了 Sigmoid，还有另外两个常用的激活函数：
- **ReLU（Rectified Linear Unit）**：$g(z) = \max(0, z)$，最常用的隐藏层激活函数。
- **Linear（线性）**：$g(z) = z$，等于不激活，用于输出层做回归时的特殊情况。

Sigmoid 仍然用于二分类的输出层，但在隐藏层中 ReLU 通常更好。

## 62 — 8.2 选择激活函数（08:25）

如何为神经网络不同层选择激活函数：

**输出层**：
- 二分类：Sigmoid
- 多分类：Softmax
- 回归（预测正数或负数）：Linear

**隐藏层**：
- 最常用：ReLU。计算快，只有 max(0, z) 这一种运算；且 ReLU 只有左侧是平的，梯度下降仍能工作。
- Sigmoid 在隐藏层不再推荐，因为两侧都是平的（梯度消失问题）。

选择规律：大多数情况下隐藏层用 ReLU，输出层根据任务选择。

## 63 — 8.3 为什么我们需要激活函数（05:32）

如果所有层都用线性激活函数（或无激活函数），多层神经网络等价于单层线性模型：
$$a^{[2]} = W^{[2]}(W^{[1]}x + b^{[1]}) + b^{[2]} = (W^{[2]}W^{[1]})x + (W^{[2]}b^{[1]}+b^{[2]}) = W'x + b'$$

多个线性变换的组合仍然是一个线性变换。所以隐藏层必须使用非线性激活函数（如 ReLU、Sigmoid），否则无论多少层，模型都只是线性模型。

## 64 — 9.1 多类（03:29）

多类分类（Multi-class Classification）：输出类别多于两个。如手写数字识别有 10 个类别（0-9）。

多类分类不同于多标签分类（Multi-label Classification），后者一个输入可以有多个标签。

## 65 — 9.2 Softmax（11:33）

Softmax 回归（Softmax Regression）是多类分类的泛化，是逻辑回归的推广。

Softmax 函数公式：
$$a_j = \frac{e^{z_j}}{\sum_{k=1}^{N} e^{z_k}}$$

每个输出 $a_j$ 可以理解为该类的概率估计，所有 $a_j$ 之和为 1。

Softmax 的损失函数：
$$L = -\log a_j \quad \text{（如果 y=j）}$$

这叫做分类交叉熵（Categorical Cross Entropy）/ 稀疏分类交叉熵（Sparse Categorical Cross Entropy）。

## 66 — 9.3 神经网络的 Softmax 输出（07:25）

手写数字识别例子：使用含 Softmax 输出层的神经网络。输出层有 10 个神经元，分别对应 0-9 的数字。Softmax 激活函数使 10 个输出都为正且和为 1，可解释为概率。

TensorFlow 中的代码：
```python
model = Sequential([
    Dense(units=25, activation='relu'),
    Dense(units=15, activation='relu'),
    Dense(units=10, activation='softmax')
])
model.compile(loss=SparseCategoricalCrossentropy())
```

这个网络给出的是多类分类的概率分布。

## 67 — 9.4 Softmax 的改进实现（09:13）

**数值稳定性问题**：当用 `softmax` 配合 `SparseCategoricalCrossentropy` 时，中间量 $e^z$ 可能非常大或非常小，导致数值精度损失。

**改进方法**：输出层用线性激活函数（即不激活），并将激活计算移到损失函数中：

```python
model = Sequential([
    Dense(units=25, activation='relu'),
    Dense(units=15, activation='relu'),
    Dense(units=10, activation='linear')
])
model.compile(loss=SparseCategoricalCrossentropy(from_logits=True))
```

`from_logits=True` 让 TensorFlow 在损失函数内部做 Softmax 计算，可以重新排列计算项，避免数值溢出。缺点是代码更难理解。

**注意**：使用 `from_logits=True` 后，`model.predict(x)` 输出的是 $z$ 值（logits），不是概率。要得到概率需手动应用 softmax。

逻辑回归同理，推荐用 `BinaryCrossentropy(from_logits=True)`。

## 68 — 9.5 多个输出的分类（Optional）（04:20）

**多标签分类（Multi-label Classification）**：一个输入可以同时有多个标签。

例子：自动驾驶中的图像识别，需要同时检测图像中是否有汽车、公交车、行人。输出是三个二进制标签的向量 `[car, bus, pedestrian]`。

实现方式：
1. 训练三个独立的神经网络，分别处理每个标签。
2. 或者训练一个神经网络，输出层有 3 个神经元，每个都使用 Sigmoid 激活（因为每个输出都是独立的二分类问题）。

多类分类和多标签分类的区分：
- 多类分类：输出是多个类别中的一个（互斥），用 Softmax。
- 多标签分类：输出是多个标签，每个标签独立存在（非互斥），用多个 Sigmoid。

## 69 — 10.1 高级优化方法（06:26）

**Adam 算法**（Adaptive Moment Estimation）：比普通梯度下降更快、更好的优化算法。

传统的梯度下降使用单一的学习率 α 更新所有参数，学习率大了会震荡，小了收敛太慢。

Adam 的核心思想：
- 为每个参数自动调整学习率：如果参数在相似方向上持续更新，就增大学习率；如果参数在震荡来回，就减小学习率。
- 对每个参数 $w_j$ 和 $b$ 都有一个独立的学习率参数。

TensorFlow 代码：
```python
model.compile(optimizer=tf.keras.optimizers.Adam(learning_rate=1e-3),
              loss=BinaryCrossentropy())
```

Adam 对初始学习率的选择更加鲁棒，但仍需尝试几个值找到最优的学习率。如今 Adam 已成为训练神经网络的事实标准。

## 70 — 10.2 卷积层（08:56）

到目前为止讲的都是密集层（Dense Layer），即每层的每个神经元都接收上一层的所有激活值。

**卷积层（Convolutional Layer）**：每个神经元只查看输入的一个局部区域（窗口）。

优点：
1. 计算更快（每个神经元只处理局部区域）
2. 需要更少的训练数据 / 更不容易过拟合

以 ECG（心电图）信号为例说明一维卷积层：第一层的每个神经元只查看输入信号的 20 个连续时间点。多个卷积层组成的网络叫做卷积神经网络（CNN）。

神经网络的很多前沿架构（如 Transformer、LSTM、Attention 模型）本质上都是在发明新类型的层，将这些层像积木一样拼装成更强大的网络。

## 71 — 11.1 决定下一步做什么（03:42）

训练完一个模型后发现性能不佳时，怎么决定下一步？

常见选项：
- 获取更多训练数据
- 使用更少特征（减少过拟合）
- 增加额外特征
- 增加多项式特征
- 减小 / 增大正则化参数 λ

我们需要一种方法系统地诊断模型问题，而不是盲目尝试。

## 72 — 11.2 模型评估（10:26）

评估模型的标准方法：将数据集划分为训练集（70%）和测试集（30%）。

流程：
- 在训练集上训练模型
- 在测试集上计算 $J_{test}$（测试误差）来评估泛化能力

对于线性回归用均方误差，对于分类用分类错误率（被错误分类的比例）。

训练误差 $J_{train}$ 可能低于测试误差 $J_{test}$，因为我们是在训练集上优化的。

## 73 — 11.3 模型选择和训练/交叉验证/测试集（14:53）

当比较多个模型（如不同多项式次数）时，不能只用训练集和测试集，需要在三组数据上评估：

- **训练集（Training set）**：60%，训练模型参数。
- **交叉验证集（Cross-validation / Dev set）**：20%，用于模型选择和超参数调优。
- **测试集（Test set）**：20%，用于最终评估泛化能力。

流程：
1. 在训练集上训练多个候选模型
2. 在交叉验证集上评估 $J_{cv}$，选择 $J_{cv}$ 最低的模型
3. 在测试集上计算 $J_{test}$ 作为最终性能报告

使用交叉验证集避免了在测试集上调参数导致的"信息泄露"，保证测试集是纯粹的无偏评估。

## 74 — 12.1 诊断偏差和方差（11:13）

高偏差（偏差大）和高方差（方差大）的诊断方法：

通过比较 $J_{train}$ 和 $J_{cv}$：
- **高偏差（欠拟合）**：$J_{train}$ 高，$J_{cv}$ 也高且与 $J_{train}$ 接近。
- **高方差（过拟合）**：$J_{train}$ 低，$J_{cv}$ 远高于 $J_{train}$。
- **既高偏差又高方差**：$J_{train}$ 高，$J_{cv}$ 更高（更多见高维数据）。
- **正好合适**：$J_{train}$ 低，$J_{cv}$ 也低。

直觉：高偏差 = 即使在训练集上也表现不好；高方差 = 在训练集上表现好但在验证集上差很多。

## 75 — 12.2 正则化和偏差/方差（10:37）

正则化参数 λ 对偏差和方差的影响：
- λ 太小：容易过拟合（高方差），$J_{train}$ 小，$J_{cv}$ 大。
- λ 太大：容易欠拟合（高偏差），$J_{train}$ 大，$J_{cv}$ 大。
- 中间的 λ：平衡偏差和方差。

选择 λ 的方法：尝试一组 λ 值，对每个 λ 训练模型，用交叉验证集选 $J_{cv}$ 最低的，再用测试集验证。

训练误差 $J_{train}$ 在计算时不含正则化项，只用原始损失。交叉验证误差和测试误差也一样。

## 76 — 12.3 建立表现基准（09:26）

如何判断 $J_{train}$ 和 $J_{cv}$ 的值是高还是低？需要建立基线（baseline）：
- 人类水平的表现
- 竞争算法的表现
- 基于经验的猜测

有了基线后：
- $J_{train}$ 与基线的差距 = 偏差问题的大小
- $J_{cv}$ 与 $J_{train}$ 的差距 = 方差问题的大小

例子：语音识别中人类错误率 10.6%，$J_{train}=10.8%$（偏差 0.2%，小），$J_{cv}=14.8%$（方差 4%，大）→ 高方差问题。

## 77 — 12.4 学习曲线（12:14）

学习曲线（Learning Curve）是横轴为训练集大小、纵轴为误差的曲线。

**高偏差的学习曲线**：$J_{train}$ 和 $J_{cv}$ 都很高且在某个点后趋于平稳（称为 plateaus）。增加更多数据不会显著改善高偏差问题，需要换更复杂的模型。

**高方差的学习曲线**：$J_{train}$ 很低，$J_{cv}$ 高，两条曲线之间有很大的差距。增加更多数据会缩小差距，因为更多数据帮助模型更好地泛化。

学习曲线是判断是否需要更多数据的重要工具。

## 78 — 12.5 再次决定下一步做什么（08:47）

根据偏差/方差诊断结果来选择行动：

**高偏差（欠拟合）时**：
- 增加更多特征 ✓
- 增加多项式特征 ✓
- 减小 λ ✓
- 获取更多数据 ✗（没用）

**高方差（过拟合）时**：
- 获取更多训练数据 ✓
- 减少特征集 ✓
- 增大 λ ✓
- 减少特征 ✗（可能会丢失有用信息，但也是一种方法）

## 79 — 12.6 偏差/方差与神经网络（10:45）

神经网络提供了一个简单的方法来处理偏差-方差权衡：

大型神经网络（很多层/很多神经元）是低偏差的模型，原则上不容易欠拟合。如果能适当正则化，也不会过拟合。

实践策略：先训练一个大的神经网络看是否在训练集上表现好（检查是否高偏差）。如果训练集表现良好且验证集也表现良好 → 完成。如果过拟合（高方差），加正则化或更多数据。

通常只要正则化恰当地选择，更大的神经网络几乎总是能做得好或更好。神经网络不遵循传统机器学习中"模型越大越容易过拟合"的直觉。

TensorFlow 代码中加 L2 正则化：
```python
model.add(Dense(units=25, activation='relu',
                kernel_regularizer=tf.keras.regularizers.l2(0.01)))
```

## 80 — 13.1 机器学习的迭代发展（07:43）

构建 ML 系统的迭代循环：
1. 决定系统的架构（模型、数据、超参数等）
2. 实现并训练模型
3. 进行误差分析（诊断偏差/方差、错误模式）
4. 根据分析结果决定下一步改进方向
5. 回到第 1 步

这个循环可以非常快（几个小时训练完成）或非常慢（几天/几周），模型训练速度直接决定了能多快地通过这个循环来提升性能。

## 81 — 13.2 误差分析（08:21）

误差分析（Error Analysis）：手动检查算法出错的样本，发现错误模式，指导下一步改进方向。

流程：从交叉验证集中挑出被错误分类的 ~100 个样本，分类统计错误类型。例如在垃圾邮件过滤中，统计：
- 有多少是钓鱼邮件分类错误的
- 有多少是假药广告分类错误的
- 有多少是伪装成正常邮件的垃圾邮件

看哪类错误最多，就优先优化哪一类。如果某一类错误只占很小比例，花大量时间优化它带来的整体收益很低。

误差分析还有一个功能：帮助决定是否需要给数据添加某些新标签。

## 82 — 13.3 添加数据（14:24）

获取更多数据的方法：

**数据增强（Data Augmentation）**：从已有数据生成更多训练样本。
- 图像：旋转、缩放、裁剪、翻转、改变对比度等。
- 音频：加入背景噪声、变速等。

**数据合成（Data Synthesis）**：用技术手段生成新数据。如 OCR 中用不同字体在屏幕上渲染文字生成合成数据。

添加数据的原则：
- 添加所有类型的数据都可以（全量收集）
- 也可以根据误差分析结果，针对性添加模型表现最差的那类数据

**迁移学习（引出）**：如果获取更多数据的方法有限，可以借助其他任务的数据，这是下一节的内容。

## 83 — 13.4 迁移学习：使用其他任务中的数据（12:11）

迁移学习（Transfer Learning）的核心思想：从相关任务中借用在大型数据集上预训练好的模型。

**流程**：
1. **预训练（Pre-training）**：在大数据集（如 ImageNet 的 100 万张图）上训练神经网络。
2. **微调（Fine-tuning）**：用你自己小数据集的参数替换输出层，可以只训练输出层参数，也可以训练所有参数。

例子：用 ImageNet 预训练的网络，将其用到自己的 X 光图像识别中。即使只有少量 X 光图像，也能取得好的效果。

**为什么有效**：预训练网络的前几层学到了通用的特征（如检测边缘、纹理），这些特征在很多图像任务中都通用。

微调有两种方式：
- 只训练输出层参数（数据量很少时）
- 训练所有参数，但用较小的学习率（数据量稍多时）

迁移学习需要注意输入数据格式的一致性（如都是图像，尺寸相同）。

## 84 — 13.5 机器学习项目的完整周期（08:45）

一个 ML 项目从概念到上线的完整流程：

1. **确定项目范围（Scoping）**：决定要做什么、定义目标。
2. **收集数据（Data Collection）**：确定需要什么数据，采集数据并标注。
3. **训练模型（Model Training）**：训练、误差分析、迭代优化。可能会回到步骤 2 收集更多数据。
4. **部署到生产环境（Deployment）**：将模型做成推断服务器，通过 API 供应用调用。

**部署后**还需要：
- **监控系统性能**：持续追踪模型表现的各项指标。
- **维护系统**：当性能下降时做模型更新。因为数据分布会随时间变化（数据漂移）。
- 生产环境的数据（用户授权前提下）可以用于后续模型改进。

**MLOps**（机器学习运维）：指系统地构建、部署和维护机器学习系统的实践。当服务大规模用户时，需要专门的软件工程团队来处理可靠性、扩展性、计算成本等问题。

## 85 — 13.6 公平、偏见与伦理（09:56）

构建 ML 系统时必须考虑的伦理问题：
- 数据偏见：如果训练数据存在对某些人群的偏见，模型也会继承甚至放大这些偏见。
- 算法公平性：模型不应因种族、性别等敏感属性做出歧视性决策。
- 透明性和可解释性：用户有权知道算法如何做决策。
- 数据隐私：收集和使用用户数据需要获得同意并保护隐私。

团队中应该有不同的视角和背景的人来讨论伦理问题，在项目开始前就设定伦理准则。

## 86 — 14.1 倾斜数据集的误差指标（11:36）

**偏斜数据集问题**：当正例非常少时（如罕见病只有 0.5% 的人得），一个只输出 0 的模型也有 99.5% 准确率，但这个模型毫无用处。

引入更有效的评估指标：

**混淆矩阵（Confusion Matrix）**：
| | 预测正 | 预测负 |
|---|---|---|
| 实际正 | TP（真阳性） | FN（假阴性） |
| 实际负 | FP（假阳性） | TN（真阴性） |

**精确率（Precision）**：预测为正中，实际为正的比例 = TP / (TP + FP)。当算法说"有疾病"时，有多大把握没错。

**召回率（Recall）**：实际为正中，被正确检测的比例 = TP / (TP + FN)。得了这种病的人中，有多大比例被检测出来。

精确率和召回率能更好地反映模型在偏斜数据集上的真实表现。预测全部为 0 的模型召回率为 0 且精确率无定义，能立即暴露问题。

## 87 — 14.2 精确率与召回率的权衡（11:50）

精确率和召回率之间存在权衡：提高阈值 → 提高精确率、降低召回率；降低阈值 → 降低精确率、提高召回率。

调节阈值的选择取决于应用场景：
- 如果误诊代价高（如做有创手术），提高阈值，追求高精确率。
- 如果漏诊代价高（如不治疗后果严重），降低阈值，追求高召回率。

**F1 Score**：自动结合精确率和召回率的单一指标：
$$F_1 = \frac{1}{\frac{1}{2}(\frac{1}{P} + \frac{1}{R})} = \frac{2PR}{P+R}$$

F1 Score 是精确率和召回率的调和平均（harmonic mean），更关注较小的值。当 P 或 R 很低时，F1 也很低，确保两项指标都不极端时得分才高。

用于比较不同算法：计算每个算法的 F1 Score，选最高的。

## 88 — 15.1 决策树模型（07:06）

决策树（Decision Tree）是一种可用于分类和回归的学习算法。

**决策树的结构**：
- **根节点（Root node）**：树的起始点，包含所有训练数据
- **决策节点（Decision node）**：根据某个特征的值进行分支，分支有左（是）和右（否）
- **叶节点（Leaf node）**：最终做出预测的节点

决策过程：从根节点开始，根据测试样本的特征值逐层判断，沿着树的分支走到叶节点，叶节点输出预测结果。

例子：分类猫/狗的决策树，根节点分叉在"耳朵形状是否尖的"，左侧（尖耳朵）的样本再按"脸型是否圆"分叉，等等。

## 89 — 15.2 学习过程（11:21）

构建决策树的核心问题：在每一个节点，选择哪个特征来分裂？

**决策树学习的目标**：选出的特征能够最大化"纯度"（purity），即让分裂后的子集尽可能同质（都是同一类）。

**选择特征的原则**：对每一个可能的特征，计算分裂后的信息增益（information gain），选择信息增益最大的特征。

构建决策树的整体思路：从根节点开始，使用所有训练样本；选择最佳分裂特征进行分裂，生成左右子节点；在每个子节点上递归重复这个过程；直到满足停止条件。

**停止条件**：
- 节点中所有样本属于同一类（纯度为 100%）
- 达到预设的最大深度
- 信息增益低于某个阈值
- 节点中的样本数低于某个阈值

## 90 — 16.1 测量纯度（07:50）

使用**熵（Entropy）**来度量不纯度（impurity）。

熵的定义：
$$H(p_1) = -p_1 \log_2(p_1) - (1-p_1) \log_2(1-p_1)$$

其中 $p_1$ 是该节点中正例（如猫）的比例。

如果 $p_1=0$，使用 $0 \log 0 = 0$ 这个约定。

熵的性质：
- $p_1 = 0$ 或 $p_1 = 1$（完全纯）：熵 = 0（最小不纯度）
- $p_1 = 0.5$（各一半）：熵 = 1（最大不纯度）
- 越接近 0 或 1，熵越小，数据集越纯

这个定义与信息论中的熵一致，值越大越混乱（不纯）。

## 91 — 16.2 选择拆分：信息增益（11:52）

**信息增益（Information Gain）**：衡量分裂后熵下降了多少。

信息增益 = 根节点的熵 - 左右子节点的加权平均熵
$$\text{IG} = H(p_1^{\text{root}}) - \left( w^{\text{left}} H(p_1^{\text{left}}) + w^{\text{right}} H(p_1^{\text{right}}) \right)$$

其中 $w^{\text{left}}$ 和 $w^{\text{right}}$ 是进入左/右子节点的样本比例。

计算过程：
1. 对每个候选特征，计算如果按该特征分裂后的信息增益
2. 选择信息增益最大的特征作为分裂特征

信息增益越大，说明分裂后纯度提升越多。

为什么用信息增益而不用直接比较加权熵？
- 信息增益给了停止分裂的依据：如果信息增益太小（低于阈值），说明继续分裂收益不大，反而增加了模型复杂度，可能导致过拟合。

## 92 — 16.3 整合（09:29）

构建决策树的完整递归算法：

1. **从根节点开始**，包含所有训练样本
2. **计算所有可能特征的信息增益**，选择信息增益最高的特征进行分裂
3. **创建左右分支**，按特征值将样本分配到对应分支
4. **递归在左右分支上重复步骤 2-3**
5. **当满足停止条件时**（见视频 #89 的四个条件），创建叶节点并预测该节点中最多数的类别

这是一个**递归算法**（recursive algorithm）：构建整棵树 = 构建更小的子树 + 组合起来。

**最大深度参数**的选择：
- 深度越大 → 模型越复杂 → 越容易过拟合
- 可以用交叉验证来选择最佳深度
- 开源库通常有不错的默认值

这个视频总结了从原始训练集到完整决策树的整个构建流程。

## 93 — 16.4 使用 one-hot 编码处理分类特征（05:26）

当分类特征可以取 k 个值（k > 2）时，使用**独热编码（One-Hot Encoding）**将特征转换为 k 个二进制特征。

例子：耳朵形状除了"pointy"和"floppy"，还有"oval"（共 3 个取值）。将其转换为：
- 特征 1（尖耳朵）：有则 1，否则 0
- 特征 2（垂耳朵）：有则 1，否则 0
- 特征 3（椭圆耳朵）：有则 1，否则 0

每个样本在三个特征中恰好有一个为 1（hot），其余为 0，因此得名 one-hot。

One-hot 编码不仅用于决策树，也用于神经网络和逻辑回归中处理分类特征。

## 94 — 16.5 连续值特征（06:54）

当特征是连续值时（如动物的体重），决策树也可以在连续特征上分裂。

**方法**：选择一个阈值，按"特征值 ≤ 阈值"分成左右子树。

**如何选择阈值**：
1. 将所有样本按该连续特征的值排序
2. 取排序后相邻值的中点作为候选阈值
3. 对每个候选阈值计算信息增益
4. 选择信息增益最高的阈值

如果有 10 个训练样本，就会有 9 个候选阈值（每个相邻值的中间点）。

如果该连续特征的信息增益高于其他所有特征，就选择该特征和最优阈值进行分裂。

例如：体重 ≤ 9 磅 → 左子树，体重 > 9 磅 → 右子树。

## 95 — 16.6 回归树（Optional）（09:51）

决策树也可以用于回归问题（预测数值）。

**与分类树的区别**：
- 分类树：用熵和信息增益来选择分裂特征
- 回归树：用**方差（Variance）**和**方差的减少（Reduction in Variance）**来选择分裂特征

**叶节点预测值**：叶节点中所有训练样本的目标值的平均数。

**方差减少的计算**（类似信息增益）：
$$\text{Reduction} = \text{Var}^{\text{root}} - \left( w^{\text{left}} \cdot \text{Var}^{\text{left}} + w^{\text{right}} \cdot \text{Var}^{\text{right}} \right)$$

选择使方差减少最大的特征来分裂。

直觉：回归树希望每个叶子节点中的 y 值变化不大（方差小），这样取均值作为预测值时才精确。

## 96 — 17.1 使用多个决策树（03:57）

**单棵决策树的弱点**：对数据的微小变化高度敏感。改变一个训练样本可能会导致根节点分裂的特征完全改变，进而生成完全不同的树。

**解决方案：树集成（Tree Ensemble）**——训练多棵决策树，让它们投票决定最终预测。

投票机制：
- 给一个测试样本，让集成中的每棵树都做出预测
- 多数票决定的类别作为最终预测
- 这使得整体算法对单棵树的行为不那么敏感，更加鲁棒

但问题在于，怎么从同一个训练集中生成多棵不同（稍有差异）的决策树？这是后续视频的内容。

## 97 — 17.2 放回抽样（04:00）

**Sampling with Replacement（有放回抽样）**：构建不同训练集的关键技术。

过程：
- 将 M 个训练样本放入一个"虚拟袋"中
- 每次随机抽取一个样本，记录后放回袋中
- 重复 M 次，生成一个同样大小的新训练集
- 新训练集中有的样本可能出现多次，有的可能完全不出现

用这种有放回抽样生成 B 个不同的训练集，在各自的训练集上训练 B 棵不同的决策树。这些训练集相似但不完全相同，因此生成的树也各不相同。

这种技术是从统计学中来的（Bootstrap）。

## 98 — 17.3 随机森林算法（06:23）

**随机森林（Random Forest）**是 bagged decision tree 的改进版。

**Bagged Decision Tree**（袋装决策树）：
1. 有放回抽样 B 次，生成 B 个训练集
2. 在每个训练集上训练一棵决策树
3. 所有树投票得出最终预测

B 的典型取值在 64 到 228 之间，设为 100 是常见选择。B 更大不会损害性能但收益递减。

**随机森林的关键改进**：在每个节点选择分裂特征时，不是从所有 n 个特征中选，而是从随机选出的 k 个特征子集中选。

典型值 $k = \sqrt{n}$（当 n 较大时）。

为什么这样做？因为即使使用了有放回抽样，根节点附近的分裂特征可能总是相同。随机化特征选择使每棵树更加不同，投票结果更稳健。

随机森林比单棵决策树更鲁棒、更准确。

## 99 — 17.4 XGBoost（07:26）

XGBoost（eXtreme Gradient Boosting）是目前最常用且最强大的决策树集成算法。开源实现运行快、效果好，在 Kaggle 竞赛中被广泛用于夺冠。

**Boosting 的核心思想（刻意练习）**：

不同于随机森林每次都从所有样本等概率抽样，Boosting 在构建第 b 棵树时，**增加前 b-1 棵树预测错误的样本被选中的概率**。这样新树会更关注之前表现不好的样本——类似弹钢琴时集中练习最难的部分。

**XGBoost 的特点**：
- 内置正则化防止过拟合
- 有很好的默认分裂标准和停止条件
- 不需要显式做有放回抽样，而是给不同样本分配不同权重
- 速度快、效率高

**代码用法**：

分类：
```python
from xgboost import XGBClassifier
model = XGBClassifier()
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
```

回归：
```python
from xgboost import XGBRegressor
model = XGBRegressor()
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
```

对于大多数应用，直接使用 XGBoost 即可。

## 100 — 17.5 什么时候使用决策树（06:55）

**决策树 vs 神经网络：如何选择？**

### 决策树 / 树集成
**优点**：
- 在**表格数据/结构化数据**（spreadsheet 形式）上表现好
- 训练速度快，允许快速迭代改进
- 小决策树可解释，人能看懂决策逻辑

**缺点**：
- 不适合非结构化数据（图像、视频、音频、文本），神经网络在这些领域更好
- 树集成较大时同样难以解释

**推荐做法**：树集成优于单棵决策树，建议使用 XGBoost。

### 神经网络
**优点**：
- 对所有类型数据都有效（结构化 + 非结构化 + 混合数据）
- 在图像、视频、音频、文本等非结构化数据上是首选
- 支持迁移学习，可用少量数据获得竞争性性能
- 可以将多个神经网络串联成更大的系统，一起用梯度下降训练

**缺点**：
- 训练速度可能较慢
- 大型网络训练时间长

### 选择总结
| 场景 | 推荐 |
|---|---|
| 表格数据/结构化数据 | 决策树/神经网络都可以，可尝试两者 |
| 图像/视频/音频/文本 | 神经网络 |
| 需要迁移学习 | 神经网络 |
| 需要快速训练迭代 | 决策树/XGBoost |
| 多个模型串联成系统 | 神经网络（可联合训练） |
