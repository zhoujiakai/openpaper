## 已掌握的关键知识点

- Depends() 声明式依赖：路由声明需要什么，框架负责准备
- yield 依赖：yield 前准备资源，yield 后清理资源，finally 保证必执行
- yield vs return：yield 能做清理（借还），return 只给不管还；不需要清理的用 return（如当前用户）
- 请求内缓存：同一依赖同一请求只执行一次
- 嵌套依赖：从叶子节点往上拓扑排序执行
- 类作为依赖：__init__ 参数可被解析即可
- Annotated 类型别名：把 Depends 封进类型别名复用
- 中间件洋葱模型：最后注册 = 最外层 = 最先进请求、最后出响应
- 中间件 vs 依赖注入：中间件是全局保安，依赖注入是按需领工具
- 认证用依赖注入不用中间件：部分路由需要认证，部分不需要
- lifespan 替代 on_event 三个优势：变量共享、关闭保证执行、ASGI 标准
- 应用级 vs 请求级：lifespan 管全局共享资源，yield 依赖管按请求借用的资源
- 请求模型：验证 + 类型转换 + 文档生成
- 响应模型：过滤输出字段，敏感数据不泄漏
- 请求/响应分两个类：请求有响应没有（password）、响应有请求没有（id）、类型可能不同
- response_model_exclude_unset：只返回实际设置了值的字段
- 异步 I/O：等 I/O 时去干别的，1 个事件循环线程处理几百并发
- 同步路由 def：在线程池（默认 40 线程）里跑，不会阻塞事件循环
- async vs def 路由：async 配异步库，不确定就写普通 def，别 async 里调阻塞代码
- FastAPI "快"的含义：I/O 并发高，不是单次执行快
- 后台任务：先返回响应，非关键操作之后慢慢做
- worker 数量：2 × CPU核心数 + 1
- WSGI vs ASGI：同步标准 vs 异步标准，就差一个 A（Asynchronous）
- Flask async：2.0+ 加了有限的 async 支持，但底层仍是 WSGI，不是原生异步

## 被纠正过的误区

- 认证应该用依赖注入，不应该用中间件（因为不是所有路由都需要认证）
- lifespan 和 yield 依赖的边界：应用级共享资源用 lifespan，请求级借用资源用 yield 依赖
- 当前用户信息是普通 return 依赖，不是 yield 依赖（不需要清理）
- 中间件注册顺序：最后注册 = 最外层，不是最先注册 = 最外层
- 中间件实际顺序示例：Auth（内层）→ CORS（中间层）→ Logging（外层），CORS 应在 Auth 外层
