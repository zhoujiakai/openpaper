## 摘要

讨论对象：序列转换模型。

旧方法：循环神经网络和卷积神经网络，包含编码器和解码器，最优秀的模型还会使用注意力机制将编码器和解码器连接起来。

新方法：论文提出：Transformer，只使用注意力机制，不再使用任何循环神经网络或卷积神经网络。

效果好：质量高、并行性好。泛化性好。

- 质量高：在 WMT 2014 英德翻译任务上，达到 28.4 BLEU，比最好的模型高 2 BLEU 以上。
- 并行性好：在 WMT 2014 英法翻译任务上，使用 8 块 GPU 上花费3.5天，达到 41.8 BLEU，成本仅是最好模型的一小部分。
- 泛化性好：在英语句法成分分析任务上，成功应用，无论训练数据是多是少。

## 2 背景

别人都试过什么？为什么还不够？

铺垫：CNN 路线，并行了，可是远距离连接仍需 O(n) 或 O(log n) 步

亮相：Transformer 的优势 O(1)，代价是平均，用多头弥补

来源：Self-attention 和 Memory Networks，注意力已经被用过，但没有当过主角

总结陈词：我们是第一个完全用 self-attention 的转换模型



## 3 模型架构

介绍了 Transformer 的构建方式：

1. 第一点是 Self-attention（自注意力机制），也就是堆叠的自注意力层，一层叠一层。
2. 第二点是逐位置的全连接层，即 FFN（前馈神经网络）。在该层中，每个 token 都会独立通过一个网络。
3. 第三点是编码器和解码器都采用了这两种组件。

这就是 Transformer 的构建方式。

### 3.1 编码器和解码器堆叠

#### 编码器

N = 6 堆叠

每层两个子层：Multi-head self-attention 和 FFN

每个子层外面都包着一层残差连接和层归一化，也就是 Residual Connection 和 Layer Normalization

所有子层的输出维度统一为 d_model = 512，这是为了残差连接能够相加。

#### 解码器

N = 6 堆叠

每层3个子层，比 Encoder 多了一个 Cross Attention，它关注的是 Encoder 的输出

使用 Masking 去修改自注意力层，不让当前位置看到后面的位置。

Offset by one position 是说输出嵌入偏移一个位置，也就是训练时把整个目标序列右移一位作为输入。

### 3.2 注意力

这里讲了注意力的通用定义：它的输入是一个 query 和一组 key-value pairs，输出是 values 的加权求和，而权重是 query 和每个 key 计算出的一个兼容度（compatibility）。

#### 3.2.1 缩放点积注意力

这一小节提出了 Transformer 的一个注意力公式，并与两种常见的注意力机制进行了对比：

1. Additive Attention（加性注意力）
2. Dot-Product Attention（点积注意力）

#### 3.2.2 多头注意力

这一个小节讲了多头注意力的五个步骤，就是将 512 维分成 8 个头，每个头是 64 维的向量去进行计算，最后再拼接起来。

#### 3.2.3 注意力在 Transformer 中的三种应用

这一小节讲了注意力在 Transformer 中的三种应用：
1. 编码器的自注意力
2. 解码器的自注意力
3. 交叉注意力

### 3.3 逐位置前馈网络

这一节讲的是前馈神经网络的计算。它是每个 Token 独自经过一个全连接网络。

### 3.4 嵌入和 Softmax

这一小节介绍了三个组件：

1. Embedding
   是将 Token 转换成 512 维的向量。

2. Softmax
   将 Decoder（也就是编码器）输出转换成词汇表上的概率分布。

3. 权重共享
   输入端的 Embedding、输出端的 Embedding 以及 Softmax 前的线性变换共用同一套权重。

### 3.5 位置编码

这一节主要讲解了以下内容：

1. 为什么要使用位置编码
   由于 Transformer 模型没有循环（RNN）和卷积（CNN）结构，它天生是无法识别序列顺序的，所以需要位置编码来提供位置信息。

2. 位置编码的实现方式
   位置编码向量会直接加到 Embedding（词嵌入）向量上。

3. 为什么选择正弦函数（Sinusoidal functions）
   (a) 泛化长度：如果使用神经网络去学习位置编码，它本质上是一个线性函数。对于训练时未见过的更长序列（即超过阈值或固定数值的 Token），模型将无法计算和处理。
   (b) 数学特性：使用正弦函数是通过数学公式计算的，不存在长度限制。
   (c) 相对位置：由于正弦函数是周期函数，这使得模型更容易学习到序列中的相对位置关系。



## 4 为什么用自注意力

这里对比了自注意力（Self-Attention）、循环神经网络（Recurrent）以及卷积神经网络（Convolutional）这三种网络的操作复杂度计算。

1. 并行性
   (a) Self-Attention 和 CNN 都是 $O(1)$
   (b) RNN 是 $O(n)$

2. 长程依赖
   (a) Self-Attention 是 $O(1)$
   (b) RNN 是 $O(n)$
   (c) CNN 是 $O(n/k)$

3. 计算量
   当 $n$ 远小于 $d$ 的时候，Self-Attention 的计算量比 RNN 更低（RNN 是 $O(n \cdot d^2)$），计算速度也更快。



## 5 训练

### 5.1 训练数据和批处理

在训练数据的组织上，要注意将序列长度近似的训练数据组织成同一批次（Batch）进行模型训练，这样可以避免浪费计算资源。

### 5.2 硬件和训练时间

这里提供了训练的数据：

1. Base 模型：训练了 12 个小时
2. Big 模型：训练了 3.5 天

以上训练均在一台 8 GPU 的机器上完成。

### 5.3 优化器

这一节主要讲解了优化器的学习率调度策略：先 Warmup 再衰减（Decay）。

具体过程如下：

1. 升温阶段（Warmup）：前 4000 步的学习率线性增长至峰值。
2. 衰减阶段：达到峰值后，学习率按步数的倒数平方根（Inverse Square Root）递减。

可以用一个具体的例子来类比：就像学车时先低速慢开，熟悉之后再加速；快到目的地时开始减速，最后精准地停到位。

### 5.4 正则化

这一小节主要讲述了论文中使用的两种正则化方法：

1. Residual Dropout
   (a) 在每个子层的输出上以及残差连接（Residual Connection）之前进行 Dropout，丢弃比率为 0.1。
   (b) 在嵌入层（Embedding）与位置编码（Positional Encoding）相加后的结果上，也会进行一次 Dropout。
   (c) 每次训练会随机丢弃 10% 的连接。

2. Label Smoothing（标签平滑）
   (a) 它的比率同样是 0.1。
   (b) 具体做法是将正确答案的概率从 1.0 降为 0.9，剩余的 0.1 概率分配给其他词汇。

通过这两种正则化手段，可以减轻模型的过拟合，并增强模型的泛化能力。

## 6 结果

### 6.1 机器翻译结果

在训练的后期，模型的 loss 会在一个最小值附近来回震荡，而不是稳定在一个点上。每个 checkpoint 都是在这个震荡中的某个瞬间状态，可能恰好处于一个局部高点或者低点。

所以我们通过平均最后几个检查点（checkpoint）的方式，把这个震荡平滑掉，从而得到一个更鲁棒的模型。

### 6.2 模型变体（消融实验）

这个消融实验通过比较以下几个变量来评估模型性能：

1. Attention Head：通过测试不同数量的头来对比效果
2. 向量维度：减少 Token 表示向量的维度（dk）
3. 模型尺寸：使用不同数量的层数进行实验
4. Dropout：分别测试 0.0、0.1 和 0.2 三种参数
5. 位置编码：对比查表法和 Sign 函数公式法

实验结果表明：
1. Attention Head：单个头的表现不太理想，最好的效果是 8 个头。
2. 向量维度：如果不使用 512 维而是减少维度，会损害模型的质量。
3. 模型大小：实验结果表明，越大的模型效果越好。
4. Dropout：使用随机 Dropout 且参数为 0.1 时，效果非常出色。
5. 位置编码：使用函数公式法比查表法要好得多。

### 6.3 英语成分句法分析

在英语成分句法分析任务中，论文实验使用的是 4 层 Transformer，而非翻译任务中常用的 6 层架构。这主要出于以下考量：

1. 数据量限制：该任务的数据量仅为 4 万句，约为英德翻译数据的 1%。在数据量较小的情况下，使用 6 层的大模型容易导致过拟合。
2. 模型性能与参数量的平衡：使用 4 层是一个合理的折中方案。
   (a) 性能充足：该配置的性能已经超过了专门的语法分析器。
   (b) 参数量较少：较低的参数量使其在小数据集上更不容易发生过拟合。

总的结论是：模型的大小应当与训练数据量相匹配。

## 7 结论

作者为这篇论文做了总结：

1. 核心贡献
   我们提出了第一个完全基于注意力机制的序列转换模型。

2. 实验效果
   模型的训练速度更快，并且在翻译任务中刷新了 SOTA 记录。

3. 未来展望
   (a) 计划将模型扩展到文本以外的模态，例如图像、音频和视频。
   (b) 研究局部受限的注意力机制，以处理长输入。

这正好对应了我们之前讨论过的 Transformer 复杂度为 O(n²) 的问题。

以上就是论文的总结。
