## 薄弱知识点强化建议

### await 底层机制（🟡）

- 遗漏点：I/O 操作的 await 是在 Future 上注册回调，不是放进 _scheduled；只有 asyncio.sleep 这种定时任务才进 _scheduled
- 建议记忆方法：按操作类型分——I/O 操作走 epoll 通知回调，定时操作走 _scheduled 堆排序到期回调
- 口诀："I/O 等 epoll 叫，定时等闹钟响"

### 协程 vs 事件循环（🟡）

- 遗漏点：把协程描述成了事件循环的行为
- 建议：严格区分——协程 = 能暂停的函数（async def），事件循环 = 调度器，Task = 协程的包装器
- 类比：协程是工人，Task 是工牌，事件循环是工头

## 被纠正过的误区

1. 多线程受 CPU 核心数限制 → 线程受内存限制，受核心数限制的是多进程
2. 协程自动检测 I/O 并切换 → 协程靠代码中的 await 显式切换，不会自动识别 I/O
3. GIL 阻止多线程用多核 → GIL 只阻止同时执行 Python 字节码，I/O 和 C 扩展会释放 GIL
4. 协程 = 事件循环 → 协程是能暂停的函数，事件循环是调度器
