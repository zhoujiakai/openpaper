## 薄弱点强化建议

### 1. ICL 定义（被纠正 1 次）
- **问题**：遗漏核心定义"不更新参数"
- **口诀**：ICL = **不改权重** + 提供示例 + 学会新任务
- **强化方式**：下次复习时先默写 ICL 定义

### 2. Few-Shot 核心价值（被纠正 2 次）
- **问题**：反复表述为"学会新任务"
- **口诀**：Few-Shot 不教新知识，只说"我要什么格式"
- **本质**：消除歧义 + 激活预训练模式，不是传授知识
- **强化方式**：这个点被纠正了 2 次仍出错，建议重点记忆

### 3. System/User Prompt 功能维度（被纠正 1 次）
- **问题**：只提到权重差异，未覆盖功能维度
- **记忆框架**：
  - System = 公司制度（角色、格式、规则）→ 持久不改
  - User = 每天任务（问题、数据、指令）→ 每轮可变
