### Q1 [选择题] `sizeof` 在数组和指针上的区别

```c
int arr[10];
int *p = arr;
printf("%zu %zu\n", sizeof(arr), sizeof(p));
```

64 位系统上输出什么？

A. `10 10`
B. `40 8`
C. `80 8`
D. `40 40`

- **正确答案**：B
- **评分要点**：选择正确即通过
- **知识点**：数组名 vs 指针 —— `sizeof(arr)` 不退化，拿到整个数组大小（10×4=40）；`sizeof(p)` 拿到指针本身大小（64位=8字节）

### Q2 [选择题] 指针解引用的副作用

```c
int x = 5;
int *p = &x;
*p = 10;
```

执行后 `x` 的值是多少？

A. `5`
B. `10`
C. 未定义
D. 编译错误

- **正确答案**：B
- **评分要点**：选择正确即通过
- **知识点**：解引用 —— p 存的是 x 的地址，`*p = 10` 直接写入 x 的内存位置，x 同步变为 10

### Q3 [选择题] 宏展开陷阱

```c
#define SQUARE(x) x * x
int result = SQUARE(2 + 3);
```

`result` 的值是？

A. `25`
B. `11`
C. `10`
D. 编译错误

- **正确答案**：B
- **评分要点**：选择正确即通过
- **知识点**：宏是纯文本替换 —— 展开后 `2 + 3 * 2 + 3`，乘法优先得 11。正确写法 `#define SQUARE(x) ((x) * (x))`

### Q4 [简答题] 数组退化成指针

解释为什么函数参数写成 `void func(int arr[])` 等价于 `void func(int *arr)`，以及传数组时还需要传什么额外信息？

- **参考答案**：
  1. C 标准规定函数参数里的数组类型编译器自动替换为指针，方括号里的数字被忽略
  2. 传入后 `sizeof(arr)` 永远是 8（指针大小），不知道数组真实大小
  3. 必须额外传入元素个数：`void func(int *arr, size_t len)`
- **评分要点**：
  1. 必须提到编译器自动替换规则
  2. 需要额外传长度参数
- **知识点**：数组退化 —— 数组名在表达式/传参中退化成首元素地址

### Q5 [简答题] malloc 后的内存状态

malloc 申请内存后，能直接使用这块内存里的数据吗？为什么？如果不想手动清零有什么替代方案？

- **参考答案**：
  1. 不能直接用，malloc 不初始化内存，里面是上次释放时残留的垃圾值
  2. 替代方案：用 `calloc(5, sizeof(int))`，自动清零
  3. 如果申请后立刻逐元素赋值，则不需要先清零再写（浪费）
- **评分要点**：
  1. 必须提到 malloc 不初始化
  2. 必须提到 calloc 作为替代
- **知识点**：malloc（不初始化）vs calloc（清零）

### Q6 [简答题] 返回栈变量指针

以下代码有什么问题？

```c
int *create_value() {
    int val = 100;
    return &val;
}
```

- **参考答案**：
  `val` 在栈上，函数返回后被回收，调用者拿到的指针指向已释放的空间。后续任何函数调用都可能覆盖这片栈空间，读到什么不可预测。这是**栈的黄金规则**：永不可返回指向栈变量的指针。正确做法是用 `malloc` 在堆上分配再返回。
- **评分要点**：
  1. 识别出返回了栈变量地址
  2. 说明函数返回后内存被回收
  3. 提到正确做法（堆分配）
- **知识点**：栈变量生命周期 —— 函数返回即回收，悬垂指针

### Q7 [简答题] const 的两种写法

`const int *p` 和 `int *const p` 有什么区别？分别说明哪种操作被禁止。

- **参考答案**：

  | 写法 | 禁止 | 允许 |
  |------|------|------|
  | `const int *p` | `*p = 10`（改值） | `p = &y`（重指向） |
  | `int *const p` | `p = &y`（重指向） | `*p = 10`（改值） |

  口诀：**const 在 `*` 左边锁值，在 `*` 右边锁针。**

- **评分要点**：
  1. 必须正确区分两种写法的禁止操作
  2. 理解 const 相对 `*` 的位置决定锁谁
- **知识点**：const 与指针的组合 —— 声明读法

### Q8 [简答题] 位运算设定位

要把一个寄存器的第 5 位设为 1，同时不影响其他位，写一行 C 代码。（假设已定义 `BIT(n) = (1 << n)`）

- **参考答案**：
  `reg |= BIT(5);` 或 `reg |= (1 << 5);`
  注意：`BIT(1) << 5` 等于 `BIT(6)` 不是 `BIT(5)`，因为移位会累加。
- **评分要点**：
  1. 使用 OR 运算符
  2. 比特位号正确（第 5 位是 `1<<5`）
- **知识点**：位运算设 1 = OR，BIT 宏用法

### Q9 [对比题] 栈 vs 堆

从分配方式、生命周期、大小、速度、增长方向五个维度对比栈和堆。

- **参考答案**：

  | 维度 | 栈 | 堆 |
  |------|----|----|
  | 分配方式 | 编译器自动 | 手动 malloc/free |
  | 生命周期 | 函数返回即回收 | 直到 free |
  | 大小 | 有限（通常 8MB） | 几乎受限于系统总内存 |
  | 速度 | 快（移动栈指针） | 慢（查找空闲块） |
  | 增长方向 | 高地址→低地址 | 低地址→高地址 |

- **评分要点**：至少覆盖 3 个以上维度，包含分配方式和生命周期
- **知识点**：内存布局 —— 栈 vs 堆

### Q11 [应用题] 指针版 swap

写一个 C 函数 `void swap(int *a, int *b)`，交换两个整数的值。

- **参考答案**：

  ```c
  void swap(int *a, int *b) {
      int tmp = *a;
      *a = *b;
      *b = tmp;
  }
  ```

  核心逻辑：暂存 `*a` → `*a` 写为 `*b` → `*b` 写为暂存值。

- **评分要点**：
  1. 参数必须是指针
  2. 通过解引用交换值
  3. 使用中间变量暂存
- **知识点**：指针作为函数参数 —— 修改外部变量的值

### Q12 [应用题] 结构体 + qsort 排序

写一个 `Student` 结构体（name 字符串、id 整数、score 浮点数），再写一个比较函数配合 `qsort` 按 score 降序排列。

- **参考答案**：

  ```c
  typedef struct {
      char *name;
      int id;
      double score;
  } Student;

  int cmp_score_desc(const void *a, const void *b) {
      const Student *s1 = (const Student *)a;
      const Student *s2 = (const Student *)b;
      if (s1->score < s2->score) return 1;   // s1 小，排后面（降序）
      if (s1->score > s2->score) return -1;  // s1 大，排前面
      return 0;
  }

  // 调用
  qsort(students, n, sizeof(Student), cmp_score_desc);
  ```

  注意：`score` 是 double，不能用 `s2->score - s1->score`（差值转 int 丢失精度）。

- **评分要点**：
  1. 结构体定义正确
  2. 比较函数签名正确（`const void *` 参数）
  3. 强转保留 const
  4. double 比较用 if-else 而非相减
- **知识点**：结构体 + 函数指针 + qsort 回调
