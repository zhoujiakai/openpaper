## 已掌握的关键知识点

### 指针系统
- `*` 双重角色：声明时描述类型，使用时解引用
- 所有指针在 64 位系统上均为 8 字节，类型只影响解引用规则
- 函数传参永远是值传递，改外部变量需传指针

### 数组
- 数组名 ≠ 指针：sizeof 下不退化，`&arr` 也不退化
- 传数组给函数必退化成指针，必须额外传长度
- 字符串 = `\0` 结尾的字符数组

### 动态内存
- malloc 不初始化（垃圾值），calloc 清零
- 每个 malloc 必有一个 free；free 后置 NULL 防 double free
- 永不可返回栈上变量的指针

### 结构体
- 成员按大小降序排列可省内存
- 对齐规则：成员起始地址 = 自身大小的整数倍

### 位运算
- 设 1 用 OR，清 0 用 AND NOT，翻转用 XOR
- 多位域操作：先清除再写入

### 宏
- 纯文本替换，所有参数和整体必须加括号
- 能用 inline 时优先 inline

## 被纠正过的误区

- ~~`int *p` 是 32 位~~ → 指针变量在 64 位系统是 8 字节，`int` 描述的只是指向目标的类型
- ~~`c` 产出 .o 是汇编阶段~~ → 平时 `gcc -c` 实际是编译+汇编合并，单独汇编是 `.s` → `.o`
- ~~unsigned int 范围是"32 位"~~ → 范围是 `0 ~ 2³²-1`，32 位只是宽度
- ~~数组优化规则"只能从大变小"~~ → 对齐规则是成员起始地址为自身大小的整数倍，大小降序只是优化策略而非规则
- ~~函数指针参数不检查类型~~ → `(int, int)` 是类型系统的一部分，编译器严格执行检查
- ~~BIT(1) << 5 等于 BIT(5)~~ → 移位会累加，BIT(1)<<5 = 1<<(1+5) = 1<<6 = BIT(6)
- ~~`const` 强转时不需要保留 `const`~~ → 丢掉 const 编译器会警告，比较函数只读数据应保留
