## 8 — 2.1二元分类（08:24）
- [x] 4.9%

二元分类：输出标签y∈{0,1}。符号约定：X为(nx,m)矩阵（nx=特征维度，m=样本数），Y为(1,m)标签矩阵。以图片识别猫为例：将RGB三通道像素(64×64×3=12288)拉成一维特征向量。目标：学习分类器，输入x输出ŷ=P(y=1|x)。

## 9 — 2.2逻辑回归（06:00）
- [x] 5.3%

逻辑回归=二元分类的基础算法。公式：ŷ = σ(wᵀx + b)，其中σ(z)=1/(1+e⁻ᶻ)为sigmoid函数，输出[0,1]的概率值。参数：权重w(nx维向量)和偏置b(标量)。当ŷ≥0.5时预测y=1，否则y=0。b类似于线性回归中的截距项，独立处理更方便向量化实现。

## 10 — 2.3逻辑回归损失函数（08:12）
- [x] 5.8%

损失函数L(ŷ,y) = -[y·log(ŷ) + (1-y)·log(1-ŷ)]。直觉：y=1时L=-log(ŷ)→让ŷ趋近1；y=0时L=-log(1-ŷ)→让ŷ趋近0。代价函数J(w,b)=1/m·ΣL，是所有样本的平均损失。此损失函数来自最大似然估计，凸函数保证梯度下降找到全局最优。不要用平方误差，会导致非凸优化问题。

## 11 — 2.4梯度下降（11:24）
- [x] 6.4%

梯度下降迭代更新：w:=w-α·∂J/∂w，b:=b-α·∂J/∂b。α是学习率（超参数），控制每步更新的大小。∂J/∂w表示J对w的偏导数（斜率方向）。算法沿负梯度方向（最陡下降方向）更新参数，逐步收敛到最小值。计算图是理解梯度的有效工具。

## 12 — 2.5导数（07:11）
- [x] 6.9%

导数回顾：f(a)=3a → df/da=3（斜率处处为3）。f(a)=a² → df/da=2a（斜率随a变化）。导数=函数在一点的瞬时变化率≈Δf/Δa当Δa→0。在计算图中，"dvar"表示最终输出对变量var的偏导数（即反向传播中的梯度传递）。

## 13 — 2.6更多导数示例（10:28）
- [x] 7.5%

常见导数：f(a)=log(a)→df/da=1/a；f(a)=a³→3a²；f(a)=c（常数）→0。链式法则：如J(a,b,c)=3(a+bc)，令u=bc，v=a+u，J=3v。则dJ/da=3·1·1=3，dJ/db=3·1·c=3c，dJ/dc=3·1·b=3b。反向传播=从输出端逐步向输入端计算梯度链式传递。

## 14 — 2.7计算图（03:34）
- [x] 7.7%

计算图=将复杂函数分解为基本运算节点构成DAG（有向无环图）。例如J(a,b,c)=3(a+bc)：节点1:u=bc→节点2:v=a+u→节点3:J=3v。正向传播=从左到右计算中间值和最终输出。反向传播=从右到左计算梯度。计算图使梯度计算系统化、模块化，是深度学习框架（TensorFlow/PyTorch）的核心抽象。

## 15 — 2.8计算图求导数（14:35）
- [x] 8.5%

反向传播的核心：从输出（J）开始逐个节点计算∂J/∂var。链式法则：若var影响某个中间变量mid，而mid影响J，则∂J/∂var=(∂J/∂mid)·(∂mid/∂var)。在代码中，dvar存储∂J/∂var的值。计算图方法与符号约定是理解神经网络反向传播的关键基础。

## 16 — 2.9逻辑回归梯度下降（06:43）
- [x] 8.9%

单个样本的反向传播：z=wᵀx+b → a=σ(z) → L(a,y)。梯度：da=-(y/a-(1-y)/(1-a))，dz=a-y（神奇的简化！源于sigmoid+交叉熵的组合性质），dw=x·dz，db=dz。更新：w:=w-α·dw，b:=b-α·db。dz=a-y这个简洁形式是sigmoid搭配交叉熵损失的最大似然推导结果。

## 17 — 2.10m示例上的梯度下降（08:01）
- [x] 9.4%

m个样本的批量梯度下降：(1)初始化J=0,dw=0,db=0(2)遍历每个样本累加损失和梯度(3)J/=m,dw/=m,db/=m(4)更新参数。缺点：显式for循环效率低，需要向量化提速。两个for循环：外层迭代轮次(epoch)，内层遍历m个样本。后面将用向量化消除内层循环。

## 18 — 2.11向量化（08:05）
- [x] 9.9%

向量化=用矩阵运算替代显式for循环：z=wᵀX+b一次计算所有m个样本。NumPy利用CPU/GPU的SIMD并行指令，实测速度可达for循环的300+倍。经验法则：任何时候避免显式for循环，用NumPy内置向量/矩阵运算。示例：np.dot(w.T, X)比逐元素计算快百倍。GPU比CPU更适合大规模矩阵乘法。

## 19 — 2.12更多向量化示例（06:20）
- [x] 10.2%

NumPy向量化操作示范：np.exp(v)→逐元素指数；np.log(v)→逐元素对数；np.abs(v)→逐元素绝对值；v**2→逐元素平方；np.maximum(v,0)→ReLU。原则：当需要对数组每个元素做运算时，使用NumPy内置函数，不要写for循环。内置函数底层是C/Fortran实现的向量化操作。

## 20 — 2.13向量化逻辑回归（07:33）

## 21 — 2.14向量化逻辑回归的梯度输出（09:38）
- [x] 11.2%

向量化反向传播：dZ=A-Y (所有样本的dz)，dw=1/m·X·dZᵀ，db=1/m·np.sum(dZ)。一步完成所有样本梯度计算。实现流程：(1)正向计算A(2)计算dZ=A-Y(3)计算dw,db(4)更新参数。单次迭代无任何for循环。多轮迭代的外层for循环暂时无法消除。

## 22 — 2.15Python中的广播（11:07）
- [x] 11.9%

广播规则：(m,n)矩阵与(1,n)向量运算时，后者自动复制m行→(m,n)再进行逐元素运算。同理(m,n)与(m,1)也可广播。标量与矩阵运算总是可以广播。示例：计算各食物中营养素占比：cal=A.sum(axis=1)→percentage=100*A/cal.reshape(3,1)。reshape确保列向量形状，避免广播歧义。

## 23 — 2.16python numpy 向量的注释（06:50）
- [x] 12.3%

## 25 — 2.18逻辑回归损失函数的解释（选修）（07:15）
- [x] 12.9%

损失函数推导：P(y|x)=ŷʸ·(1-ŷ)^(1-y)（当y=1时P=ŷ，y=0时P=1-ŷ）。取对数得logP=y·logŷ+(1-y)·log(1-ŷ)。损失函数=-logP（负对数似然），最小化损失=最大化概率。代价函数来自所有样本独立同分布假设下的最大似然估计。

## 27 — 3.1神经网络概览（04:27）
- [x] 14.1%

单隐层神经网络：输入层→隐藏层→输出层。计算流程：z^[1]=W^[1]x+b^[1]→a^[1]=σ(z^[1])→z^[2]=W^[2]a^[1]+b^[2]→a^[2]=σ(z^[2])。上标[L]表示第L层。逻辑回归可看作无隐层的神经网络。添加隐层使网络能学习非线性决策边界。

## 28 — 3.2神经网络的表现形式（05:15）
- [x] 14.4%

单隐层NN：输入层n^[0]=nx，隐层n^[1]=4（4个神经元），输出层n^[2]=1。参数W^[1]为(4,nx)，b^[1]为(4,1)；W^[2]为(1,4)，b^[2]为(1,1)。每层计算：z^[l]=W^[l]a^[l-1]+b^[l]，a^[l]=g^[l](z^[l])。输入层a^[0]=x。符号a^[l]既指该层输出（激活值），也是下一层的输入。

## 29 — 3.3计算神经网络的输出（09:58）
- [x] 15.0%

单隐层前向传播详细推导：隐层第i个神经元：z_i^[1]=w_i^[1]T·x+b_i^[1]，a_i^[1]=σ(z_i^[1])。向量化：Z^[1]=W^[1]X+b^[1]→A^[1]=σ(Z^[1])→Z^[2]=W^[2]A^[1]+b^[2]→A^[2]=σ(Z^[2])。W^[1]的第i行对应第i个神经元的所有权重。每个神经元类似一个迷你逻辑回归。

## 30 — 3.4多样本向量化（09:06）
- [x] 15.5%

将m个样本堆叠为矩阵：X(nx,m)包含所有样本特征。前向传播向量化：Z^[1]=W^[1]X+b^[1]→A^[1]=σ(Z^[1])→Z^[2]=W^[2]A^[1]+b^[2]→A^[2]=σ(Z^[2])。此时Z^[1]为(n^[1],m)，A^[1]为(n^[1],m)。列j对应第j个样本。水平存储方案（样本按列排列）与大多数文献一致。

## 31 — 3.5向量化实现的解释（07:38）
- [x] 16.0%

向量化原理：W^[1]X等价于把所有样本x^(j)通过W^[1]做线性变换，结果按列排列。行i存储第i个神经元对所有样本的输出。b^[1]广播到(n^[1],m)。关键：X的每一列是一个样本，W^[1]的每一行对应一个神经元的权值向量。向量化实现消除了遍历样本的for循环。

## 32 — 3.6激活函数（10:57）
- [x] 16.6%

常用激活函数：(1)Sigmoid σ(z)=1/(1+e⁻ᶻ)，输出[0,1]，仅用于二分类输出层(2)Tanh tanh(z)=(eᶻ-e⁻ᶻ)/(eᶻ+e⁻ᶻ)，输出[-1,1]，均值接近0，几乎总优于sigmoid（隐层）(3)ReLU max(0,z)，最常用，计算快，缓解梯度消失(4)Leaky ReLU max(0.01z,z)，解决ReLU的"死神经元"问题。选择经验：隐层默认ReLU或tanh，二分类输出层用sigmoid。

## 33 — 3.7为什么需要非线性激活函数（05:36）
- [x] 16.9%

如果隐层全用线性激活（恒等函数），多层退化为单层线性变换：a^[2]=W^[2](W^[1]x+b^[1])+b^[2]=(W^[2]W^[1])x+(W^[2]b^[1]+b^[2])=W'x+b'。无论多少层都等价于一次线性变换，无法拟合非线性决策边界。异常：输出层（回归）可用线性激活函数。隐层必须用非线性激活才有深度意义。

## 34 — 3.8激活函数的导数（07:58）
- [x] 17.4%

(1)Sigmoid: g'(z)=g(z)(1-g(z))，即a(1-a)(2)Tanh: g'(z)=1-(tanh(z))²，即1-a²(3)ReLU: g'(z)=0(z<0),1(z>0)，z=0处不可导但实现中取0或1均可。在反向传播中需要用到这些导数计算dZ=dA⊙g'(Z)，其中⊙表示逐元素乘法。

## 35 — 3.9神经网络的梯度下降（09:58）
- [x] 18.0%

两隐层神经网络反向传播：(1)计算dZ^[2]=A^[2]-Y(2)dW^[2]=1/m dZ^[2]A^[1]T(3)db^[2]=1/m np.sum(dZ^[2],axis=1)(4)dZ^[1]=W^[2]TdZ^[2]⊙g^[1]'(Z^[1])(5)dW^[1]=1/m dZ^[1]X^T(6)db^[1]=1/m np.sum(dZ^[1],axis=1)。关键：dZ^[1]计算中用到W^[2]T dZ^[2]，链式法则将误差从输出层反向传播到隐层。

## 36 — 3.10反向传播的直觉（选修）（15:49）
- [x] 18.9%

反向传播直觉：∂J/∂z^[l]可理解为第l层第i个神经元对最终损失的责任程度。链式法则逐层传递误差信号。输出层：dZ^[L]=A^[L]-Y（"预测减标签"）。隐层：dZ^[l]=W^[l+1]TdZ^[l+1]⊙g^[l]'(Z^[l])。整体流程：正向计算并缓存Z值→反向逐步计算各层梯度。

## 37 — 3.11随机初始化（07:58）
- [x] 19.4%

权重必须随机初始化！若全部初始化为0，每层所有神经元学习到相同特征（对称性问题）。推荐：W^[l]=np.random.randn(shape)*0.01（小随机数），b^[l]=np.zeros(shape)。0.01因子：sigmoid/tanh在小z值附近梯度最大，大z值区域梯度饱和。深度网络可用Xavier/He初始化（后续课程详讲）。

## 39 — 4.1深L层神经网络（05:52）
- [x] 20.6%

深层网络符号：L=层数，n^[l]=第l层神经元数，a^[l]=g^[l](z^[l])。输入层算第0层(a^[0]=x)，不计入L。深层=隐层多。为什么深层更好？复合函数：每层学习不同抽象层次的特征，浅层学简单模式（边缘），深层学复杂概念（人脸/物体）。电路理论类比：某些函数用深层表示只需O(log n)节点，浅层需要指数级节点。

## 40 — 4.2深层网络中的正向传播（07:16）
- [x] 21.0%

L层网络的通用正向传播：for l=1 to L: Z^[l]=W^[l]A^[l-1]+b^[l]，A^[l]=g^[l](Z^[l])。预测值ŷ=A^[L]。缓存所有Z^[l]以供反向传播使用。每层的W^[l]维数：(n^[l], n^[l-1])；b^[l]：(n^[l], 1)。正确匹配各层维度是调试NN的关键。

## 41 — 4.3正确的矩阵维数（11:10）
- [x] 21.7%

维度公式：W^[l]：(n^[l], n^[l-1])，b^[l]：(n^[l], 1)，dW^[l]同W^[l]，db^[l]同b^[l]。向量化后的维度：Z^[l],A^[l]：(n^[l], m)，X=A^[0]：(n^[0], m)。检查维度是否正确是调试NN最有效的方法之一。反向传播的dZ^[l]、dA^[l]维度与Z^[l]、A^[l]相同。

## 42 — 4.4为什么深度这么有理（10:34）
- [x] 22.3%

深度理由：(1)人脑类比：视觉皮层分层处理（V1边→V2形状→V4物体→IT概念）(2)电路理论：某些函数（如奇偶校验、多层级异或）用深层表示复杂度为O(log n)，浅层需要指数级节点(3)特征复用：深层每一层学习可重用的特征表示组合(4)端到端学习避免了手工设计多层特征的繁琐。深层≠一定要非常深，根据问题选择。

## 43 — 4.5为深层神经网络构建模块（08:34）
- [x] 22.8%

实现深层NN的构建块：单层前向：输入A^[l-1]→Z^[l]=W^[l]A^[l-1]+b^[l]→A^[l]=g(Z^[l])→输出A^[l]并缓存Z^[l]。单层反向：输入dA^[l]→dZ^[l]=dA^[l]⊙g'(Z^[l])→dW^[l]=1/m dZ^[l]A^[l-1]T→db^[l]=1/m sum(dZ^[l])→dA^[l-1]=W^[l]T dZ^[l]→输出dA^[l-1]。L层网络框架：encode→forward→compute_loss→backward→update。

## 44 — 4.6正向和反向传播（10:30）
- [x] 23.4%

完整训练流程：(1)初始化参数W^[l],b^[l](2)前向传播遍历所有层，缓存Z^[l](3)计算ŷ=A^[L]和损失J(4)反向传播计算dW^[l],db^[l](5)梯度下降更新：W^[l]=W^[l]-α dW^[l], b^[l]=b^[l]-α db^[l]。重复2-5至收敛。前向传播和反向传播是完全对称的结构：前向逐层变换输入，反向逐层分解误差梯度。

## 45 — 4.7参数vs超参数（07:17）
- [x] 23.8%

参数（parameter）：模型学习的W,b，通过梯度下降自动优化。超参数（hyperparameter）：控制算法的变量，需手动设定——学习率α、迭代次数、隐层数L、隐层单元数n^[l]、激活函数选择。超参数直接影响最终参数。调参经验：(1)从不试一次就放弃(2)多尝试不同取值(3)直觉随时间积累。深度学习是高度经验性的领域。

## 46 — 4.8这与大脑的关系是什么（03:18）
- [x] 24.0%

神经网络与生物大脑仅有抽象类比：(1)单个神经元比生物神经元简单得多(2)我们尚未理解大脑的学习机制，反向传播在大脑中并未被证实存在(3)深度学习更多是借助数学和工程方法解决ML问题，而非模拟大脑。因此目前将深度学习称为"深度神经网络"更合适，而非"人工大脑"。

## 47 — 1.1训练 开发 测试集（12:05）
- [x] 24.7%

数据集划分：训练集（Training）、交叉验证/开发集（Dev/Validation）、测试集（Test）。传统小数据时代：70/30或60/20/20。大数据时代（百万级）：98/1/1即可。关键原则：(1)Dev和Test必须来自同一分布(2)Dev集用于调参和选模型(3)Test仅最后评估一次(4)没有Test也可以（仅Dev），但增加过拟合Dev的风险。

## 48 — 1.2偏见 方差（08:47）
- [x] 25.2%

偏差（Bias）vs 方差（Variance）：高偏差=欠拟合（训练误差大，Dev误差大）。高方差=过拟合（训练误差小，Dev误差大）。最优=训练和Dev误差都小。高偏差+高方差可能同时出现（部分区域过拟合部分欠拟合）。判断标准：以贝叶斯最优误差（~0%）为基准，训练误差5%→偏差问题，Dev误差6%→方差问题。

## 49 — 1.3机器学习的基本配方（06:22）
- [x] 25.6%

基本诊断与解决流程：高偏差？→ 更大的网络/更长的训练/不同的架构。高方差？→ 更多数据/正则化/不同的架构。先解决偏差（训练误差达标），再解决方差（训练-Dev差距）。关键：不要同时做两个方向，逐个击破。偏差-方差权衡在DL时代弱化，大网络+更多数据往往同时降低偏差和方差。

## 50 — 1.4正则化（09:43）
- [x] 26.2%

L2正则化：J_reg=J+λ/(2m)||W||²_F（Frobenius范数），b通常不加正则化。在梯度下降中，等价于每次更新前将W乘以(1-αλ/m)（权重衰减）。λ是正则化超参数，越大约束越强。L1正则化（λ|W|）使权重稀疏（许多W分量为0），有利于特征选择和压缩模型。L2通常更常用。

## 51 — 1.5为什么正则化可以减少过拟合（07:10）
- [x] 26.6%

正则化抑制过拟合的原理：λ增大→W变小→z=W·a+b变小→tanh(z)近似线性区域→网络更接近线性模型→难以拟合过于复杂的决策边界→方差降低。高λ时网络的信息容量被压缩，只保留最关键的信号。梯度下降时λ越大，W衰减越剧烈。

## 52 — 1.6正规化抛弃（09:26）
- [x] 27.1%

Dropout正则化：每轮训练中随机丢弃（置零）一定比例的神经元（常见keep_prob=0.8隐层，0.5-0.8输入层）。使网络不依赖任何单一特征，强制分布式表示。实现：生成随机掩码D^[l]=np.random.rand(…) < keep_prob → A^[l]=A^[l]*D^[l]/keep_prob。除以keep_prob补偿期望值缩水。测试时不dropout，也不缩水。

## 53 — 1.7理解抛弃（07:05）
- [x] 27.5%

Dropout直觉：(1)强制每个神经元学会多种特征组合而非依赖特定输入(2)类似每次训练一个更小的子网络，最终集成多个子网络的投票效果(3)输入层keep_prob可接近1（信息损失容忍度低），隐层可用0.5-0.8。不同层可有不同keep_prob。过拟合严重的层用更小的keep_prob。缺点：代价函数J不再单调递减，难以调试梯度下降收敛。

## 54 — 1.8其他的正则化方法（08:25）
- [x] 28.0%

其它正则化手段：(1)数据增强（Data Augmentation）：翻转/裁剪/旋转图像生成伪新数据，以极低成本增加数据量(2)早停法（Early Stopping）：Dev误差不再下降时停止训练，防止过拟合。缺点：同时优化J和防止过拟合，违反了正交化原则。L2正则化或数据增强更正交化。

## 55 — 1.9归一化输入（05:31）
- [x] 28.3%

输入归一化两步：(1)零均值化：μ=1/m Σx^(i)，x=x-μ(2)归一化方差：σ²=1/m Σ(x-μ)²，x=x/σ。使所有特征尺度一致。不做归一化时损失函数等高线极不对称，梯度下降之字形缓慢收敛。归一化后损失函数更"圆"，梯度下降更快收敛。训练和测试必须用相同的μ和σ（从训练集计算）。

## 56 — 1.10梯度消失 爆炸（06:08）
- [x] 28.7%

梯度消失/爆炸：深层网络中，若W的元素稍大于1，激活值和梯度指数级增长（爆炸）；若稍小于1，指数级衰减（消失）。L层网络梯度正比于W^L，使深层难以训练。表现：浅层梯度几乎为零，参数几乎不更新；深层梯度极大，更新震荡。解决方案：细心初始化权重（He/Xavier）。

## 57 — 1.11深度网络权值初始化（06:13）
- [x] 29.1%

缓解梯度消失/爆炸的初始化：(1)Xavier初始化：W~N(0,1/n^[l-1])，适合tanh激活(2)He初始化：W~N(0,2/n^[l-1])，适合ReLU激活。原理：控制每层输出方差≈输入方差。实现：W^[l]=np.random.randn(shape)*np.sqrt(2/n^[l-1])（He）。也有除以(n^[l-1]+n^[l])/2的变体。良好的初始化是训练深层网络的关键前提。

## 58 — 1.12梯度的数值近似（06:36）
- [x] 29.4%

数值梯度验证：双侧差分法 f'(θ)≈(f(θ+ε)-f(θ-ε))/(2ε)，误差O(ε²)。单侧差分 f'(θ)≈(f(θ+ε)-f(θ))/ε，误差O(ε)。双侧更精确（误差来自泰勒展开项的抵消）。取ε=10⁻⁷左右。用于梯度检查，验证反向传播的实现是否正确。

## 59 — 1.13梯度检查（06:35）
- [x] 29.8%

梯度检查实现：(1)把所有W^[l], b^[l]展开拼接为一个大向量θ(2)同样展开dW^[l], db^[l]为dθ(3)对每个i计算dθ_approx[i]=(J(θ+ε e_i)-J(θ-ε e_i))/(2ε)(4)比较dθ_approx和dθ：||dθ_approx-dθ||₂/(||dθ_approx||₂+||dθ||₂)。若比值≈10⁻⁷→正确；≈10⁻⁵→检查；≈10⁻³→有bug。逐个参数调试定位错误。

## 60 — 1.14梯度检查实施须知（05:19）
- [x] 30.1%

梯度检查注意事项：(1)仅在调试时使用，训练时关闭（极慢）(2)检查失败→逐个组件比对找bug(3)别忘了正则化项也会贡献梯度(4)Dropout不能同时启用，会导致梯度不一致(5)偶然情况：参数很小时梯度正确，参数变大后逐渐失败。建议初始化后立即检查一次，训练快结束时再检查一次。ε=10⁻⁷是推荐默认值。

## 61 — 2.1小批量梯度下降（11:29）
- [x] 30.8%

三种梯度下降：(1)批量GD：用全部m个样本，准确但慢(2)随机GD：每次1个样本，快但噪声大、震荡、无法利用向量化(3)Mini-batch GD：每次mini_batch_size个样本，折中方案。若mini_batch_size=m（全批量，蓝色），收敛最稳定但每步慢。若=1（随机梯度，紫色），震荡严重。实践中选中间值64/128/256/512（2的幂），适合GPU内存。

## 62 — 2.2理解小批量梯度下降（11:19）
- [x] 31.5%

Mini-batch超参数：batch size。大小选择：(1)小训练集(<2000)：直接用批量GD(2)大训练集：2的幂（64/128/256/512），适配CPU/GPU内存。Mini-batch GD代价函数不是单调下降的（每步用不同数据子集），呈现带噪声的趋势。逐个epoch遍历所有mini-batch称为一个epoch。需调整学习率应对噪声。

## 63 — 2.3指数加权平均（05:59）
- [x] 31.8%

指数加权移动平均（EWMA）：v_t=β·v_{t-1}+(1-β)·θ_t。v_t≈过去1/(1-β)天的平均。β=0.9→10天平均；β=0.98→50天平均（更平滑但滞后）。β越大曲线越平滑但对变化的响应越慢。EWMA是后面Momentum、RMSprop、Adam优化算法的基础组件，只需一行代码和极少内存。

## 64 — 2.4理解指数加权平均（09:42）
- [x] 32.4%

EWMA展开：v_t=(1-β)Σβ^(t-i)θ_i。当β^(1/(1-β))≈1/e时，权重衰减到1/e以下，约1/(1-β)项贡献主要权重。指数加权平均的优势：(1)极低内存开销：只需存储一个v值(2)一行代码实现(3)虽非最精确但实践足够。缺点是初始阶段偏差较大。

## 65 — 2.5指数加权平均数的偏差修正（04:12）
- [x] 32.6%

偏差修正（Bias Correction）：EWMA初期v_0=0导致前几个值严重偏低。修正：v_t/(1-β^t)。t=2,β=0.98时v_2需除以1-0.98²=0.0396（约放大25倍），使估计接近真实值。随着t增大，1-β^t→1，修正自动消失。实践中若不在意初期偏差或预热期可接受，可跳过偏差修正。

## 66 — 2.6动量梯度下降（09:21）
- [x] 33.2%

动量（Momentum）梯度下降：V_dW=β·V_dW+(1-β)·dW，V_db=β·V_db+(1-β)·db，更新W=W-α·V_dW，b=b-α·V_db。通常β=0.9。直觉：纵向（震荡方向）正负交替，EWMA使均值趋近0→减少震荡；横向（正确方向）持续一致，EWMA保持方向→加速收敛。像球滚下山，获得"动量"。

## 67 — 2.7RMSprop（07:42）
- [x] 33.6%

RMSprop：S_dW=β·S_dW+(1-β)·dW²（逐元素平方），S_db=β·S_db+(1-β)·db²。更新：W=W-α·dW/√(S_dW+ε)，b=b-α·db/√(S_db+ε)，ε=10⁻⁸防除零。直觉：震荡大的方向S大→除以大数缩小步长；震荡小的方向S小→除以小数放大步长。自动为每个参数分配独立的自适应学习率。

## 68 — 2.8适应性矩估计（Adam）算法优化（07:08）
- [x] 34.0%

Adam=Momentum+RMSprop。V_dW=β₁·V_dW+(1-β₁)·dW（动量），S_dW=β₂·S_dW+(1-β₂)·dW²（RMSprop）。偏差修正：V_corrected=V_dW/(1-β₁^t)，S_corrected=S_dW/(1-β₂^t)。更新：W=W-α·V_corrected/(√S_corrected+ε)。默认超参：α需要调整，β₁=0.9，β₂=0.999，ε=10⁻⁸，通常默认即好用。实际最常用的优化器。

## 69 — 2.9学习速率衰减（06:45）
- [x] 34.4%

学习率衰减：训练初期用大α快速接近最优，后期用小α精细化收敛。常见策略：(1)α=α₀/(1+decay_rate·epoch_num)(2)指数衰减α=α₀·0.95^epoch(3)阶梯衰减(4)α=α₀/√epoch_num。在小批量GD中，固定α的噪声使参数在最优值附近震荡，衰减α可减小震荡。Adam通常不依赖手动衰减，因自适应机制已经做了类似的效果。

## 70 — 2.10局部最优解问题（05:24）
- [x] 34.7%

高维空间直觉修正：在2D图中看似局部最优的点，在高维空间（比如20000维）中极可能是鞍点（某些方向是极小，某些方向是极大）。真正所有方向都是极小的局部最优极罕见。DL的主要障碍不是局部最优，而是鞍点和平坦区域（Plateaus）导数趋近零→学习极慢。Momentum/RMSprop/Adam能加速逃离鞍点和平原区域。

## 71 — 3.1参数调整过程（07:11）
- [x] 35.2%

调参优先级：学习率α最重要 → 其次是β（动量）、mini_batch_size、隐层单元数 → 再次是层数L、学习率衰减。调参方法：(1)随机网格搜索（不要网格！因为不同参数重要程度不同）(2)粗糙→精细：先大范围随机定位好区域，再在该区域精细化搜索。使用对数尺度而非线性尺度搜索（如α: 0.0001到1之间，用log均匀取样）。

## 72 — 3.2使用适当的标准来选择超参数（08:51）
- [x] 35.7%

对数尺度采样：α用log scale，如logα~U(log0.0001, log1)即10^r, r~U(-4,0)。β（动量系数0.9-0.999）用1-β的对数尺度：1-β~10^r，因β从0.9变0.9005影响远小于0.999变0.9995。批量size和隐层单元数用线性尺度。两种调参策略：Panda（熊猫）方式——一只模型看护到底；Caviar（鱼子酱）方式——并行训练多个模型对比。

## 73 — 3.3实践中的超参数调整 熊猫vs鱼子酱（06:52）
- [x] 36.1%

Panda策略（保姆式）：计算资源有限时，花一整天看守一个模型的训练曲线，根据曲线变化动态调整学习率等参数。Caviar策略（并行式）：计算资源充足时，同时训练多个模型（不同超参数），一天后比较学习曲线选择最优。关键：不要在没试过之前就基于直觉排除超参数取值范围。至少每几个月重新评估一次调参策略。

## 74 — 3.4网络中的正常化激活（08:56）
- [x] 36.6%

Batch Normalization：对每层z^[l]的每个神经元做归一化。μ=1/m Σz_i，σ²=1/m Σ(z_i-μ)²，z_norm=(z-μ)/√(σ²+ε)，最终输出z̃=γ·z_norm+β（γ,β是新的可学习参数）。注意Batch Norm中的β不是Momentum的β！γ和β让网络可以恢复恒等映射（γ=√σ²,β=μ时z̃≈z），赋予表达能力。

## 75 — 3.5将Batch Norm拟合到神经网络中（12:56）
- [x] 37.4%

Batch Norm在NN中的应用：在z^[l]=W^[l]a^[l-1]+b^[l]之后，a^[l]之前插入BN(z^[l])。由于BN中的β会覆盖偏置b^[l]的效果（z-μ抵消了常数偏移），实践中可以去掉b^[l]或将其设0。BN除了归一化还有轻微正则化效果（mini-batch的噪声引入随机性）。前向传播时用当前mini-batch的均值和方差；梯度需通过BN反向传播。

## 76 — 3.6为什么Batch Norm有效（11:40）
- [x] 38.0%

BN有效的原因：(1)减少内部协变量偏移（Internal Covariate Shift）：前层参数更新改变后层输入分布，BN保持各层输入稳定(2)使梯度对参数尺度和初始值不那么敏感(3)允许更大学习率→更快收敛(4)轻微正则化效果：每个mini-batch的μ和σ有噪声，类似于给z添加了少量噪声→防止过拟合。BN让深层网络的训练变得更鲁棒。

## 77 — 3.7测试时的Batch Norm（05:47）
- [x] 38.4%

测试时BN采用训练阶段μ和σ的指数加权平均作为全局估计（而非测试batch的统计量）。在训练过程中持续追踪：μ_global=β·μ_global+(1-β)·μ_minibatch。测试时每个样本独立预测，用μ_global和σ²_global进行归一化。大多数DL框架（TF/PyTorch）自动处理此逻辑。

## 78 — 3.8Softmax回归（11:48）
- [x] 39.1%

Softmax=多分类的逻辑回归。C分类问题输出层有C个神经元。softmax函数：a_i=e^{z_i}/(Σ_{j=1}^C e^{z_j})，满足Σa_i=1，可解释为各分类的概率。与sigmoid的关系：sigmoid是C=2时的特例。输入x→线性变换z=Wx+b→softmax→概率分布ŷ。预测取最大概率对应的类别。

## 79 — 3.9训练一个softmax分类器（10:08）
- [x] 39.7%

Softmax损失函数：交叉熵L(ŷ,y)=-Σy_j·log(ŷ_j)，其中y是one-hot编码（仅正确类别为1）。反向传播：dZ^[L]=ŷ-y（与sigmoid形式一致！这是交叉熵+softmax的优雅性质）。C=2时，softmax退化为sigmoid，交叉熵退化为二元交叉熵。多分类任务的最终输出层激活用softmax。

## 83 — 1.2正交化（10:39）
- [x] 41.6%

正交化（Orthogonalization）：设计独立的"旋钮"控制不同的属性。如电视的独立高度、宽度、亮度、色度旋钮。ML中：(1)训练集表现→调整网络大小/优化器(2)Dev集表现→正则化/更多数据(3)测试集表现→更多Dev数据(4)真实场景表现→改变Dev集/代价函数。避免一个改进措施同时影响多个指标，导致无法判断效果来源。

## 84 — 1.3单数评价指标（07:17）
- [x] 42.0%

单一评价指标（Single Number Evaluation Metric）让你能快速比较不同模型。准确率单一数字即可排序。精确率（Precision）和召回率（Recall）难以同时比较→用F1分数（调和平均）：2/(1/P+1/R)变为单一指标。调和平均比算术平均更重视较低值（类似并联电阻）。有多个指标时，始终尝试组合成单一数字评价。

## 85 — 1.4满足和优化指标（05:59）
- [x] 42.3%

优化指标（Optimizing）+满足指标（Satisficing）：选一个指标作为优化目标（如准确率最大化），其余设定阈值作为满足条件（如推理时间≤100ms，模型大小≤50MB）。这样明确优化目标的同时保证基本约束。满足指标的关键词："at least" "no more than"。任何项目都应只有1个优化指标+N个满足指标。

## 86 — 1.5训练 开发 测试分布（06:36）
- [x] 42.7%

## 89 — 1.8为什么选择人类水平表现（05:47）
- [x] 44.0%

贝叶斯最优误差：理论上界。人类水平表现（Human-Level Performance, HLP）作为贝叶斯误差的代理。在超越HLP之前：(1)人类标记数据确保质量(2)误差分析可由人来判断(3)偏差/方差分析以HLP为基准。训练误差与HLP的差距=可避免偏差(Avoidable Bias)。训练与Dev误差的差距=方差。HLP作为锚点让偏差-方差分析更精准。

## 90 — 1.9可避免的偏见（07:00）
- [x] 44.4%

可避免偏差：训练误差与贝叶斯最优误差（≈HLP）之间的差距，即本可以减少但尚未减少的偏差。举例：训练误差8%，Dev误差10%。若HLP≈1%，则7%是可避免偏差→优先减偏差（更大模型/更长训练）。若HLP≈7.5%，则0.5%是可避免偏差，2.5%是方差问题→优先减方差（更多数据/正则化）。同一组数据因HLP不同而有完全不同的改进策略。

## 91 — 1.10理解人类水平表现（11:13）
- [x] 45.1%

HLP的定义选择：用可获得的最优误差。如典型医生3%错误，经验医生1%，专家团队0.4%，则贝叶斯误差≤0.4%（用专家团队值）。判断准则：训练误差 - HLP_gap = 可避免偏差，Dev误差 - 训练误差 = 方差。当训练误差>HLP时专注减偏差，当Dev误差>训练误差时专注减方差。当模型接近甚至超越HLP时，偏差-方差分析变得困难。

## 92 — 1.11超越人类水平表现（06:22）
- [x] 45.5%

## 93 — 1.12提高您的模型性能（04:37）

## 95 — 2.1进行误差分析（10:33）
- [x] 47.2%

## 101 — 2.7迁移学习（11:18）
- [x] 51.3%

迁移学习：用任务A学到的知识帮助任务B。如ImageNet训练好的CNN→复用前几层→输出层替换为放射科诊断。当任务B数据很少时特别有效。做法：冻结预训练权重（或小学习率微调），仅训练替换的新输出层。何时有效：(1)任务A和B输入相同类型（都是图像）(2)A的数据量远大于B(3)A的低层特征对B有用。这是CV领域最常用的技巧之一。

## 102 — 2.8多任务学习（13:00）
- [x] 52.0%

多任务学习（Multi-Task Learning）：一个网络同时学习多个相关任务。如自动驾驶同时检测行人、车辆、交通标志。损失函数=各任务损失之和：J=Σ L(ŷ_t, y_t)。适用于：(1)共享底层特征的一组任务(2)各任务数据量相近(3)足够大的网络能同时学好所有任务。比各任务独立训练效果好（共享表征→正则化效果）。目标检测（分类+定位）天然是多任务学习。

## 103 — 2.9什么是端到端深度学习（11:48）
- [x] 52.7%

端到端深度学习：用单个神经网络替代多阶段流水线。如语音识别：传统音频→特征提取→音素识别→词→转录，端到端直接音频→文本。优点：(1)让数据说话，不加入人类先验偏见(2)简化系统设计。缺点：(1)需要海量数据(2)排除了人工设计的有效组件(3)可解释性差。并非总是最优，需结合数据量和问题复杂度判断。

## 104 — 2.10是否使用端到端深度学习（10:20）
- [x] 53.3%

端到端vs多阶段决策框架：X→Y是否可直接学习？关键：(1)数据量：端到端需要海量标注数据，中间如果有各阶段的标注数据（如人脸关键点）则多阶段更现实(2)问题结构：如果中间表示有明确的物理意义（如人脸姿态），多阶段可引入领域知识。经验：数据少时用多阶段（手工特征+独立组件），数据多时倾向端到端。自动驾驶分多模块（感知→规划→控制）而非纯端到端。

## 106 — 1.1计算机视觉（05:45）
- [x] 54.6%

CV挑战：输入维度极大（1000×1000×3=3百万），直接全连接→参数爆炸（数十亿）→过拟合+算力不可行。卷积操作利用图像的两个性质：(1)特征局部相关性（像素只与周围有关）(2)平移不变性（同一特征可出现在图像的任何位置）。卷积=稀疏连接+参数共享→大幅减少参数量，使深层CV网络成为可能。

## 107 — 1.2边缘探测示例（11:31）
- [x] 55.3%

卷积操作基础：滤波器(f核)在输入图像上滑动，逐元素相乘求和生成特征图。垂直边缘检测：filter=[[1,0,-1],[1,0,-1],[1,0,-1]]（左边亮→正激活；右边亮→负激活；均匀→零）。水平边缘：将上述转置。卷积结果的大小：n×n图像与f×f滤波器卷积→(n-f+1)×(n-f+1)。AlexNet开创性地用CNN在大规模图像分类上超越传统方法，开启了CV深度学习时代。

## 108 — 1.3更多边缘探测（07:58）
- [x] 55.8%

边缘检测滤波器的可学习性：传统CV用手工设计的滤波器（Sobel:[[1,0,-1],[2,0,-2],[1,0,-1]]，Scharr:[[3,0,-3],[10,0,-10],[3,0,-3]]），而CNN将滤波器值作为参数通过反向传播学习→自动发现对任务最优的滤波器。这使网络不仅能检测边缘，还能检测角落、纹理、颜色对比等任意视觉模式。

## 109 — 1.4填充（09:50）
- [x] 56.4%

Padding问题：每次卷积缩小图像（n→n-f+1）→深层网络图像消失+边缘信息被丢弃。解决方案：填充（Padding），最常见的是Same Padding（输出=输入尺寸），p=(f-1)/2（f通常为奇数）。Valid Padding=不填充。Same Padding使卷积后尺寸不变→可设计更深网络。若stride=1，输出=(n+2p-f)+1。几乎现代所有CNN都使用padding。

## 110 — 1.5卷积步长（09:02）
- [x] 56.9%

步长（Stride）：滤波器滑动的间隔。若步长s=2，跳一步移动，输出进一步缩小。输出尺寸公式：⌊(n+2p-f)/s⌋+1，向下取整。p常用值：Valid(p=0)、Same(p=(f-1)/2，当s=1)。CNN中跨步卷积可替代池化层用于下采样，是某些架构（如ResNet中）的首选降维策略。

## 111 — 1.6三维卷积（10:45）
- [x] 57.5%

RGB图像卷积：输入n×n×n_c（n_c=通道数），滤波器f×f×n_c（滤波器通道数必须=输入通道数），输出为2D特征图。多个滤波器→多个输出通道。如6个3×3×3滤波器→输出n×n×6（6通道特征图）。第i个滤波器与输入三维卷积得到输出第i个通道。滤波器数量=输出通道数。

## 112 — 1.7卷积网络的一层（16:11）
- [x] 58.5%

卷积层完整操作：(1)输入a^[l-1](n_H×n_W×n_c)(2)与f^[l]×f^[l]×n_c的滤波器卷积(3)加偏置b^[l]（每个滤波器一个标量）(4)通过激活函数g→输出a^[l]。Z^[l]维度：n_H^[l]×n_W^[l]×n_c^[l]，其中n_c^[l]=该层滤波器数量。参数统计=所有滤波器参数+偏置。若10个3×3×3滤波器，参数=10×(27+1)=280个。卷积层的参数独立于输入尺寸！

## 113 — 1.8卷积网络的简单示例（08:32）
- [x] 58.9%

典型CNN结构：Conv→Pool→Conv→Pool→FC→FC→Softmax。趋势：n_H,n_W逐渐减小（通过stride或pooling），n_c逐渐增加（更多滤波器学习更多高级特征）。激活体积从3D（宽×高×通道）逐渐变成1D向量，最后送入全连接层做分类。通道数逐渐增加的直觉：从浅层简单特征（边缘）到深层复杂特征（物体部件）需要更大的表示容量。

## 114 — 1.9池化层（10:26）
- [x] 59.6%

池化（Pooling）：降维和固定化。Max Pooling（最常用）：在每个f×f区域取最大值——直觉：保留最强特征信号，抑制弱响应噪声。Average Pooling：取均值，偶尔用于深层。超参数f=2, s=2为最常见设置（尺寸减半）。池化层无学习参数（仅f和s是超参数），梯度反向传播仅通过被选中的最大值位置。

## 116 — 1.11为什么用卷积（09:41）
- [x] 60.9%

卷积的优势（与全连接对比）：(1)参数共享：一个滤波器在整个图像上使用同一组权重→只学1次特征模式，无论它在图像的哪个位置(2)稀疏连接：每个输出单元只连接输入的小块区域(f×f)→大幅减少计算量。如10个3×3×3滤波器仅280参数，对应的全连接需要14M参数。这两个特性使CNN成为具有平移等变性的高效特征提取器。

## 117 — 2.1为什么要进行案例研究？（03:08）
- [x] 61.0%

案例学习重要性：(1)研究成功架构→理解设计原理(2)将优秀设计映射到自己的问题(3)无需从头发明，站在巨人的肩膀上。经典架构LeNet→AlexNet→VGG→ResNet→Inception代表了CV技术演进的主线。好的CV研究者都熟悉这些架构，设计新模型的灵感常来自对经典架构的深入理解。

## 118 — 2.2经典网络（18:20）
- [x] 62.1%

经典CNN演进：(1)LeNet-5（1998,LeCun）：最早真正商用的CNN架构(2)AlexNet（2012）：与LeNet类似但更大更深，ReLU激活，多GPU，Local Response Normalization（现较少用）(3)VGG-16（2015）：极度简单的架构——仅3×3卷积堆叠，但13个卷积层+3个全连接层共138M参数，证明了深度+简单设计的威力。VGG-16的统一设计模式：小多层CONV→POOL→通道翻倍→重复。

## 119 — 2.3残差网络（07:08）
- [x] 62.5%

ResNet（残差网络）：添加跳跃连接（skip connection），使a^[l+1]直接加到数层后的输出z^[l+2]：a^[l+2]=g(z^[l+2]+a^[l])。这相当于网络学习的不是完整的映射H(x)，而是残差F(x)=H(x)-x。跳跃连接使梯度可以直接传递到浅层（恒等映射的梯度为1）→极大缓解梯度消失→允许训练极深网络（152层甚至1000+层）。

## 120 — 2.4为什么使用残差网络（09:13）
- [x] 63.1%

ResNet成功原理：(1)恒等映射路径使信息无损前传→深层至少与浅层一样好（最差情况残差块学到F(x)→0，整体就是恒等映射）(2)梯度通过跳跃连接直接回传→缓解梯度消失(3)ResNet可看作浅层网络+残差调整→训练深层比训练浅层更容易。实践：残差块使得152层ResNet的ImageNet错误率比VGG-19低得多却更快训练。

## 121 — 2.5网络中的网络及1x1卷积（06:41）
- [x] 63.5%

1×1卷积：对每个空间位置，将输入的所有通道通过1×1×n_C→n_C'变换。本质是每个空间位置的独立全连接层（通道变换），不涉及空间信息融合。两用途：(1)降维/升维：大幅减少/增加通道数而不改变空间尺寸（如从256→32通道压缩）(2)引入非线性：在通道上添加非线性变换。Inception网络大量使用1×1卷积作为计算瓶颈（bottleneck）。

## 122 — 2.6初始网络动机（10:15）
- [x] 64.0%

Inception动机：不同尺寸的滤波器捕获不同尺度特征（1×1=局部，3×3=中等，5×5=全局），不做选择而全都要。Inception模块包含多个并行分支：1×1 Conv、3×3 Conv、5×5 Conv、Max Pooling，所有输出在通道维度拼接。问题：计算量巨大。解决方案：用1×1卷积先降维再卷积（bottleneck层），大幅减少计算量。

## 123 — 2.7初始网络（08:47）
- [x] 64.6%

GoogLeNet/Inception v1架构：22层，由Inception模块堆叠+中间softmax辅助分类器（加梯度信号对抗梯度消失，训练时有效，推理时去掉）。核心为Inception模块：1×1 Conv→缩减→3×3/5×5 Conv→并行→拼接。无全连接层（用全局平均池化代替）→参数仅5M（远少于AlexNet的60M和VGG的138M）。Inception v2/v3/v4持续改进（因子分解+批归一化+残差）。

## 124 — 2.8MobileNet（16:19）
- [x] 65.5%

MobileNet：为移动和嵌入式设备设计的轻量级CNN。核心创新：深度可分离卷积（Depthwise Separable Convolution）。(1)Depthwise Conv：每个输入通道独立卷积（f×f×1一个filter）(2)Pointwise Conv：1×1卷积合并所有通道信息。计算量对比：标准卷积f²·n_C·n_C'·n_H·n_W vs 深度可分离f²·n_C·n_H·n_W + n_C·n_C'·n_H·n_W，比例约为1/n_C'+1/f²≈1/8到1/9。大幅减少计算量。

## 125 — 2.9MobileNet架构（08:33）
- [x] 66.0%

MobileNet v1/v2：(1)v1：堆叠30层深度可分离卷积，两种width multiplier（通道缩放因子α）和resolution multiplier（分辨率缩放因子ρ）进一步调节模型大小和速度。α∈{1,0.75,0.5,0.25}控制模型容量(2)v2：引入倒置残差瓶颈块（先升维→DWConv→降维）+线性瓶颈（去ReLU保留信息）+跳跃连接。MobileNet是实用部署的标杆，平衡精度与速度。

## 126 — 2.10EfficientNet（03:40）
- [x] 66.2%

EfficientNet：用神经架构搜索+复合缩放策略。核心思想：同时均匀地扩大深度（更多层）、宽度（更多通道）、分辨率（更大输入），比例由搜索算法确定。EfficientNet-B0到B7通过统一的复合系数φ缩放，在相同参数量下远超ResNet/DenseNet/SENet等模型。结论：平衡地缩放所有维度比单独扩大某一维度（如仅加深）更高效。

## 128 — 2.12迁移学习（08:48）

## 131 — 3.1目标定位（11:55）
- [x] 69.0%

目标定位vs分类：分类→图像属于哪一类；定位→还要给出边界框坐标。输出标签y = [p_c, b_x, b_y, b_h, b_w, c₁, c₂, c₃]：p_c为是否有目标（1/0），(b_x,b_y,b_h,b_w)为边界框中心坐标和高宽，c为类别。损失函数：p_c用逻辑回归损失，边界框用MSE（平方误差），类别用softmax交叉熵（若无目标则仅计算p_c）。

## 132 — 3.2地标检测（05:57）

地标检测（Landmark Detection）：输出图像中关键点的坐标。如人脸检测输出64个点（眼角、鼻尖、嘴角等），姿态检测输出手腕肘肩等坐标。标签=地标坐标序列。训练需标注地标的监督数据，同样用CNN+平方误差损失回归坐标。所有地标位置需一致标注（固定顺序，同一角度的眼睛用同一个输出神经元）。

## 133 — 3.3目标检测（05:50）

滑动窗口检测：在不同位置、不同尺寸的窗口滑动分类器判断是否包含目标。朴素实现极其低效（每个窗口一次forward pass）。卷积实现替代全连接：将FC层转换为等价的卷积层（如4096→1×1×4096 Conv），利用卷积的共享计算特性一次前向传播处理所有窗口→大幅加速。

## 135 — 3.5边界框预测（14:32）
- [x] 71.2%

YOLO核心边界框预测：将图像划分为S×S网格，每个格点预测B个边界框（每个含p_c,b_x,b_y,b_h,b_w和类别）。边界框坐标(b_x,b_y)相对于格点的内部坐标（0-1之间），(b_h,b_w)相对于全图的比例。每个格点输出(B×5+C)个值。训练时：目标中心落在哪个格点，那个格点负责预测（p_c=1，用对应标签训练位置和类别）。

## 136 — 3.6并交比（04:19）
- [x] 71.4%

IoU（Intersection over Union）=预测框与真实框的交集面积/并集面积。衡量检测精确度：(1)IoU≥0.5→一般认为是正确检测(2)IoU≥0.7→较严格标准。IoU独立于检测器架构，可作为评价指标和损失函数引导优化。IoU在0和1之间，越高越好。定位误差相较于分类误差通常用IoU替代，因为它同时考虑了位置和尺寸的精确度。

## 137 — 3.7非极大值抑制（08:03）
- [x] 71.9%

非极大值抑制（NMS）解决同一目标被多次检测的问题：(1)丢弃所有p_c≤阈值的框(2)选p_c最大的框为预测(3)移除所有与当前预测IoU≥0.5的重叠框(4)重复2-3直到所有框被处理。NMS是几乎所有目标检测器的后处理步骤，确保每个目标仅输出一个最优检测框。阈值的设定影响召回率和精确率的平衡。

## 138 — 3.8锚框（09:43）
- [x] 72.5%

Anchor Boxes：处理同一格点中存在多个目标（如人和车在格点内重叠）。预设K种形状的锚框（宽矮、瘦高等），每个格点对每种锚框预测一个边界框。输出(b_x,b_y,b_h,b_w)是相对于锚框形状的偏移量。训练时选择与真实框IoU最大的锚框负责预测该目标。通过多样化锚框覆盖常见宽高比提高对小目标、重叠目标的检测能力。

## 139 — 3.9YOLO（You Only Look Once）算法（07:02）
- [x] 72.9%

YOLO算法整合：(1)将图像分为S×S网格(2)每个格点用K个锚框预测图像中的目标(3)预测输出S×S×K×(5+C)维张量(4)训练：ground truth与锚框最大IoU匹配(5)推理：NMS去除重复检测。YOLO的"you only look once"体现于整张图片只有一次前向传播→输出所有检测结果→速度极快，适用于实时视频检测。后续版本持续改进Anchor设计、多尺度预测、损失函数。

## 140 — 3.10区域推荐网络（选修）（06:28）
- [x] 73.3%

R-CNN系列：(1)R-CNN：用选择性搜索生成~2000个候选区域→CNN提取特征→SVM分类→慢(2)Fast R-CNN：一次CNN前向→RoI Pooling对齐各候选区域→分类+回归→大幅提速(3)Faster R-CNN：用RPN（Region Proposal Network）替代选择性搜索，CNN内嵌候选区域生成，实现端到端训练。YOLO思路相反：直接回归，而非"提议再分类"。两者在精度和速度上有不同的侧重点。

## 141 — 3.11 用u-net进行语义分割（07:22）
- [x] 73.7%

语义分割（Semantic Segmentation）：为每个像素分配类别标签（而非仅给出边界框）。U-Net架构：编码器（下采样路径，类似标准CNN提取特征）+解码器（上采样路径，逐步恢复空间分辨率）+跳跃连接（将编码器的同尺度特征拼接到解码器→保留空间细节信息）。U-Net最早用于医学图像（细胞分割），现广泛应用于各类像素级标注任务。

## 142 — 3.12 转置卷积（07:40）
- [x] 74.1%

转置卷积（Transposed Convolution/Deconvolution）：上采样操作，输入小→输出大。方法：将输入值乘以滤波器的每个元素，"分散"放置到输出上（相邻位置产生重叠则累加）。可学习的上采样方式。用于解码器中从低分辨率特征图恢复高分辨率输出。注意：这是卷积的反向传播形式，并非数学逆运算，名称为"转置"更准确而非"逆卷积"。

## 143 — 3.13 u-net 结构灵感（03:22）
- [x] 74.3%

U-Net设计理念：编码器压缩空间尺寸但增加通道→压缩无效信息，提取语义特征。解码器恢复空间尺寸但减少通道→填充空间细节。跳跃连接直接传递同尺度的编码器特征→高分辨率空间的局部化信息不会因为压缩而丢失。这种U形结构+跳跃连接已成为分割任务的标准范式，变形应用于各种生成模型中。

## 144 — 3.14 u-net 结构（07:42）
- [x] 74.8%

U-Net完整结构细节：输入572×572×1→编码器：多次Conv(3×3)+ReLU+MaxPool(2×2)，通道翻倍→Bottleneck：1024通道→解码器：上采样(2×2转置卷积)+拼接对应编码器特征图+两次Conv(3×3)→输出层：1×1 Conv→388×388的softmax分割图。用镜像padding解决边缘裁剪问题。损失函数常用交叉熵或Dice系数。

## 145 — 4.1.1什么是人脸识别（04:38）
- [x] 75.1%

人脸验证（1:1）vs 人脸识别（1:K）：验证=问"这是你吗？"输出Yes/No；识别=问"在K个人的数据库中，这是谁？"输出身份。验证的错误率低不代表识别错误率低（K越大越难）。Live face detection（活体检测）防止照片/视频欺骗。本节聚焦基于深度特征嵌入的验证/识别方法。

## 146 — 4.1.2单样本学习（04:45）
- [x] 75.3%

单样本学习（One-shot Learning）：只给每个身份一张或很少的训练照片→学习判定新照片是否属于该人。标准分类方法不可行（训练样本极少+类别数持续变化）。解决方案：学习相似度函数d(img1, img2)，返回两张脸属于同一个人的"距离"。d(img1,img2)≤τ→同一人，d>τ→不同人。学习相似度度量是做人脸识别的关键。

## 147 — 4.1.3孪生神经网络（04:52）
- [x] 75.6%

Siamese Network（孪生网络）：两个完全相同（共享参数）的CNN分别编码两张人脸为固定维度的特征向量f(x)。相似度度量：d(x₁,x₂)=||f(x₁)-f(x₂)||²₂。训练目标：同一个人→编码向量距离近，不同人→距离远。参数共享是使度量有意义的必要条件（否则两张同样的脸可能被映射到不同空间）。这种双塔结构在对比学习和检索任务中广泛使用。

## 148 — 4.1.4Triplet Loss三元组损失（15:31）
- [x] 76.5%

Triplet Loss：每次训练使用三个样本——Anchor(A), Positive(P,同人), Negative(N,不同人)。损失函数：L(A,P,N)=max(||f(A)-f(P)||²-||f(A)-f(N)||²+α, 0)。α（margin）保证同人和不同人之间有足够间距：||A-P||²+α<||A-N||²。三元组选择策略很关键：选P/N不是随机的，需选"最难的"P（最不像A的同人）和"最难的"N（最像A的不同人）才能有效训练。

## 149 — 4.1.5人脸验证和二进制分类（06:06）
- [x] 76.9%

将人脸验证转化为二分类：输入两张图片的编码f(x₁)和f(x₂)→计算差异向量|f(x₁)-f(x₂)|→输入Logistic Regression→输出0/1(同一人/不同人)。训练时孪生网络+分类器端到端联合训练。预测时将数据库中所有人物的编码预先计算好→新照片只需1次前向→与所有存储编码快速比较（高效K近邻检索）。

## 150 — 4.2.1什么是神经风格迁移（02:03）
- [x] 77.0%

神经风格迁移（Neural Style Transfer）：将内容图像（C）的内容与风格图像（S）的风格融合，生成一张新图像（G）。例子：照片内容+梵高画风→融合图像。使用预训练CNN（通常是VGG-19）提取特征→优化生成图片G使其同时匹配C的内容特征和S的风格特征→G不断迭代直到满足两者的权衡。

## 151 — 4.2.2什么是深度卷积神经网络学习（07:58）
- [x] 77.5%

CNN特征可视化：浅层（layer 1）检测简单模式→边缘、角、特定颜色；中间层（layer 5）检测复杂的纹理和物体部件→圆形、网格、狗的眼睛；深层（layer 10+）检测语义概念→人脸、动物、汽车。特征从具体/局部逐渐变为抽象/全局。理解这些有助于设计代价函数：浅层特征适合做风格匹配（含纹理信息），深层特征适合做内容匹配（含语义信息）。

## 152 — 4.2.3代价函数（04:00）
- [x] 77.7%

NST代价函数：J(G)=α·J_content(C,G)+β·J_style(S,G)。J_content度量G和C在深层激活上的差异；J_style度量G和S在多层风格矩阵上的差异；α,β控制内容vs风格的权重比例。优化过程：(1)随机初始化G（噪声图）(2)用梯度下降最小化J(G)，每次迭代G更新使J降低→G逐渐融合内容和风格。

## 153 — 4.2.4内容代价函数（03:38）
- [x] 77.9%

内容代价 J_content(C,G)=1/2·||a^[l](C)-a^[l](G)||²，其中a^[l]是预训练VGG网络第l层的激活值。选择中间层（如第5卷积层）效果最好——太浅产生与原图几乎没有区别的重建，太深层可能丢失空间结构。内容代价本质上是在深层语义特征空间中做L2距离最小化。

## 154 — 4.2.5风格代价函数（13:18）
- [x] 78.7%

风格代价基于Gram矩阵：G^[l]_{ij}=Σ_k a^[l]_{ik}·a^[l]_{jk}（第i和第j个特征图在所有空间位置的内积之和）。Gram矩阵捕捉不同特征图之间的相关性（即风格="纹理搭配"）。风格代价 J_style^[l](S,G)=1/(4n_c²n_H²n_W²)·||G^[l](S)-G^[l](G)||²_F。通常使用多层的风格代价加权求和（覆盖不同尺度的纹理模式），使生成图同时拥有粗细粒度风格元素。

## 155 — 4.2.6 1维和3维推广（09:09）
- [x] 79.2%

卷积推广：1D卷积用于序列数据（心电图、文本字符级模型），滤波器沿时间维度滑动。3D卷积用于体积数据（CT扫描、MRI、视频），滤波器在3个空间维度上卷积。与2D卷积的核心相同：局部感受野+参数共享→减少计算量和参数量。序列处理上，1D CNN常用于文本分类或作为RNN的替代（并行计算优势）。

## 156 — 1.1为什么用序列模型（03:01）
- [x] 79.4%

序列模型处理序列数据：语音识别（时域信号→文本）、音乐生成（空→音符序列）、情感分类（文本→情感）、DNA序列、机器翻译（文本→文本）、视频活动识别、命名实体识别。输入输出都可能是序列，长度不一定相同。核心挑战：捕捉序列中的时序依赖关系。

## 157 — 1.2注释（09:16）
- [x] 79.9%

符号约定：x^(i)<t>表示第i个样本的第t个时间步（上标i=样本索引，尖括号t=时间步索引）。T_x^(i)=第i个样本输入序列长度，T_y^(i)=输出序列长度。x^(i)<t>是one-hot向量（字典大小为词汇表+未知词<EOS>+填充<PAD>）。监督学习的训练集={(x^(i), y^(i))}，包含对齐的输入输出序列。

## 158 — 1.3循环神经网络模型（16:32）
- [x] 80.9%

标准RNN：每个时间步t的隐状态a^<t>=g(W_aa a^<t-1>+W_ax x^<t>+b_a)，输出ŷ^<t>=g(W_ya a^<t>+b_y)。同一个参数矩阵W_aa、W_ax、W_ya在所有时间步共享→参数不随序列长度增长。前向传播从t=1到T_x逐步计算。RNN的缺点：(1)每个时刻输出仅依赖它之前的输入(2)梯度消失使长程依赖难以学习。后面讲解GRU、LSTM、双向RNN来解决这些问题。

## 159 — 1.4通过时间的反向传播（06:12）
- [x] 81.3%

BPTT（Backpropagation Through Time）：将RNN按时间展开为深层网络（T_x层共享参数）→反向传播按时间倒序计算损失L=∑L^<t>对各参数的梯度。Δt时：∂L/∂W_aa = ∂L/∂a^<t>·(∂a^<t>/∂W_aa)，其中∂a^<t>/∂W_aa包含a^<t-1>和W_aa·∂a^<t-1>/∂W_aa。梯度通过时间步的连乘，容易导致消失或爆炸。

## 160 — 1.5不同类型的RNNs（09:34）
- [x] 81.8%

RNN架构变体：(1)Many-to-Many（Tx=Ty）：命名实体识别，逐词标注→双向RNN（Bi-RNN）使用(2)Many-to-One（Tx>Ty=1）：情感分类，输入文本序列→最后时刻输出情感→最后隐状态h_Tx馈入分类器(3)One-to-Many（Tx=1,Ty>1）：音乐生成，输入单个音符/空→逐时刻生成音符序列(4)Many-to-Many（Tx≠Ty）：机器翻译，不等长输入输出（encoder-decoder主干架构，下单元详解）。

## 161 — 1.6语言模型和序列生成（12:02）
- [x] 82.5%

语言模型：计算句子出现的概率P(y^(1), y^(2), ..., y^(Ty))。用RNN语言模型：softmax输出层预测下一个词的条件概率P(y^<t>|y^<1>,...,y^<t-1>)。训练数据=大规模语料库，每个句子通过RNN逐词预测，损失=-Σ log P(y^<t>|之前）。最后的softmax层维度=词汇表大小（可能10万+），是RNN中计算量最密集的部分。

## 162 — 1.7对新序列的采样（08:39）
- [x] 83.0%

序列采样（生成）：(1)训练好的RNN语言模型(2)随机选择初始词(3)根据softmax输出概率分布随机采样下一个词(4)将采样词作为下一时刻的输入(5)重复直到采样到<EOS>或达到最大长度。有两种采样策略：(a)基于概率的随机采样（多样性好但可能不连贯）(b)贪心选最大概率（连贯但重复单调）。Characters-level LM用字符级建模可生成新词，但训练更慢。

## 163 — 1.8RNNs的梯度消失（06:29）
- [x] 83.4%

RNN梯度消失：长序列中，时间步t的梯度经过W_aa^(T-t)传播，若W_aa的特征值<1→梯度指数衰减→T-t很大时早期输入几乎不影响输出→模型无法学习远距离依赖。梯度爆炸可通过梯度裁剪（梯度范数>阈值时缩小）解决，但梯度消失更难处理。解决方案：门控RNN（GRU/LSTM），通过门控机制维持长期记忆流。

## 164 — 1.9门控循环单元（GRU）（17:07）
- [x] 84.4%

GRU（Gated Recurrent Unit）：引入c^<t>（记忆单元/memory cell）和两个门控：(1)更新门Γ_u=σ(W_u[c^<t-1>,x^<t>]+b_u)→控制记忆更新程度(2)相关门Γ_r=σ(W_r[c^<t-1>,x^<t>]+b_r)→控制"候选"记忆对过去信息的依赖度。c̃^<t>=tanh(W_c[Γ_r⊙c^<t-1>, x^<t>]+b_c)→候选新记忆；c^<t>=Γ_u⊙c̃^<t>+(1-Γ_u)⊙c^<t-1>→新旧记忆的加权融合。简化版（无相关门）也常用。GRU=两个门+LSTM精简，效果近似但训练更快。

## 165 — 1.10长短期记忆（LSTM）（09:54）
- [x] 85.0%

LSTM有三个门+独立c和a：(1)遗忘门Γ_f=σ(W_f[a^<t-1>,x^<t>]+b_f)→决定丢弃哪些旧记忆(2)更新门Γ_u=σ(W_u[a^<t-1>,x^<t>]+b_u)→决定写入哪些新信息(3)输出门Γ_o=σ(W_o[a^<t-1>,x^<t>]+b_o)→控制记忆到输出的流动。候选记忆c̃^<t>=tanh(W_c[a^<t-1>,x^<t>]+b_c)，c^<t>=Γ_u⊙c̃^<t>+Γ_f⊙c^<t-1>，a^<t>=Γ_o⊙tanh(c^<t>)。LSTM比GRU更灵活但参数更多。实践中二者差异不大，任选其一即可。

## 166 — 1.11双向RNN（08:20）
- [x] 85.5%

双向RNN（Bidirectional RNN）：两个RNN同时运行→一个从左到右，一个从右到左→各时刻的预测结合两个方向的隐状态：ŷ^<t>=softmax(W_y[a_fwd^<t>, a_bwd^<t>]+b_y)。使预测可以"看到过去和未来"，对需要上下文的问题（如NER、翻译、补全）效果远超单向。缺点：需要完整序列→不能用于实时流式预测。双向RNN常与GRU/LSTM结合使用，是NLP任务的标配。

## 167 — 1.12深度RNNs（05:17）
- [x] 85.8%

多层RNN：多层LSTM/GRU堆叠。第l层输入=第l-1层的a^<t>序列（取代原始输入x^<t>）。层数通常3-5层已经较深（因序列本身已在时间维度有长链式计算）。增加层数提高抽象能力但训练更困难→可以使用跳跃连接。堆叠RNN+双向+GRU/LSTM是许多NLP问题的高性能基线方案。

## 168 — 2.1词表示（10:08）
- [x] 86.4%

词表示的两大范式：(1)One-hot向量：维度=词汇表大小，每个词仅1位为1。问题：向量的内积=0→无法建模词之间的关系（orange和apple语义接近但one-hot完全正交）(2)词嵌入（Word Embedding）：每个词映射到低维稠密向量（如300维），语义相似的词向量空间距离近。类比：Man→Woman≈King→Queen。词嵌入是NLP从"离散符号"走向"连续语义"的关键一步。

## 169 — 2.2使用词嵌入（09:23）
- [x] 86.9%

使用词嵌入的好处：(1)迁移学习：在大语料上预训练词向量→迁移到词量小的任务中→类似CV的预训练迁移学习(2)维数大幅降低（100K→300）→模型参数少→抗过拟合(3)语义相近的词的激活模式也相近→泛化能力更强。三步法：从大语料（1B+词）训练词嵌入→迁移到新任务（如NER，10K词标注）→小任务上可选微调。

## 170 — 2.3词嵌入的性能（11:55）
- [x] 87.6%

词嵌入类比推理：向量满足e_man - e_woman ≈ e_king - e_queen → 解argmax_w sim(e_w, e_king-e_man+e_woman)。相似度常用余弦相似度：sim(u,v)=uᵀv/(||u||·||v||)。t-SNE将高维嵌入投影到2D可视化的聚类模式→语义相同（星期/国家/动词）的词自然聚集。嵌入矩阵E是(dim_embedding, vocab_size)，每个词的向量=e_i=E·o_i（稀疏矩阵乘法的实际实现常用查表/embedding lookup）。

## 171 — 2.4矩阵嵌入（05:58）
- [x] 88.0%

嵌入矩阵E是(embed_size, vocab_size)的矩阵，每列是词e_i。实际上通过embedding lookup获得词的向量而非真正做矩阵乘法×one-hot（太浪费）。PyTorch中nn.Embedding(vocab_size, embed_dim)，输入词索引，输出嵌入向量。如果训练中微调嵌入层的话，嵌入矩阵→可训练的参数。学习词嵌入的主要方法：语言模型目标（上下文预测）、Word2Vec（skip-gram/CBOW）、GloVe。

## 172 — 2.5学习词嵌入（10:09）
- [x] 88.5%

语言模型学嵌入：输入前几个词→预测下一个词→词的嵌入向量作为byproduct被优化。CBOW（Continuous Bag of Words）：用上下文词（前后各k个）的嵌入平均预测中心词。Skip-gram：用中心词的嵌入预测上下文词（更常用的word2vec变体）。Skip-gram对低频词效果更好，学习到的向量空间"加法"性质最显著。

## 173 — 2.6词转换成向量形式（12:48）
- [x] 89.3%

简化的Skip-gram模型：(1)取中心词c和上下文词t(2)输入c的嵌入e_c (3)用softmax在所有词汇上预测t的概率：P(t|c)=exp(θ_tᵀe_c)/Σ exp(θ_jᵀe_c)。问题：分母对所有词汇求和→当词典有10万+词时计算代价极高。解决方案：分层softmax（Hierarchical Softmax，用二叉树加速到O(log V)）和负采样（Negative Sampling，下节详解）。

## 174 — 2.7负采样（11:54）
- [x] 90.0%

负采样：将极度多分类问题转化为二分类：输入(中心词c,候选词t)→预测t是否在c的上下文中→正例是实际的上下文窗口内的词，负例随机采样k个词（k≈5-20）。优化：P(y=1|c,t)=σ(θ_tᵀe_c)。损失函数=正例的logσ(θ_tᵀe_c)+Σ负例logσ(-θ_tᵀe_c)。每个样本仅更新正例词+几个负例词的参数→计算量与词汇量无关！实际实现时负例按频率^0.75的分布采样，减少超高频词的权重。

## 175 — 2.8GloVe 词向量（11:09）
- [x] 90.6%

GloVe（Global Vectors for Word Representation）：利用全局共现统计矩阵X_ij（词j出现在词i上下文中的次数）。目标函数：最小化Σ f(X_ij)·(θ_iᵀe_j+b_i+b̃_j-log X_ij)²，f为权重函数截断极端高频词。GloVe=矩阵分解方法，利用所有共现统计→训练速度快且词向量质量好。Word2Vec和GloVe效果相当，可根据具体需求和数据选择。

## 176 — 2.9情感分类（07:38）
- [x] 91.1%

情感分类问题：缺乏大量有情感标签的文本，但有大量无标签文本。解决方案：在大语料上预训练词嵌入→对评论使用嵌入→取所有词嵌入的平均或求和→送入softmax分类器。简单平均模型忽略词序（"good"和"not good"嵌入平均相同→可能误判），更复杂的用RNN捕捉语序结构→"not good"的RNN隐状态自然包含否定信息→多分类情感准确率大幅提升。

## 177 — 2.10词嵌入除偏（11:09）
- [x] 91.7%

词嵌入的性别/种族偏差：如"man:computer_programmer as woman:homemaker"→嵌入从训练语料中学会了社会偏见。除偏方法：(1)识别偏差方向：计算she/he等定义性别对的差异向量平均→主成分方向(2)中和非性别词：将babysitter/doctor等不应有性别关联的词投影到正交方向，消除性别成分(3)均衡化性别词：boy/girl这类应有性别但不应有某种"强势/弱势"蕴含的词对→调整到关于正交平面等距。嵌入偏差从数据中来，数据本身就是有偏的。

## 178 — 3.1基础模型（06:19）
- [x] 92.1%

Seq2Seq模型：编码器RNN读取输入序列，输出编码向量→解码器RNN以此向量为条件逐步生成输出序列。机器翻译：Je suis étudiant(encoder)→向量表示→I am a student(decoder)。条件语言模型：解码器预测P(y^<t>|y^<1>,...,y^<t-1>, encoder_hidden)，概率=前序词+编码器总结的信息。生成时采样或搜索最优序列。

## 179 — 3.2选择最有可能的句子（08:57）
- [x] 92.6%

机器翻译=条件语言模型→求argmax_y P(y|x)。贪心搜索(Greedy Search)：逐个最大值选择→不能保证全局最优（"Jane is visiting"比"Jane is going"局部分高但整体可能不好）。穷举搜索：词汇表V，序列长T→V^T种组合，完全不可行。实际用集束搜索（Beam Search），在搜索质量和计算效率间取得折中。

## 180 — 3.3集束搜索（11:55）
- [x] 93.3%

集束搜索（Beam Search）算法：每一步保留B（beam width=束宽）个概率最高的候选序列路径。步骤：(1)第一步计算所有V个词的P(y^(1)|x)，保留top B(2)对每个B路径，计算所有V个下一个词的概率P(y^(2)|x, history) (3)从B×V中选top B路径(4)重复到生成<EOS>或达到最大长度。B=1→贪心搜索；B越大结果越好但计算越多。常用B=3~10。

## 181 — 3.4细化集束搜索（11:01）
- [x] 94.0%

长度归一化：对长句，连乘大量P(y^(t)|...)使概率极小，直接优化容易过早结束（偏爱短句）。目标函数改为 argmax_y (1/T_y^α)·Σ log P(y^(t)|...)，α=0.7~1。α<1：不完全长度归一化，对短句仍略有偏好但保持了长句合理性。B的选择：产品可用B=10左右，研究为了刷分可用B=1000/3000。但B过大反而降低准确率（搜索噪声大于收益）。

## 182 — 3.5集束搜索中的错误分析（09:44）
- [x] 94.5%

集束搜索误差分析：判断是RNN还是集束搜索导致错误。比较→对于y*（人类最优翻译）和ŷ（模型输出），计算P(y*|x)和P(ŷ|x)：(1)P(y*)>P(ŷ)→RNN错了（应选y*却没给高分）→改进RNN(2)P(ŷ)>P(y*)→集束搜索错了（选了ŷ没选y*，但y*概率更高）→扩大B或改进搜索。先排查问题源头，再针对性修复。

## 183 — 3.6Bleu分数（选修）（16:27）
- [x] 95.5%

BLEU（Bilingual Evaluation Understudy）分数：机器翻译的自动评价指标。计算n-gram精确率（翻译中n-gram命中参考翻译的比例），取geometric平均。BP（Brevity Penalty）惩罚过短的翻译：BP=min(1, exp(1-reference_len/output_len))；最终BLEU=BP·exp(Σ w_n·log p_n)。BLEU在0-1之间，但绝对高分罕见（人类级别约0.3-0.4）。广泛应用于MT评测，是NLP领域最知名的自动评价指标之一。

## 184 — 3.7注意力模型直觉（09:42）
- [x] 96.0%

注意力机制核心思想：翻译长句时，不再将整个源句压缩为一个固定长度的编码向量→而是让解码器在生成每个目标词时"关注"源句的不同部分。直觉类似人类翻译：不是背下整个句子再译，而是边看源句不同位置边写出目标词。Bi-RNN编码器产生所有隐状态→解码器通过注意力权重加权求和这些隐状态，动态获得当前生成所需的源句上下文。

## 185 — 3.8注意力模型（12:23）
- [x] 96.8%

注意力机制计算：(1)编码器：Bi-RNN产生每个源时刻的隐藏状态a^<t'>（包含前后文信息）(2)解码器状态s^<t>与每个a^<t'>计算"注意力能量"e^<t,t'>=a(s^<t-1>, a^<t'>)（小前馈网络或点积）(3)softmax得到注意力权重α^<t,t'>(4)上下文向量C^<t>=Σα^<t,t'>·a^<t'>→加权源句隐状态(5)解码器输入=C^<t>+上一步输出(6)输出ŷ^<t>。注意力权重可视化显示期望的对齐模式（词序对应+语序调整）。

## 186 — 3.9语音识别（08:54）
- [x] 97.3%

语音识别的Seq2Seq处理：输入=音频频谱（预处理：分帧+梅尔频谱/声学特征），输出=对应文字。传统方法用音素→单词过中间层；端到端方法为音频→文字（纯seq2seq+注意力或CTC损失）。CTC（Connectionist Temporal Classification）使RNN能直接对齐不等长的音频和文字序列→ttt_hhheeee_→合并重复+删除空白→"the"。现代语音识别普遍过渡到基于Transformer/Conformer的端到端系统。

## 187 — 3.10触发词检测（05:03）
- [x] 97.6%

触发词/唤醒词检测（如"OK Google""Hey Siri"）：输入音频流，当听到触发词时输出1，否则0。不平衡问题：绝大多数时间在输出0。解决方案：每次输出1时，强制后续若干时刻也标1→增大正样本的持续时间。用RNN逐时间步做二分类。模型需要小到能在设备端实时运行。边缘部署要求低延迟+低功耗，限制模型容量。

## 188 — 3.11Transformer网络直觉（05:30）
- [x] 97.9%

Transformer彻底抛弃RNN的串行结构，全注意力+并行化。主要组件：(1)自注意力层（Self-Attention）→捕捉词之间所有位置对的关系(2)前馈层→逐位置的独立变换(3)多头注意力→多个"注意头"捕捉不同子空间的关系(4)位置编码→为全并行结构注入时序信息。由于RNN的时间展开转为全并行，训练速度极大提升，成为目前NLP/CV/Audio几乎所有领域的底层架构。

## 189 — 3.12自注意力机制（11:44）
- [x] 98.6%

Self-Attention公式：Attention(Q,K,V)=softmax(QKᵀ/√d_k)·V。Q(Query)、K(Key)、V(Value)均从同一输入通过不同的线性变换得到：Q=XW_Q，K=XW_K，V=XW_V。QKᵀ计算每对词之间的"相关性"分数，√d_k缩放防止softmax梯度饱和，加权V得到每个词融合所有词信息后的新表示。每个输出位置直接"看到"输入序列中的所有位置→完全替代RNN的时间步计算。

## 190 — 3.13多头注意力机制（08:21）
- [x] 99.1%

多头注意力（Multi-Head Attention）：运行多组独立的自注意力（h头，不同W_Q,W_K,W_V）→在低维子空间捕捉不同语义关系→拼接所有头的输出→再线性变换：MultiHead(X)=Concat(head_1,...,head_h)·W_O。直觉类比CNN的多滤波器：每个头学习不同类型的依赖（语法/语义/指代等）。Multi-Head并行计算，每一头维度=d_k=d_model/h。

## 191 — 3.14Transformer网络（13:11）
- [x] 99.8%

Transformer完整架构：(1)Encoder：N层，每层=Multi-Head Self-Attention→Add&Norm(残差+层归一化)→Feed Forward(逐位置全连接)→Add&Norm(2)Decoder：N层，每层=Masked Multi-Head Self-Attention（不能看见未来词）→Add&Norm→Cross-Attention（以解码器输出为Q，编码器输出为K/V→整合源语言信息）→Feed Forward→Add&Norm(3)最后Linear→Softmax输出词概率。Transformer是当今NLP（BERT/GPT/T5）、CV（ViT）、多模态等几乎所有前沿模型的基本架构。
