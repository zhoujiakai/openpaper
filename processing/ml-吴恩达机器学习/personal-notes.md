## 16 — 4.2 实现梯度下降（10:00）
- [x] 11.3%，20260519周二

吴恩达抛出了梯度下降的计算公式，并且讲了梯度下降的计算方法。就是学习率乘以导数，要注意同时更新，不能先更新 w 再更新 b。一共就这4行代码，要注意代码顺序。要先计算出新的 w，b，再同时更新，不能直接$w=w-\alpha ···$。
$$
tmp\_w = w - \alpha \frac{\partial J(w,b)}{\partial w} \\
tmp\_b = b - \alpha \frac{\partial J(w,b)}{\partial b} \\
w = tmp\_w \\
b = tmp\_b
$$

这里面的导数部分，吴恩达在下一小节会具体讲。

## 18 — 4.4 学习率（09:04）
- [x] 12.7%，20260519周二

吴恩达讲了4种情况：

- 学习率太大：跳过局部最小值，梯度下降可能无法收敛，甚至发散。
- 学习率太小：学习速度太慢，并且可能无法跳出局部最小值。
- 局部最小值的位置导数为零：这导致即使调大学习率，w = w + dJ = w + 0，仍然无法跳出局部最小值。
- 当越靠近局部最小值时，导数接近0，w = w + $\alpha$ dJ，即使$\alpha$固定，更新幅度也会自动变小。

## 19 — 4.5 线性回归中的梯度下降（06:37）
- [x] 13.4%，20260519周二

吴恩达推导了导数的计算公式，只需要一页PPT非常简单，不看也没关系不影响使用，公式如下（w 和 b 记得同时更新）：
$$
w = w - \alpha \frac{1}{m} \sum_{i=1}^{m}               
  (f_{w,b}(x^{(i)}) - y^{(i)}) x^{(i)} \\

$$
吴恩达还提出了一点，平方误差这种成本函数是一个凸函数，是一个弓形图，这种成本函数不存在多个局部最小值，只会有唯一最小值。

## 26 — 6.2 特征缩放 part 2（07:35）
- [x] 18.3%，20260519周二

吴恩达介绍了3种特征缩放的方法：

- 最简单的一种：如果 $300\le x_i \le 2000，就直接x_{i,scaled}=\frac{x_i}{2000}$，就是说直接除以该特征的最大值，这样能够保证所有值都在 [0, 1] 之间。
- 均值归一化：如果 $300\le x_i \le 2000，先计算均值 \mu_i=average(xi)，之后  x_{i,scaled}=\frac{x_i-\mu_i}{2000-300}$，这样能保证所有值都在 [-1, 1] 之间。
- Z-score归一化：需要计算每个特征的标准差$\sigma_i和均值\mu_i，之后x_{i,scaled}=\frac{x_i-\mu_i}{\sigma_i}$。

我们的目标是让特征的尺度尽量在 [-3, 3] 之间，如果尺度在+-0.001或者 +-100之类的，就需要进行缩放，加快梯度下降收敛。

至于什么情况选择哪种缩放方法，吴恩达没讲。下小节讲怎么判断梯度下降是否收敛，下下小节讲如何为梯度下降选择一个合适的学习率。

## 28 — 6.4 学习率的选择（06:07）==lab==
- [x] 19.7%，20260519周二

上小节吴恩达讲了正常的学习曲线。本小节吴恩达先讲了不正常的几种学习曲线：

- 毛茸茸地下降，或者一路上升，就可能是学习率太大了，一路上升也有可能是梯度下降的更新符号写成正号了。
- 如果学习曲线下降速率太慢，就可能是学习率设置太小了。

接着吴恩达分享了自己的经验，就是 $\alpha$ 从 0.001 开始，3倍3倍地往上乘，比如 `0.001, 0.003, 0.01, 0.03, 0.1, 0.3, 1, ···`，这样不断试，同时观察每一次的学习曲线，找到最正常的那个曲线就找到比较合适的 $\alpha$ 了。

在找最合适的 $\alpha$ 的时候，只需要进行几次梯度下降迭代就可以了，不需要迭代很多次。

## 34 — 8.1 逻辑回归的代价函数（12:00）==lab==
- [x] 23.9%，20260520周三，看视频21分钟，笔记30分钟

吴恩达解释平方误差成本函数不适合逻辑回归成本函数的原因：

- 线性回归的模型的公式，使用平方误差的成本函数是凸函数。这里展示一个图，J-w,b的凸函数图。
- 逻辑回归的模型的公式，使用平方误差的成本函数不是凸函数，是毛茸茸的存在很多局部最小值的成本函数。这里展示一个图，J-w,b的非凸函数图。

吴恩达讲了损失函数的概念，成本函数 $J(\mathbf{w},b)$ 的求和符号内部的式子就叫损失函数 $L(f_{\mathbf{w},b}(\mathbf{x}^{(i)}), y^{(i)})$，损失函数衡量单个训练样本的表现，成本函数衡量整个训练集的表现。

逻辑回归的损失函数是：
$$
L(f_{\mathbf{w},b}(\mathbf{x}^{(i)}), y^{(i)}) =
\begin{cases}
-\log(f_{\mathbf{w},b}(\mathbf{x}^{(i)})) & \text{if } y^{(i)} = 1 \\
-\log(1 - f_{\mathbf{w},b}(\mathbf{x}^{(i)})) & \text{if } y^{(i)} = 0
\end{cases}
$$
成本函数是：
$$
J(\mathbf{w},b) = \frac{1}{m}\sum_{i=1}^{m}L(f_{\mathbf{w},b}(\mathbf{x}^{(i)}), y^{(i)})
$$
这样整体的成本函数就是凸函数，就可以用梯度下降找到全局最优值。证明逻辑回归的成本函数是凸函数是纯数学的部分，超出这门课的范围。

## 39 — 10.3 正则化代价函数（09:04）
- [x] 27.5%，20260520周三

缩小指定特征的参数，可以在目标函数中加上正则化项：

$$
\min_{\mathbf{w},b} \frac{1}{2m}\sum_{i=1}^{m}(f_{\mathbf{w},b}(\mathbf{x}^{(i)})-y^{(i)})^2 + 1000w_3^2 + 1000w_4^2
$$

如果不知道应该缩小哪些特征的参数，就直接对所有特征进行惩罚，修改后的成本函数公式：
$$
J(\mathbf{w},b) = \frac{1}{2m}\sum_{i=1}^{m}(f_{\mathbf{w},b}(\mathbf{x}^{(i)})-y^{(i)})^2 + \frac{\lambda}{2m}\sum_{j=1}^{n}w_j^2
$$

成本函数中加上的部分叫做正则化项，$\lambda$ 叫做正则化参数，$\frac{1}{2m}$ 可以使得数据集变大了之后，$\lambda$ 有可能继续正常工作，不用跟着数据集大小调整。b 参数一般不需要进行正则化。

新的目标函数：

$$
\min_{\mathbf{w},b} J(\mathbf{w},b) = \min_{\mathbf{w},b} \frac{1}{2m}\sum_{i=1}^{m}(f_{\mathbf{w},b}(\mathbf{x}^{(i)})-y^{(i)})^2 + \frac{\lambda}{2m}\sum_{j=1}^{n}w_j^2
$$

- 最小化第一项，鼓励算法通过最小平方误差来很好地拟合训练数据；
- 最小化第二项，算法试图使参数 $w_j$ 保持较小。

$\lambda$ 的值指定了两者的相对重要性：

- 如果$\lambda$ 接近0，模型可能会过拟合，多项式曲线为了拟合数据表现得极为扭曲；
- 如果$\lambda$ 设置为$10^{10}$,模型就会欠拟合，因为$w_j$ 变得非常小，模型变成由 b 决定的水平线。

下两个小节吴恩达分别实现把正则化应用到线性回归和逻辑回归，避免过拟合。

## 53 — 4.2 前向传播的一般实现（07:53）
- [x] 37.3%，20260522周五

吴恩达实现了一个`dense()`函数和一个`sequential()`函数，从而实现了前向传播。是很简单的代码，如下：

```python
W = np.array([
  [1, -3, 5],
  [2, 4, -6]])
b = np.array([-1, 1, 2])
a_in = np.array([-2, 4])

def dense(a_in, W, b, g):
  units = W.shape[1]
  a_out = np.zeros(units)
  for j in range(units):
    w = W[:, j]
    z = np.dot(w, a_in) + b[j]
    a_out[j] = g(z)
  return a_out

def sequential(x):
  a1 = dense(x, W1, b1, sigmoid)
  a2 = dense(a1, W2, b2, sigmoid)
  a3 = dense(a2, W3, b3, sigmoid)
  f_x = dense(a3, W4, b4, sigmoid)
  return f_x
```

## 65 — 9.2 Softmax（11:33）
- [x] 45.8%，20260523周六

吴恩达在幻灯片左右对比逻辑回归和 Softmax 回归的算法公式，两个公式比较相似，并且都是所有类别的概率加起来等于1。Softmax 回归的算法公式如下（N 个类别，每个类别 j 有自己的参数 $\mathbf{w}_j, b_j$）：
$$
z_j = \mathbf{w}_j \cdot \mathbf{x} + b_j \quad (j = 1, \dots, N) \\\\
a_j = \frac{e^{z_j}}{\sum_{k=1}^{N} e^{z_k}} = P(y = j \mid \mathbf{x})
$$
吴恩达接着对比逻辑回归和 Softmax 回归的成本函数，两个公式也都是计算交叉熵，Softmax 回归的成本函数（交叉熵损失）如下：
$$
L(\mathbf{a}, y) =
\begin{cases}
-\log(a_1) & \text{if } y = 1 \\
-\log(a_2) & \text{if } y = 2 \\
\quad\vdots \\
-\log(a_N) & \text{if } y = N
\end{cases} \\\\
J(\mathbf{w},b) = \frac{1}{m}\sum_{i=1}^{m}L(\mathbf{a}^{(i)}, y^{(i)})
$$

## 67 — 9.4 softmax的改进实现（09:13）
- [x] 47.2%，20260523周六

吴恩达拿出一个非常直观的代码计算的例子：

```python
x1 = 2.0 / 10000  # x1 = 0.000200000000000000
x2 = 1 + (1/10000) - (1 - 1/10000)  # x2 = 0.000199999999999978
```

问题在于存在数字舍入误差，吴恩达拿出在tensorflow中进行更准确的计算的方法。

吴恩达先用逻辑回归的例子实现这种方法，然后用Softmax回归的例子实现这种方法。

逻辑回归损失函数的计算公式：$L=-y\log(a)-(1-y)\log(1-a)$，其中 $a=\frac{1}{1+e^{-z}}$，先算 a（含指数和除法），再取 log，两次运算放大了舍入误差，导致精度丢失。

- 逻辑回归的改进：输出层不用 sigmoid，改用 `linear` 激活直接输出 logits；损失函数加 `from_logits=True`。预测时需要手动补一步 sigmoid。

    ```python
    model = Sequential([
        Dense(units=25, activation='relu'),
        Dense(units=15, activation='relu'),
        Dense(units=1, activation='linear')   # 不放 sigmoid
    ])
    model.compile(loss=BinaryCrossentropy(from_logits=True))
    model.fit(X, Y, epochs=500)
    logits = model(X_new)
    f_x = tf.nn.sigmoid(logits)   # 预测时手动 sigmoid
    ```

- Softmax 的改进：同理，输出层用 `linear`，损失函数加 `from_logits=True`，预测时手动补 softmax。两版代码的区别就是 `Dense` 最后一层 activation 写不写、loss 加不加 `from_logits=True`。

    ```python
    model = Sequential([
        Dense(units=25, activation='relu'),
        Dense(units=15, activation='relu'),
        Dense(units=10, activation='linear')  # 不放 softmax
    ])
    model.compile(loss=SparseCategoricalCrossentropy(from_logits=True))
    model.fit(X, Y, epochs=100)
    logits = model(X_new)
    f_x = tf.nn.softmax(logits)  # 预测时手动 softmax
    ```

## 77 — 12.4 学习曲线（12:14）
- [x] 54.2%，20260523周六

吴恩达定义学习曲线：学习算法在不用数据量下的表现。吴恩达拿出房价预测的例子，绘制了3个图：

- 正常的学习曲线：指定学习算法为二阶多项式，从1个训练数据到6个训练数据，偏差和方差的变化曲线。偏差不断变大后变平，方差不断变小后变平。
- 高偏差的学习曲线：
    - 表现：指定学习算法为一阶多项式，学习算法的特征提取能力弱，学习曲线看起来和上面一样，实际上画出基准性能的水平线，发现学习曲线变平之后，再增加多少数据，曲线都处在基准性能的损失值以上。
    - 建议：先尝试其他方法降低模型的偏差，而不是投入成本获取更多训练数据。
- 高方差的学习曲线：
    - 表现：指定学习算法为三阶多项式和很低的正则化参数，随着训练数据增加，模型高方差的特征就是两条曲线变平之后，偏差和方差离得很开，而基准性能的损失在两条曲线中间，而偏差可能比较低甚至接近0。
    - 建议：增加训练数据，模型有机会变成正常的学习曲线，也就是偏差靠近基准性能，方差靠近偏差。

吴恩达给出了一个经验：如果有1000个训练数据，可以用100，200，…，1000个训练数据分别进行训练，绘制学习曲线，判断模型是高偏差还是高方差。缺点是成本高，一般不会这么干，吴恩达说保留这个直觉对理解学习算法表现有帮助。
