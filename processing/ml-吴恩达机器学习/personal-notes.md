## 16 — 4.2 实现梯度下降（10:00）
- [x] 11.3%，20260519周二

吴恩达抛出了梯度下降的计算公式，并且讲了梯度下降的计算方法。就是学习率乘以导数，要注意同时更新，不能先更新 w 再更新 b。一共就这4行代码，要注意代码顺序。要先计算出新的 w，b，再同时更新，不能直接$w=w-\alpha ···$。
$$
tmp\_w = w - \alpha \frac{\partial J(w,b)}{\partial w} \\
tmp\_b = b - \alpha \frac{\partial J(w,b)}{\partial b} \\
w = tmp\_w \\
b = tmp\_b
$$

这里面的导数部分，吴恩达在下一小节会具体讲。

## 18 — 4.4 学习率（09:04）
- [x] 12.7%，20260519周二

吴恩达讲了4种情况：

- 学习率太大：跳过局部最小值，梯度下降可能无法收敛，甚至发散。
- 学习率太小：学习速度太慢，并且可能无法跳出局部最小值。
- 局部最小值的位置导数为零：这导致即使调大学习率，w = w + dJ = w + 0，仍然无法跳出局部最小值。
- 当越靠近局部最小值时，导数接近0，w = w + $\alpha$ dJ，即使$\alpha$固定，更新幅度也会自动变小。

下一个小节吴恩达会将代价函数和梯度下降结合，讲解第一个学习算法，也就是线性回归算法。

## 19 — 4.5 线性回归中的梯度下降（06:37）
- [x] 13.4%，20260519周二

吴恩达推导了导数的计算公式，只需要一页PPT非常简单，不看也没关系不影响使用，公式如下（w 和 b 记得同时更新）：
$$
w = w - \alpha \frac{1}{m} \sum_{i=1}^{m}               
  (f_{w,b}(x^{(i)}) - y^{(i)}) x^{(i)} \\     
                                                                                
b = b - \alpha \frac{1}{m} \sum_{i=1}^{m}               
  (f_{w,b}(x^{(i)}) - y^{(i)})
$$
吴恩达还提出了一点，平方误差这种成本函数是一个凸函数，是一个弓形图，这种成本函数不存在多个局部最小值，只会有唯一最小值。

## 26 — 6.2 特征缩放 part 2（07:35）
- [x] 18.3%，20260519周二

吴恩达介绍了3种特征缩放的方法：

- 最简单的一种：如果 $300\le x_i \le 2000，就直接x_{i,scaled}=\frac{x_i}{2000}$，就是说直接除以该特征的最大值，这样能够保证所有值都在 [0, 1] 之间。
- 均值归一化：如果 $300\le x_i \le 2000，先计算均值 \mu_i=average(xi)，之后  x_{i,scaled}=\frac{x_i-\mu_i}{2000-300}$，这样能保证所有值都在 [-1, 1] 之间。
- Z-score归一化：需要计算每个特征的标准差$\sigma_i和均值\mu_i，之后x_{i,scaled}=\frac{x_i-\mu_i}{\sigma_i}$。

我们的目标是让特征的尺度尽量在 [-3, 3] 之间，如果尺度在+-0.001或者 +-100之类的，就需要进行缩放，加快梯度下降收敛。

至于什么情况选择哪种缩放方法，吴恩达没讲。下小节讲怎么判断梯度下降是否收敛，下下小节讲如何为梯度下降选择一个合适的学习率。

## 28 — 6.4 学习率的选择（06:07）
- [x] 19.7%，20260519周二

上小节吴恩达讲了正常的学习曲线。本小节吴恩达先讲了不正常的几种学习曲线：

- 毛茸茸地下降，或者一路上升，就可能是学习率太大了，一路上升也有可能是梯度下降的更新符号写成正号了。
- 如果学习曲线下降速率太慢，就可能是学习率设置太小了。

接着吴恩达分享了自己的经验，就是 $\alpha$ 从 0.001 开始，3倍3倍地往上乘，比如 `0.001, 0.003, 0.01, 0.03, 0.1, 0.3, 1, ···`，这样不断试，同时观察每一次的学习曲线，找到最正常的那个曲线就找到比较合适的 $\alpha$ 了。

在找最合适的 $\alpha$ 的时候，只需要进行几次梯度下降迭代就可以了，不需要迭代很多次。

本小节结束有一个代码实验，对特征进行了缩放，以及比较不同学习率的效果。
