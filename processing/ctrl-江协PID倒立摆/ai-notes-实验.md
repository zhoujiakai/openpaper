### 关键硬件原理

**编码器测速原理**：AB 相正交编码器输出两路方波，相位差 90°。STM32 定时器配置为编码器模式，每个脉冲边沿计数一次。转速 = 单位时间计数值 ÷ 编码器线数 ÷ 减速比。

**角度传感器原理**：MPU6050 内置三轴陀螺仪 + 三轴加速度计。原始数据需融合（互补滤波或 DMP）得到欧拉角。更新周期是 PID 调控周期 T 的下限。

**导电滑环的必要性**：没有滑环，摆杆转几圈就会把传感器线缆拧断。滑环将电气连接从旋转部件传到固定部件。

## 代码结构

### 工程文件组织

```
Project/
├── main.c               # 主函数，系统初始化 + 主循环
├── pid.c / pid.h        # PID 算法模块（位式 + 增量式）
├── motor.c / motor.h    # 电机驱动（TB6612 封装）
├── encoder.c / encoder.h # 编码器读取（定时器编码器模式）
├── serialplot.c/.h      # SerialPlot 数据发送协议
├── button.c / button.h  # 按钮扫描 + 去抖
├── adc.c / adc.h        # 电位器 ADC 读取
├── timer.c / timer.h    # 定时器中断（调控周期 T）
└── mpu6050.c / .h       # 角度传感器驱动（倒立摆专用）
```

### 程序执行流程

```
启动
 ├─ 初始化：时钟、GPIO、定时器、ADC、UART、电机驱动
 ├─ 主循环：
 │   ├─ 扫描按钮 → 切换模式 / 启停 / 保存参数
 │   ├─ 读取 4 路 ADC → 映射到 Kp、Ki、Kd、目标值
 │   ├─ 根据当前模式显示状态（LED / 串口）
 │   └─ 等待定时器标志位
 │
 └─ 定时器中断（调控周期 T = 10ms）：
     ├─ 1. 读编码器 → 当前速度 / 位置
     ├─ 2. 读角度传感器 → 当前角度（倒立摆）
     ├─ 3. 计算误差 error = 目标值 - 当前值
     ├─ 4. 调用 PID_Compute() → 输出 u
     ├─ 5. 限幅：u 映射到 PWM 占空比范围（0~1000）
     ├─ 6. Motor_SetOutput(u) → 更新 PWM
     └─ 7. SerialPlot_Send(target, actual, u) → 串口发送数据
```

### 调控周期 T 的选择

- 编码电机定速/定位：T = 10ms（编码器分辨率够，电机响应速度合适）
- 倒立摆：T = 5ms（姿态传感器更新周期的下限，系统不稳定需要快响应）
- T 不能小于传感器更新周期，否则读取的是旧数据

### 三种程序架构的权衡

课程演示了三种写法，实际选用**定时器中断**方案（方案 2）：
- 方案 1（主循环 delay）：简单但有时间漂移
- 方案 2（定时器中断）：精确，业内标准。注意 ISR 里不能调用非重入函数
- 方案 3（中断置标志，主循环轮询）：灵活但可能被主循环阻塞

## 设备驱动

### 电机驱动 — TB6612

```
控制真值表（一路为例）：
AIN1=0, AIN2=0 → 刹车（停止）
AIN1=1, AIN2=0 → 正转，PWMA 控制占空比
AIN1=0, AIN2=1 → 反转，PWMA 控制占空比
AIN1=1, AIN2=1 → 刹车
```

- PWM 频率：10-20kHz（高于人耳听觉范围，避免啸叫）
- STM32 定时器输出 PWM：TIMx CH1/CH2，PWM 模式 1
- GPIO 控制方向脚：两个推挽输出
- 封装接口：`Motor_SetSpeed(int16_t speed)`，正值为正转，负值为反转，内部处理方向和占空比

### 编码器读取

- STM32 定时器配置为**编码器模式**（Encoder Mode）
- 输入脚：TIMx CH1 (A 相) + CHx CH2 (B 相)，GPIO 配置为浮空输入
- 编码器模式 3：TI1 和 TI2 的每个边沿都计数，四倍频，最高分辨率
- 计数值直接读取 `TIMx->CNT`
- 速度计算：每 T 时间读取一次 CNT，`speed = (cnt_current - cnt_last) / T / encoder_lines / gear_ratio`
- 位置计算：直接累计 CNT 差值，`position = cumulative_counts / encoder_lines / gear_ratio * 360°`

### SerialPlot 数据发送

SerialPlot 是 PC 端开源工具，通过串口接收数据并实时绘制波形，用于 PID 调参可视化。

**通信协议**：自定义二进制帧格式
```
帧头(2B) | 通道数(1B) | 数据类型(1B) | 数据(4B×N) | 帧尾(2B)
```

典型发送 3 条曲线（目标值、实际值、输出量）：

```c
// 每调控周期发送一次
typedef struct {
    uint8_t header[2];   // 0xAA 0xAA
    uint8_t channels;    // 3
    uint8_t type;        // float = 4
    float data[3];       // target, actual, output
    uint8_t tail[2];     // 0x55 0x55
} SerialPlotFrame;
```

SerialPlot 配置：串口号 + 波特率（115200/256000），导入帧格式定义文件。

调试价值：你能直观看到 PID 调参对阶跃响应的影响 —— 超调量、调节时间、稳态误差，肉眼可见。

### 电位器（旋钮）ADC 读取

- 4 路 ADC，轮询或 DMA 模式
- 12 位分辨率（0-4095），映射到 Kp/Ki/Kd/目标值 的取值范围
- Kp: 0.0 ~ 50.0，Ki: 0.0 ~ 5.0，Kd: 0.0 ~ 20.0
- 目标值：定速 0 ~ 最高转速，定位 0 ~ 360°（或 -180° ~ 180°）

### 按钮处理

- GPIO 输入，配置内部上拉（按下为低电平）
- 软件去抖：连续两次读取（间隔 20ms）均检测到按下才算有效
- 功能分配：模式切换、启停、参数保存/加载

### 角度传感器（MPU6050）— 倒立摆专用

- I2C 通信（SCL/SDA），地址 0x68
- 读取加速度计和陀螺仪原始数据
- 互补滤波融合得到角度：`angle = 0.98 * (angle + gyro*dt) + 0.02 * acc_angle`
- 更新周期 ~5ms，是倒立摆调控周期 T 的下限

## 实验：电机定速控制

### 目标

让编码电机稳定在用户设定的目标转速，旋钮实时改变目标，电机跟随。

### 控制策略

**PI 控制**（不需要 D）。定速控制不存在"冲过头"的问题 —— 速度低于目标就一直低于目标，不会反弹震荡。所以只用到 PI 两个参数。

### 实验步骤

1. **硬件连接**：ST-Link 接 STM32，TB6612 供电和电机接线，编码器接入定时器 GPIO
2. **驱动验证**：不跑 PID，直接用固定 PWM 驱动电机，用 SerialPlot 观察空载转速是否正确被读到，确认编码器方向正负一致
3. **基础 PI 调参**：
   - 先只调 P：从 0 开始逐步增大 Kp，观察速度如何逼近目标。Kp 太小 → 速度上不去，Kp 太大 → 速度波动大
   - 再加 I：加入 Ki，消除稳态误差。观察 Ki 从小到大时，速度如何从"差一点"变成"刚好贴上"
4. **旋钮在线调参**：用 4 个旋钮分别控制 Kp、Ki、目标速度。实时拧动，观察 SerialPlot 波形变化
5. **验证**：快速扭转目标旋钮，观察阶跃响应 —— 上升时间、超调（PI 不应该有明显超调）、稳态精度

### 预期现象

| 参数情况 | SerialPlot 波形表现 | 原因 |
|---------|-------------------|------|
| Kp 太小 | 实际速度远低于目标，差距很大 | P 项力量不够推动电机克服摩擦 |
| Kp 合适，Ki=0 | 实际速度接近目标但始终差 2-3% | 稳态误差，P 项在误差很小时输出太小 |
| Kp+Ki 合适 | 实际速度快速追上目标并稳定 | PI 配合到位 |
| Ki 太大 | 速度超调后缓慢回落 | I 项累积过多 |
| Kp 太大 | 速度在目标附近明显抖动 | 比例增益过大，放大了编码器的微小误差 |

### 关键代码模式

```c
// 定时器中断中执行（每 10ms）
void PID_SpeedControl(void) {
    // 1. 读当前速度
    float current_speed = Encoder_ReadSpeed();  // 单位：转/分钟

    // 2. 读目标速度（旋钮 ADC）
    float target_speed = ADC_ReadTargetSpeed();

    // 3. PI 计算
    float error = target_speed - current_speed;
    float output = PID_Compute(&pid_speed, error);

    // 4. 输出限幅并驱动
    int16_t pwm = (int16_t)CLAMP(output, -1000, 1000);
    Motor_SetSpeed(pwm);

    // 5. 发给 SerialPlot 画图
    SerialPlot_Send(target_speed, current_speed, output);
}
```

## 实验：电机定位控制

### 目标

让编码电机转到并停在指定的角度位置（如 180°），旋钮改变目标位置。

### 控制策略

**PID 控制**（需要 D 项）。位置控制会有"冲过头"的问题 —— 电机惯性会让它冲过目标位置再倒回来，产生震荡。D 项提供阻尼，抑制震荡。

### 与定速控制的区别

| 维度 | 定速控制 | 定位控制 |
|------|---------|---------|
| 控制对象 | 速度 | 位置（编码器脉冲累计） |
| 被控量来源 | 单位时间内的编码器增量 | 编码器累积计数值 |
| 所需 PID 项 | PI | PID（需要 D 项抑制震荡） |
| 误差特征 | 速度误差，不会自然震荡 | 位置误差，惯性导致冲过目标并反弹 |

### 实验步骤

1. **位置计算**：用编码器累积计数值换算角度。`position = (float)total_counts / lines / gear_ratio * 360.0`
2. **零点校准**：启动时当前位置记为 0°，或手动触发校准
3. **调参顺序（P → D → I）**：
   - 先调 P：设一个 180° 的目标，观察电机转过去后的行为
   - 加 D：如果冲过去又倒回来（震荡），加入 D 项抑制。逐步增大 D 直到震荡消失
   - 加 I：如果停在 178° 不动了（有稳态误差），加入 I 消除
4. **阶跃测试**：快速切换目标（如 0° → 180°），观察响应曲线
5. **验证精度**：多次重复定位，记录最终位置偏差

### 预期现象

| 现象 | 原因 | 对策 |
|------|------|------|
| 冲过目标，来回摆几次才停 | Kp 太大 / 缺 D 项 | 降低 Kp，或增大 Kd |
| 停在目标附近但差一点 | 缺 I 项，有稳态误差 | 增大 Ki |
| 修正很慢，过很久才到位 | Kp 太小 | 增大 Kp |
| 在目标点附近高频抖动 | Kd 太大（放大了噪声） | 降低 Kd，或加低通滤波 |
| 不动 | 误差过小被死区过滤 / P 太小 | 检查死区阈值，增大 Kp |

### 关键代码模式

```c
// 定时器中断中执行（每 10ms）
void PID_PositionControl(void) {
    // 1. 读当前位置（累计角度）
    float current_pos = Encoder_ReadPosition();  // 单位：度

    // 2. 读目标位置
    float target_pos = ADC_ReadTargetPosition();

    // 3. PID 计算
    float error = target_pos - current_pos;
    float output = PID_Compute(&pid_position, error);

    // 4. 输出限幅并驱动
    int16_t pwm = (int16_t)CLAMP(output, -1000, 1000);
    Motor_SetSpeed(pwm);  // 位置环输出直接给电机

    // 5. 发给 SerialPlot
    SerialPlot_Send(target_pos, current_pos, output);
}
```

## 实验：倒立摆双环 PID 控制

### 目标

让倒立摆维持直立状态，并能在外力扰动下自动恢复。进阶目标：从自然下垂状态自动摆起。

### 为什么需要双环

单环 PID 控制一个量。倒立摆有两个相互耦合的目标：**角度（要竖直）** 和 **位置（不能跑偏）**。

你只有一个执行器（减速电机控制支点水平移动），却要同时满足两个目标。双环的解决方案是嵌套：

```
外层（角度环）的 PID 输出 → 作为内层（速度环）的目标值
```

逻辑链：
1. 摆杆右倾 → 角度环说"支点需要向右加速跟上"
2. 角度环输出一个速度指令 → 给速度环
3. 速度环驱动电机执行

### 控制框图

```
目标角度(0°) ─→ [角度 PID] ─→ 目标速度 ─→ [速度 PID] ─→ PWM ─→ 电机 ─→ 支点移动
              ↑ 当前角度（MPU6050）             ↑ 当前速度（编码器）
```

- **外环（角度环）**：输入 = 目标角度 0° - 当前角度，输出 = 目标速度。**这个环用了 P + D**。角度环需要 D 项预测角度变化趋势，提前"接住"摆杆。一般不需要 I（积分会把摆杆推偏）。
- **内环（速度环）**：输入 = 目标速度 - 当前速度，输出 = PWM。**这个环用了 PI**。同定速控制，不需要 D。

### 硬件搭建要点

1. **摆杆自由旋转**：支点用轴承，尽量减小摩擦
2. **导电滑环走线**：MPU6050 固定在摆杆上，信号线经滑环接到 STM32。滑环的接触电阻会影响 I2C 通信质量，注意走线尽量短
3. **角度传感器安装**：MPU6050 装在摆杆质心上方或下方均可，只要初始化时标定好零角度（竖直向上）
4. **机械平衡检查**：不加控制时，摆杆自然下垂。支点在导轨上应平滑移动，无明显卡滞

### 实验步骤

**第一阶段：角度环单独调试**

1. MPU6050 驱动验证：用 SerialPlot 观察角度读数，手动摆动摆杆，确认读数连续无跳变
2. 角度环 PD 调参：
   - Kp（角度 P）：摆杆倾斜 10°，电机应明显在"扶"它
   - Kd（角度 D）：加入后摆杆回正时不应有大幅度超调
   - 手动推一下摆杆，观察恢复过程

**第二阶段：双环联调**

3. 速度环 PI 调参：同定速控制实验
4. 双环级联运行：
   - 摆杆直立，观察是否能在零点附近稳定
   - 轻推摆杆，观察恢复速度和震荡情况
5. 抗扰动测试：用手指轻点摆杆，观察恢复过程

**第三阶段：自动启摆（进阶）**

6. 启摆策略：摆杆从自然下垂状态开始，系统先做一个大幅度的周期摆动（能量注入），当摆动幅度接近直立时，切换到双环 PID 控制
7. 能量判断：当 `|angle - 180°|`（偏离下垂位置的角度）超过某个阈值（如 150°），意味着摆杆已经摆到了接近直立的位置，切入控制

### 预期现象

| 现象 | 原因 | 对策 |
|------|------|------|
| 摆杆直立但缓慢漂移 | 角度环缺 I 或角度传感器有偏置 | 检查 MPU6050 零点标定 |
| 摆杆在零点附近高频抖动 | 角度环 Kd 太大，放大传感器噪声 | 降低 Kd，或加不完全微分 |
| 摆杆倒了电机还在转 | 角度环 Kp 不够大，无法克服重力 | 增大角度环 Kp |
| 支点跑偏，摆杆直立但一直在移动 | 角度环输出无 I 导致的自然漂移 | 接受（双环不需要位置锁定），或再加外层位置环 |
| 自动启摆失败 | 能量不足或切换时机不对 | 增大启摆幅度，调整切换阈值 |
| 摆杆恢复了但震荡很久 | 角度环 Kd 太小 | 增大 Kd |

### 关键代码模式

```c
// 定时器中断中执行（每 5ms）
void InvertedPendulum_Control(void) {
    // 1. 读传感器
    float current_angle = MPU6050_ReadAngle();   // 摆杆角度
    float current_speed = Encoder_ReadSpeed();    // 支点速度

    // 2. 外环 — 角度 PD（目标角度 = 0°，即竖直向上）
    float angle_error = TARGET_ANGLE - current_angle;  // TARGET_ANGLE = 0
    float target_speed = PID_Angle_Compute(angle_error); // 输出作为速度环目标

    // 3. 内环 — 速度 PI
    float speed_error = target_speed - current_speed;
    float output = PID_Speed_Compute(speed_error);

    // 4. 输出到电机
    int16_t pwm = (int16_t)CLAMP(output, -1000, 1000);
    Motor_SetSpeed(pwm);
}
```

### 调参经验

倒立摆调参的难点在角度环。经验顺序：

1. 先锁定内环（速度 PI）：单独调好，确保速度响应又快又稳
2. 再调外环（角度 PD）：从小到大加角度 Kp，直到摆杆能"站住"（哪怕抖）
3. 加角度 Kd：抑制抖动，加到摆杆稳定且恢复快
4. 微调：轻推摆杆，如果恢复太慢加角度 Kp，如果震荡加角度 Kd
