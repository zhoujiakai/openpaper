00代码组织结构 — 以倒立摆为例

这五层形成一个依赖层级，下层不依赖上层：

调用方向 ↓ User (main.c + PID.c) ├── 调用 → │ Hardware (Motor, Encoder, RP, AD, Key, LED, OLED, Serial) │ ├── 调用 → │ │ System (Timer, Delay) │ │ └── 调用 → │ │ Library (stm32f10x_*.h) │ │ └── 依赖 → │ │ Start (stm32f10x.h, core_cm3.h) │ └── 调用 → │ Library (stm32f10x_gpio, _tim, _adc, _usart, _rcc...) │ └── 依赖 → │ Start └── 调用 → System (Timer, Delay) └── 调用 → Library → Start
实战意义：如果要改倒立摆的控制算法，只需要改 User/main.c。如果要加新传感器（如 MPU6050），在 Hardware/ 下新增一个驱动文件，再在 User/main.c 里 #include 进来就行。Start/ 和 Library/ 是 SDK 级代码，完全不需要动。

02驱动层代码逐行精读

所有实验都依赖这 6 个驱动模块。它们构成了 "硬件抽象层"，让 PID 逻辑代码完全不用关心底层寄存器。

2.1 PWM 驱动 — 用定时器产生 20kHz 方波

文件：Hardware/PWM.c + PWM.h

硬件映射：TIM2 通道 1 → PA0（复用推挽输出）

工程参数计算：

  • 系统时钟 72MHz → PSC = 36-1 → 定时器计数频率 = 72M / 36 = 2MHz
  • ARR = 100-1 → PWM 周期 = 100 / 2MHz = 50μs → PWM 频率 = 20kHz
  • 20kHz 高于人耳听觉上限（~16kHz），电机不会发出啸叫声
  • CR = 0~100 → 占空比 = CCR / 100，步进 1%
为什么 PSC=36-1 而不是别值？ 需要 CCR 范围有足够的粒度（100 级步进），同时 PWM 频率 >16kHz。 如果 PSC=72-1，ARR 仍用 100-1，则 PWM 频率 = 10kHz，会听到电机啸叫。 如果 ARR=1000-1，则频率降为 2kHz，虽然也能转，但噪声非常大。

PWM_Init 关键配置：

// 1. 时基单元
TIM_TimeBaseInitStructure.TIM_Period = 100 - 1;     // ARR：决定 PWM 周期
TIM_TimeBaseInitStructure.TIM_Prescaler = 36 - 1;  // PSC：计数脉冲分频

// 2. 输出比较
TIM_OCInitStructure.TIM_OCMode = TIM_OCMode_PWM1;       // PWM 模式 1：CNT
TIM_OCInitStructure.TIM_OCPolarity = TIM_OCPolarity_High; // 输出极性：高有效
TIM_OCInitStructure.TIM_Pulse = 0;                      // 初始占空比 0%，电机不动

PWM_SetCompare1：封装 TIM_SetCompare1(TIM2, Compare)，参数 0~100 对应 0%~100% 占空比。

一句话总结：PWM 模块 = TIM2_CH1(PA0) 输出 20kHz PWM，占空比由 CCR1 寄存器决定，范围 0~100 对应 0%~100%。

2.2 Motor 驱动 — 方向脚 + PWM 组成 H 桥控制

文件：Hardware/Motor.c + Motor.h

引脚映射：

引脚功能连接
PA0 (TIM2_CH1)PWM 信号TB6612 的 PWMA 脚
PB12GPIO 推挽输出TB6612 的 AIN1 脚（方向控制）
PB13GPIO 推挽输出TB6612 的 AIN2 脚（方向控制）

Motor_SetPWM 的 H 桥控制真值表：

Motor_SetPWM(n)PB12 (AIN1)PB13 (AIN2)PWMA 占空比电机状态
n > 00（低）1（高）n%正转，转速正比于 n
n = 0000%刹车停止
n < 01（高）0（低）|n|%反转，转速正比于 |n|
n=100（正满）01100%正转最高速
n=-100（反满）10100%反转最高速
为什么 n=0 时不设置为 0，1（正转且占空比 0%）？ 因为占空比 0% 时 PWM 输出恒低，AIN=0,1 + PWM恒低 = 电机一端接地一端高阻，不会刹车。 AIN=0,0 是 TB6612 的显式刹车状态，电机会快速停止。对于 PID 停止控制，刹车比自然滑行更好。

代码拆解：

void Motor_SetPWM(int8_t PWM)
{
    if (PWM >= 0)                            // 正转或停止
    {
        GPIO_ResetBits(GPIOB, GPIO_Pin_12); // AIN1 = 0
        GPIO_SetBits(GPIOB, GPIO_Pin_13);   // AIN2 = 1
        PWM_SetCompare1(PWM);               // 占空比 = PWM/100
    }
    else                                    // 反转
    {
        GPIO_SetBits(GPIOB, GPIO_Pin_12);   // AIN1 = 1
        GPIO_ResetBits(GPIOB, GPIO_Pin_13); // AIN2 = 0
        PWM_SetCompare1(-PWM);              // 占空比 = |PWM|/100
    }
}

注意参数类型是 int8_t，范围 -128~127。上层 PID 限幅到 ±100，正好匹配。

2.3 Encoder 驱动 — 定时器编码器模式 + 四倍频

文件：Hardware/Encoder.c + Encoder.h

硬件映射：TIM3 CH1 (PA6) = A 相，TIM3 CH2 (PA7) = B 相

编码器工作原理（四倍频）：

AB 相编码器输出两路相位差 90° 的方波。STM32 定时器在编码器模式下，通过检测 A、B 两路的上升沿和下降沿来计数：

  • 每转脉冲数 = 编码器线数（物理线数）× 4（四倍频）
  • 如果编码器线数是 1024，则每转计数 4096
  • 向上计数 = 正转，向下计数 = 反转（取决于接线）

初始化关键配置：

void Encoder_Init(void)
{
    // 1. 时基单元 — ARR 设最大值，防止溢出重置
    TIM_TimeBaseInitStructure.TIM_Period = 65536 - 1;// 16 位计数器最大值
    TIM_TimeBaseInitStructure.TIM_Prescaler = 1 - 1;  // 不分频，用原始编码器脉冲

    // 2. 输入滤波 — 0xF 最大滤波，抑制编码器信号毛刺
    TIM_ICInitStructure.TIM_ICFilter = 0xF;

    // 3. 编码器模式 3：A、B 两路的所有边沿都计数（四倍频）
    TIM_EncoderInterfaceConfig(TIM3,
        TIM_EncoderMode_TI12,       // 两路全用
        TIM_ICPolarity_Rising,      // A 相不反相
        TIM_ICPolarity_Falling);    // B 相不反相
}
TIM_ICPolarity_Rising/Falling 在编码器模式下的真实含义： 这里的 Rising/Falling 已经不再表示边沿极性，而是表示是否对输入信号反相。 Rising = 不反相，Falling = 反相。如果电机正转时计数值在减小，把 B 相的参数改成 Rising 即可翻转计数方向。

Encoder_Get — 增量读取：

int16_t Encoder_Get(void)
{
    int16_t Temp;
    Temp = TIM_GetCounter(TIM3);  // 读取当前计数值
    TIM_SetCounter(TIM3, 0);    // 立即清零，下次读取的是增量
    return Temp;
}

这是整个系统中最重要的设计之一：

  • 返回值不是"累计转了多少圈"，而是"自上次读取以来转了多少刻度"
  • 如果每隔 40ms 读取一次，返回值就正比于转速（单位时间的位移 = 速度）
  • 电机的标准转速换算：Speed_rps = Encoder_Get() / 408.0 / 0.04（转/秒）
  • 注意 Temp 是 int16_t（有符号），正转返回正数，反转返回负数
为什么是 int16_t？ 40ms 内电机最大转速对应的编码器增量不会超过 ±32767。使用 int16_t 刚好合适，且自动处理正反转符号。 如果读取周期更长（如 100ms），需要改用 int32_t 防止溢出。

速度 → 位置的转换（不在 Encoder 模块中，但依赖 Encoder_Get）：

Speed = Encoder_Get();      // 获取速度（单位：编码器刻度/40ms）
Location += Speed;          // 位置 = 累积速度（单位：编码器刻度）

这里的位置是"相对初始位置的编码器刻度数"，电机转一圈约 408 个刻度（取决于具体编码器线数和减速比）。

2.4 Timer 中断 — 1ms 精确节拍

文件：System/Timer.c + Timer.h

硬件：TIM1 高级定时器，更新中断，1ms 周期

计算：72MHz / 72(PSC) / 1000(ARR) = 1000Hz = 1ms 间隔

TIM_TimeBaseInitStructure.TIM_Period = 1000 - 1;   // ARR = 999
TIM_TimeBaseInitStructure.TIM_Prescaler = 72 - 1;  // PSC = 71

中断服务函数的模板：

void TIM1_UP_IRQHandler(void)
{
    if (TIM_GetITStatus(TIM1, TIM_IT_Update) == SET)  // 检查更新中断标志
    {
        /* 你的控制代码写在这里，每 1ms 执行一次 */

        TIM_ClearITPendingBit(TIM1, TIM_IT_Update);      // 必须清除中断标志
    }
}

中断里能做什么，不能做什么：

  • 能做：读传感器（编码器、ADC）、算 PID、写 PWM 寄存器
  • 不能做：长时间延时（阻塞中断）、printf等重入函数、大数组栈分配

计次分频（software timer）是课程中最重要的中断编程技巧：

static uint16_t Count;       // static 保留值，默认 0

Count ++;                       // 每 1ms 自增
if (Count >= 40)             // 满 40 次 = 40ms
{
    Count = 0;                // 清零重计
    /* 每 40ms 执行一次的控制代码 */
}

这样只用了一个 1ms 定时器，就获得了 5ms、40ms、50ms 等多个控制周期。

2.5 RP 电位器 — ADC2 四路轮询

文件：Hardware/RP.c + RP.h

引脚：PA2/PA3/PA4/PA5 = ADC2 通道 2/3/4/5

设计关键：同一时刻只能读取一个通道。RP_GetValue(n) 中动态切换规则组通道，然后触发一次转换。

uint16_t RP_GetValue(uint8_t n)
{
    if (n == 1)
        ADC_RegularChannelConfig(ADC2, ADC_Channel_2, 1, ...);
    else if (n == 2)
        ADC_RegularChannelConfig(ADC2, ADC_Channel_3, 1, ...);
    // ... 3, 4 同理

    ADC_SoftwareStartConvCmd(ADC2, ENABLE);             // 触发一次转换
    while (ADC_GetFlagStatus(ADC2, ADC_FLAG_EOC) == RESET);// 等待完成
    return ADC_GetConversionValue(ADC2);               // 返回 0~4095
}

在主循环中的参数映射（3 个旋钮调 Kp/Ki/Kd，1 个设目标值）：

Kp = RP_GetValue(1) / 4095.0 * 2;              // 范围 0~2
Ki = RP_GetValue(2) / 4095.0 * 2;              // 范围 0~2
Kd = RP_GetValue(3) / 4095.0 * 2;              // 范围 0~2

03单环 PID：定速控制的代码实现

3.1 位置式 PID 公式在 C 代码中的精确表达

位置式 PID 的数学公式：

k u(k) = Kp·e(k) + Ki·∑ e(j) + Kd·[e(k) - e(k-1)] j=0

在 C 代码中的精确对应：

// 位置式 PID 公式的 C 语言翻译
Out = Kp * Error0                          // 比例项：当前误差 × Kp
    + Ki * ErrorInt                        // 积分项：累积误差 × Ki
    + Kd * (Error0 - Error1);              // 微分项：误差变化率 × Kd
变量数学含义C 变量名更新方式
e(k)当前周期误差Error0Error0 = Target - Actual
e(k-1)上一周期误差Error1Error1 = Error0（在上次）
∑e误差累积ErrorIntErrorInt += Error0
u(k)控制器输出Out公式计算得出
位置式 vs 增量式的区别：
  • 位置式：输出 = 公式直接算出的绝对 PWM 值。代码如上面的公式。
  • 增量式：输出 = 上次输出 + 增量。代码 Out += Kp*(Error0-Error1) + Ki*Error0 + Kd*(Error0-2*Error1+Error2)
位置式更直观，课程以位置式为主线。增量式的优势是无积分饱和，但需要小心输出累积。

3.2 误差处理的三个关键步骤

每个 PID 周期执行六个步骤，缺一不可：

// 步骤 1：获取实际值（从传感器读）
Actual = Encoder_Get();         // 编码器增量 = 当前速度

// 步骤 2：更新误差历史
Error1 = Error0;                // 上次误差 ← 当前误差
Error0 = Target - Actual;       // 当前误差 = 目标 - 实际

// 步骤 3：误差积分（带 Ki 保护）
if (Ki != 0)                   // 仅当积分项启用时
    ErrorInt += Error0;         // 累加当前误差
else
    ErrorInt = 0;                // Ki=0 时清零积分，防止积压

// 步骤 4：PID 计算
Out = Kp * Error0 + Ki * ErrorInt + Kd * (Error0 - Error1);

// 步骤 5：输出限幅
if (Out > 100) { Out = 100; }    // 防止 PWM 超限
if (Out < -100) { Out = -100; }  // 负方向同理

// 步骤 6：执行控制
Motor_SetPWM(Out);              // 输出到电机

3.3 实验 02 完整中断代码逻辑

═══════════════ 定速控制中断流程（1ms 基准，40ms 控制周期） ═══════════════ TIM1_UP_IRQHandler() 每 1ms 进入一次 │ ├─ Key_Tick() ← 按键消抖 tick │ ├─ Count++ ← 计次分频器 │ └─ if Count >= 40 ← 40ms 才进入控制循环 │ │ │ ├─ Actual = Encoder_Get() ← 读编码器增量（速度） │ ├─ Error1 = Error0 ← 迭代误差历史 │ ├─ Error0 = Target - Actual ← 计算当前误差 │ ├─ ErrorInt += Error0 ← 积分累积（Ki≠0时） │ ├─ Out = Kp*Error0 + Ki*ErrorInt + Kd*(Error0-Error1) ← PID │ ├─ Out 限幅到 [-100, 100] ← 保护 │ └─ Motor_SetPWM(Out) ← 执行 │ └─ TIM_ClearITPendingBit() ← 清除标志

为什么控制周期选 40ms？

  • 40ms 内电机正常转速下编码器增量约 50~150（可分辨）
  • 太短（如 5ms）：编码器增量太小，信号噪声占比大，微分项会放大噪声
  • 太长（如 100ms）：控制延迟大，系统响应慢，容易震荡

04双环 PID：定速 + 定位串级控制

4.1 为什么需要双环而不是单环定位

单环定位的问题：位置环输出直接给电机 PWM，外环输出变化剧烈（从 100 瞬间跳到 -100）。电机响应跟不上，结果要么冲过头很远，要么到达后震荡。

双环串级的思路：

┌──────────┐ ┌──────────┐ 目标位置 ──→[外环 位置 PID]──→ 目标速度 ──→[内环 速度 PID]──→ PWM ──→ 电机 ↑ 实际位置 ↑ 实际速度 │ │ 编码器累积 编码器增量

外环思考"要往哪转、转多快"，内环负责"怎么转到这个速度"。外环输出被限幅到 ±20，意味着电机不会超过 20% 满速去追踪位置，大大减少了冲过头。

4.2 实验 12 双环裸写 vs 实验 13 struct 封装

实验 12 的代码用两套独立变量实现了双环，实验 13 改用 PID_t 结构体 + PID_Update 函数封装。对比一下代码变化：

实验 12（裸写）— 内环变量：

float InnerTarget, InnerActual, InnerOut;
float InnerKp, InnerKi, InnerKd;
float InnerError0, InnerError1, InnerErrorInt;

实验 13（封装）— 替换为：

PID_t Inner = {
    .Kp = 0.3, .Ki = 0.3, .Kd = 0,
    .OutMax = 100, .OutMin = -100
};

PID_t Outer = {
    .Kp = 0.3, .Ki = 0, .Kd = 0.4,
    .OutMax = 20, .OutMin = -20   // 外环输出限幅更小，控制速度上限
};

核心的串级连接（唯一的"让两个环连起来"的代码）：

Inner.Target = Outer.Out;    // 外环输出 → 内环目标值

中断中的执行流程（12 和 13 逻辑相同，只是 13 用 PID_Update 替代了手动计算）：

每 40ms 执行一次（内环计数器触发） Speed = Encoder_Get() Location += Speed ── 内环（速度环） Inner.Actual = Speed PID_Update(&Inner) ← 自动完成 Error0/1、积分、PID 计算、限幅 Motor_SetPWM(Inner.Out) 每 40ms 执行一次（外环计数器触发，与内环同频） ── 外环（位置环） Outer.Actual = Location PID_Update(&Outer) ← 同上 Inner.Target = Outer.Out ← ★ 串级连接点

4.3 PID 结构体封装 — PID_Update 函数精读

PID.h 结构体定义：

typedef struct {
    float Target;        // 目标值（可被外部修改）
    float Actual;        // 实际值（每周期由中断写入）
    float Out;           // PID 输出值（计算得出）

    float Kp;            // 比例系数
    float Ki;            // 积分系数
    float Kd;            // 微分系数

    float Error0;        // 本次误差 e(k)
    float Error1;        // 上次误差 e(k-1)
    float ErrorInt;      // 累积误差 ∑e

    float OutMax;        // 输出上限
    float OutMin;        // 输出下限
} PID_t;

PID_Update 函数全景：

void PID_Update(PID_t *p)
{
    // ① 误差更新
    p->Error1 = p->Error0;                    // 上次 ← 当前
    p->Error0 = p->Target - p->Actual;       // 当前 ← 目标 − 实际

    // ② 积分累积（带 Ki 保护，避免调试时积分爆炸）
    if (p->Ki != 0)
        p->ErrorInt += p->Error0;            // 启用积分时累加
    else
        p->ErrorInt = 0;                      // Ki=0 时清零积分

    // ③ 位置式 PID 公式
    p->Out = p->Kp * p->Error0
            + p->Ki * p->ErrorInt
            + p->Kd * (p->Error0 - p->Error1);

    // ④ 输出限幅
    if (p->Out > p->OutMax) { p->Out = p->OutMax; }
    if (p->Out < p->OutMin) { p->Out = p->OutMin; }
}
Ki 保护的设计智慧：调试时会把 Ki 设 0 先调 PD，此时如果 ErrorInt 一直累加，哪怕调到 Ki 不为 0，之前积累的巨大积分值也会瞬间输出满幅 PWM，电机失控。所以 if (Ki != 0) ... else ErrorInt = 0 是防止"历史债"的正确做法。

使用方式总结：

// 中断里调用，以倒立摆角度环为例
AnglePID.Actual = Angle;       // ① 写入实际值
PID_Update(&AnglePID);         // ② 一次完成 PID 计算（Target 已在 main 设置）
Motor_SetPWM(AnglePID.Out);    // ③ 读取输出并执行

05倒立摆双环 PID 控制

5.1 倒立摆的控制框图

倒立摆有两个被控量：角度（摆杆要竖直）和位置（横杆不能跑偏）。课程代码用双环串级结构：

┌──────────────── 倒立摆双环控制结构 ────────────────┐ │ │ 位置目标 ──→ [位置环 PD] ──→ 角度偏移 ──→ [角度环 PID] ──→ PWM ──→ 电机 │ ↑ 横杆实际位置 ↑ 摆杆实际角度 │ │ │ │ 编码器累积 角度传感器 AD_GetValue() │ │ 串级连接：AnglePID.Target = CENTER_ANGLE - LocationPID.Out │ 含义：位置环的输出是 "偏离中心角度 Δ" │ 正输出 → 角度目标下降 → 摆杆左倾 → 横杆右移 │ 负输出 → 角度目标上升 → 摆杆右倾 → 横杆左移 └────────────────────────────────────────────────────┘

角度环目标值的含义：

AnglePID.Target = CENTER_ANGLE - LocationPID.Out;
// 当 位置环 Out = 0  →  角度目标 = 中心角度（竖直）
// 当 位置环 Out > 0  →  角度目标 < 中心角度（摆杆略倾）
//                       电机为纠正摆杆角度而移动横杆
为什么是 CENTER_ANGLE - LocationPID.Out 而不是 +？ 这取决于角度传感器的安装方向。如果位置环输出正数时想让横杆向右移动，需要让摆杆右倾（角度传感器值减小），所以是减号。如果实际效果方向反了，把减号改加号或调整 Kp 符号即可。本质上是符号约定问题。

课程中的角度传感器是电位器式，通过 AD 值直接判断摆杆位置：

倒下自动停止：

if (! (Angle > CENTER_ANGLE - CENTER_RANGE   // 500
    && Angle < CENTER_ANGLE + CENTER_RANGE))

}

当 RunState 为 0 时，中断中执行的是 Motor_SetPWM(0)，电机停止。不是简单的 "停止 PID 计算"，而是 "停止 PID + 停止电机 + 继续采样显示"。

按键控制横杆位置：

if (KeyNum == 2)                              // K2 按下 → 横杆正转一圈
{

    if (LocationPID.Target > 4080)          // 最多正转 10 圈

改变的是外环目标值。外环 PD 侦测到位置偏差后，控制横杆移动到目标位置。因为外环输出 + 角度环偏移的串级结构，摆杆在横杆移动过程中保持直立。

06自动启摆：有限状态机实现

6.1 启摆策略与状态设计

实验 16 的核心是用有限状态机实现自动启摆。摆杆从自然下垂状态开始，通过周期性地给电机施加瞬时驱动力，用共振原理逐步增大摆杆摆动幅度，直到摆杆进入直立区间后切入 PID 控制。

完整状态转移图：

停止状态(0) │ K1 按下 ▼ ┌──→ 判断状态(1) ←──────────────────────────┐ │ │ │ │ ├── 右侧最高点 → 21 左转_START_PWM ──→ 22 延时_START_TIME ──→ 23 右转_-START_PWM ──→ 24 延时_START_TIME ──→ 电机停 → 1 │ │ │ ├── 左侧最高点 → 31 右转_-START_PWM ──→ 32 延时_START_TIME ──→ 33 左转_START_PWM ──→ 34 延时_START_TIME ──→ 电机停 → 1 │ │ │ └── 进入中心区间 → 4 PID控制 │ │ └──────────────────────────────┘ 摆杆倒下 → 0

状态编码：0=停止，1=判断，21-24=左施力序列，31-34=右施力序列，4=PID 控制。

为什么状态码是 21-24 而不是 2-5？ 编码设计：十位数表示方向（十位=2 是左施力，十位=3 是右施力），个位数是序列中的步骤（1=施力，2=延时，3=反向施力，4=结束延时）。这是一种工程上的自文档化技巧——看到状态码就知道它在做什么。

6.2 实验 16 关键代码分析

判断摆杆是否到达最高点是启摆逻辑的核心：

/* 每 40ms 判断一次 */
/* 连续采样 3 次角度值，以 40ms 为间隔 */
Angle2 = Angle1;            // 120ms 前的角度
Angle1 = Angle0;            // 80ms 前的角度
Angle0 = Angle;             // 当前角度

/* 判断右侧最高点：3次角度都在右侧区间，且中间值最小 */
if (Angle0 > CENTER_ANGLE + CENTER_RANGE
 && Angle1 > CENTER_ANGLE + CENTER_RANGE
 && Angle2 > CENTER_ANGLE + CENTER_RANGE
 && Angle1 < Angle0
 && Angle1 < Angle2)
{
    RunState = 21;   // 在右侧最高点 → 向左施加驱动力
}

这是一个三样本峰值检测器。Angle1 是中间值，如果它比 Angle0 和 Angle2 都小，说明摆杆经过了角度最小值（最高点），此时给一个同方向的瞬时推力，就能在共振点注入能量。

施力序列（以左侧施力 31→34 为例）：

// 状态 31：立刻向右转
Motor_SetPWM(-START_PWM);       // 右转，力度由 START_PWM(35) 决定
CountTime = START_TIME;         // 计时器初始化为 START_TIME(100)
RunState = 32;                // 自动转入延时

// 状态 32：延时 100ms
CountTime --;                   // 每 1ms 减 1
if (CountTime == 0) RunState = 33;

// 状态 33：立刻向左转
Motor_SetPWM(START_PWM);
CountTime = START_TIME;
RunState = 34;

// 状态 34：延时 100ms
CountTime --;
if (CountTime == 0)
{
    Motor_SetPWM(0);           // 施力结束，电机停
    RunState = 1;              // 回到判断状态
}

每个施力周期=100ms 右转 + 100ms 左转 = 200ms = 5Hz。这个频率需要匹配摆杆的自然摆动频率（由摆杆长度决定）。实际调试时通过修改 START_TIME 来匹配。

判断进入直立区间：

/* 连续 2 次角度都在中心区间内，就认为稳定在直立位置附近 */
if (Angle0 > CENTER_ANGLE - 500 && Angle0 < CENTER_ANGLE + 500
 && Angle1 > CENTER_ANGLE - 500 && Angle1 < CENTER_ANGLE + 500)
{
    Location = 0;                  // 重置位置参考
    AnglePID.ErrorInt = 0;          // 清积分，防止启摆累积的误差积分干扰
    LocationPID.ErrorInt = 0;
    RunState = 4;                  // 切入 PID 控制
}

07核心设计模式总结

7.1 代码中用到的关键技术模式

模式出现在哪里为什么用
计次分频所有实验的中断用一个 1ms 定时器生成 5/40/50ms 多种周期，不需要多个定时器
结构体封装PID_t + PID_Update一个结构体 = 一个 PID 实例，多环复用同一函数
Ki 保护PID_Update 中Ki=0 时清零积分，防止调试时积分债问题
输出限幅PID_Update 末尾PID 计算的 Out 理论上无限大，必须限制到 PWM 范围（±100）
串级连接Inner.Target = Outer.Out外环的输出作为内环的目标，两个独立 PID 实例通过一行代码串联
有限状态机实验 16 启摆10 种状态覆盖停止→启摆→判断→PID 全流程
三样本峰值检测实验 16 判断最高点连续 3 次采样判别摆杆加速度方向，在共振点注入能量
