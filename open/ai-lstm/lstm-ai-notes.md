## 锚点

想象一条工厂的传送带。产品（信息）放在上面，从入口一直传到出口。传送带本身只负责平稳运输，不加工产品。

但工厂需要决定三件事：
- 上一批产品里，哪些废料该扔下传送带？
- 当前工位的新零件，哪些该放上传送带？
- 传送带上的产品，哪些该打包送出去？

LSTM 就是这么干的。传送带叫 **cell state（细胞状态）**，三个决定叫 **三门机制**。信息在传送带上靠加法流动，不会越走越淡。

---

## 核心问题

普通 RNN 为什么记不住长句子？

因为 RNN 每传一步信息，都要把上一步的记忆乘一遍权重矩阵，再过一遍激活函数。这就像把一张纸反复复印。复印三五次还看得清，复印五十次就成了一团灰。RNN 在序列长度超过 20-30 步时，梯度就会指数级缩小（vanish），早期信息彻底丢失。

LSTM 的做法是：**别让信息每次都经过复印机**。给它修一条直通高速公路，信息在上面靠加法走，想走多远走多远。

## 核心概念

### Cell State：信息高速公路

它是一条贯穿整个序列的传送带。每一步只做两件事：按比率忘掉一些旧内容，按比率加入一些新内容。更新方式是**加法**。

为什么加法重要？因为 RNN 的记忆更新是乘法链。乘法链走远了必然趋近于 0。加法链不会。

具体例子：你读一句话 "The clouds are dark"。读到 "clouds" 时，cell state 记下 "出现了天气名词"。读到 "are" 时，你不想让 "clouds" 的信息被稀释，所以遗忘门输出接近 1，保留了大部分记忆。读到 "dark" 时，输入门打开，把 "形容词修饰天气" 这个信号加进去。自始至终，"clouds" 的信息没有被反复压缩。

### Forget Gate：决定扔什么

它是一个小神经网络，看的是上一步的隐藏状态 + 当前输入，输出一个 0 到 1 之间的向量。

输出 1 表示"这块信息完全保留"，0 表示"彻底扔掉"。

例子：前文提到"昨天吃了火锅"，当前词是"今天"。遗忘门可能对"火锅"相关的维度输出接近 0 —— 时间切换了，旧菜单该扔了。

少了它会怎样？cell state 会被旧垃圾塞满，新信息进不来。

### Input Gate + Candidate：决定加什么

输入门也是 0 到 1 的向量。它控制"新候选信息"有多少能写进 cell state。

候选信息（candidate）由 tanh 生成，范围是 -1 到 1，代表"如果让我写，我想写什么"。输入门说"你写的这些，我批准多少"。

例子：当前词是 "dark"。候选信息提议写入 "形容词/天气/负面色彩"。输入门判断这个词确实有信息量，于是输出接近 1，批准写入。

少了输入门会怎样？所有新信息都一股脑塞进去，cell state 变成一锅粥。

### Output Gate：决定输出什么

cell state 更新完后，内容还在传送带上，不直接对外可见。输出门决定"当前这一步，把 cell state 的哪些部分展示给外面（也就是生成隐藏状态 h_t）"。

它先对 cell state 过一遍 tanh（把值压到 -1 到 1），再用输出门做逐元素过滤。

例子：cell state 里同时记着"天气名词"和"主语是第三人称"。当前要预测下一个词，输出门可能只放出"天气名词"这部分，因为"第三人称"对选词帮助不大。

少了输出门会怎样？每一步都得把全部记忆暴露出去，无关噪声干扰预测。

### Hidden State：对外窗口

它是 LSTM 单元对外的接口。既喂给下一个时间步，也喂给输出层做预测。

关键区分：cell state 是长期记忆库，hidden state 是当前工作窗口。一个管存，一个管用。

## 关键公式

** Forget Gate（遗忘门）**

算的是"旧记忆保留多少"。

$$f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)$$

$[h_{t-1}, x_t]$ 是把上一步隐藏状态和当前输入拼在一起。$\sigma$ 把结果压到 0 到 1 之间。接近 1 = 保留，接近 0 = 遗忘。

** Input Gate（输入门）**

算的是"新信息批准多少"。

$$i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)$$

** Candidate（候选记忆）**

算的是"我想写什么新内容"。

$$\tilde{C}_t = \tanh(W_C \cdot [h_{t-1}, x_t] + b_C)$$

用 tanh 是因为新内容可能是正向信号也可能是负向信号，范围 -1 到 1。

** Cell State 更新（核心）**

$$C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$$

$\odot$ 是逐元素相乘。旧的按遗忘门比例保留，新的按输入门比例加入。这是**加法**更新。

** Output Gate（输出门）**

算的是"当前对外展示多少"。

$$o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)$$

** Hidden State**

$$h_t = o_t \odot \tanh(C_t)$$

先对 cell state 做 tanh 压缩，再用输出门过滤。

## 逐节详解

### Section 1: RNN 的致命伤 —— 梯度消失

RNN 的隐藏状态更新是：

$$h_t = \tanh(W_h h_{t-1} + W_x x_t + b)$$

训练时用 Backpropagation Through Time（BPTT）。误差从第 t 步传回第 1 步，要连乘 t-1 个雅可比矩阵。

如果权重平均小于 1，连乘 100 次会变成 $0.9^{100} \approx 0.00003$。梯度彻底消失，前面步骤的权重永远学不到。

RNN 有效记忆长度大约 10-20 步。超过这个长度，早期输入和当前输出之间就断联了。

### Section 2: LSTM 的解法 —— 加法高速公路

LSTM 没有直接修复梯度消失的数学根因。它绕开了问题。

cell state 的更新是加法：$C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$。

对 $C_{t-1}$ 求偏导：$\partial C_t / \partial C_{t-1} = f_t$。

$f_t$ 是一个向量，每个维度是遗忘门的输出。当网络需要长期记忆时，遗忘门会学到输出接近 1。此时梯度反向传播时，乘以的是接近 1 的数，而不是一个被反复压缩的权重矩阵。

如果遗忘门平均 0.95，传 100 步后梯度是 $0.95^{100} \approx 0.006$。虽然也有衰减，但比 RNN 的 $10^{-5}$ 好太多了。

这条通路叫 **Constant Error Carousel（CEC）**，是 LSTM 能记住长期依赖的核心。

注意：LSTM 缓解的是梯度消失，对梯度爆炸帮助有限。训练时仍需梯度裁剪（gradient clipping）。

### Section 3: 三门如何协作

三门不是独立工作的，它们在同一时间步内串成一条流水线：

1. 先看当前输入和上一步输出，决定"旧记忆扔多少"（遗忘门）
2. 再决定"新信息写多少"（输入门 + 候选）
3. 更新 cell state（旧记忆保留部分 + 新信息加入部分）
4. 最后决定"对外展示多少"（输出门生成隐藏状态）

这个顺序不能乱。必须先决定扔什么，再决定加什么，否则新信息可能被自己刚扔掉的东西干扰。

### Section 4: 遗忘门偏置的初始化技巧

实际训练中，遗忘门偏置通常初始化为 1.0 或更高。这样初始状态下遗忘门输出接近 1，意味着"先记住一切"。

为什么这样做？因为网络早期训练时还没学会该忘什么。如果初始就忘得太多，长期信息在训练初期就丢了，网络永远学不会长期模式。

这个技巧在 2015 年被 Gers 等人明确提出，现在几乎是默认做法。

### Section 5: 从 LSTM 到实际代码

PyTorch 中一行就能创建一个 LSTM 层：

```python
self.lstm = nn.LSTM(
    input_size=embedding_dim,
    hidden_size=hidden_size,
    num_layers=num_layers,
    batch_first=True,
    dropout=dropout if num_layers > 1 else 0,
    bidirectional=bidirectional,
)
```

`hidden_size` 决定 cell state 和 hidden state 的维度。越大能记的模式越多，计算也越重。

`num_layers` 是堆叠层数。两层 LSTM 意味着第一层的 hidden state 作为第二层的输入。

`bidirectional=True` 会同时跑一个正向和一个反向 LSTM，把两个 hidden state 拼起来。适合整句理解任务（如情感分类），不适合生成任务（因为生成时看不到未来）。

前向传播返回三个东西：
- `lstm_out`：每个时间步的输出（所有 hidden state）
- `h_n`：最后时刻的隐藏状态
- `c_n`：最后时刻的 cell state

分类任务通常只取 `h_n[-1]`（最后一层最后时刻的 hidden state）喂给全连接层。

## 与其他方法的对比

### RNN → LSTM 的演进

RNN 只有一个隐藏状态，它同时承担"记忆"和"输出"两个职责。这两个目标打架：记忆需要稳定，输出需要变化。LSTM 把它们拆成 cell state（记忆）和 hidden state（输出），各司其职。

RNN 的信息流动靠乘法链。LSTM 给记忆修了一条加法高速公路。

### LSTM vs GRU

GRU 是 LSTM 的轻量版，2014 年提出。

GRU 把遗忘门和输入门合并成一个"更新门"（update gate），取消了 cell state，把长期记忆和短期输出合并在隐藏状态里。

GRU 只有 2 个门（更新门 + 重置门），LSTM 有 3 个门 + 1 个候选。GRU 参数大约是 LSTM 的 2/3。

| 维度 | LSTM | GRU |
|------|------|-----|
| 门数量 | 3（遗忘/输入/输出） | 2（更新/重置） |
| 是否有独立长期记忆 | 有（cell state） | 无 |
| 参数规模 | 较大 | 较小 |
| 训练速度 | 较慢 | 较快 |
| 表达能力 | 更强 | 稍弱 |
| 中小数据集 | 可用 | 通常表现相当 |

面试结论：两者效果通常接近。数据集小、算力有限时优先 GRU。需要捕捉极复杂长依赖时优先 LSTM。

### LSTM vs Transformer

这是 2024-2025 年面试必考题。

Transformer 用 Self-Attention 直接建模序列中任意两个位置的关系，不管相距多远。LSTM 必须一步一步传，远距离信息要过几十上百个时间步。

| 维度 | LSTM | Transformer |
|------|------|-------------|
| 长距离依赖 | 有上限（~50-200 步） | 理论上无上限 |
| 并行计算 | 不能，必须按时序串行 | 可以，全部并行 |
| 训练速度 | 慢（GPU 利用率低） | 快（适合大规模并行） |
| 参数量 | 相对少 | 巨大 |
| 推理速度 | 较快（适合流式数据） | 较慢（attention 计算量 $O(n^2)$） |
| 小数据量 | 表现好 | 容易过拟合 |
| 可解释性 | 门控信号可分析 | attention 权重可可视化 |

为什么大模型不用 LSTM？核心原因是**无法并行**。时序依赖导致它没法像 Transformer 那样在 GPU 上并行处理整个序列。训练大模型时，LSTM 的速度劣势是不可接受的。

但 LSTM 没有完全被淘汰。以下场景 2025 年仍在用：
- 实时流式处理（在线预测、边缘设备）
- 小数据量时序预测
- 对延迟敏感、序列不太长的任务
- 需要和 Transformer 混合使用的场景（如早期层用 LSTM 提取局部时序特征）

## 面试高频考点

| 考点 | 他真正想知道的 | 回答要点 | 常见错误 |
|------|---------------|----------|----------|
| RNN 为什么有梯度消失？ | 是否理解 BPTT 的连乘机制 | 反向传播时梯度要连乘权重矩阵和激活函数导数，反复压缩导致指数级衰减。类比：复印件的复印件 | 只说"因为序列长"，不讲数学根因 |
| LSTM 怎么解决梯度消失？ | 是否理解 cell state 的特殊结构 | 核心是 cell state 的加法更新。$\partial C_t/\partial C_{t-1} = f_t$，遗忘门接近 1 时梯度几乎不衰减。这是 Constant Error Carousel | 只说"有三扇门"，不解释为什么是加法而不是乘法 |
| 三个门分别干什么？ | 是否能把门和具体功能一一对应 | 遗忘门：旧记忆保留多少。输入门：新信息写入多少。输出门：当前对外展示多少。口诀："先扔、再写、最后看" | 混淆输入门和输出门的作用 |
| cell state 和 hidden state 的区别？ | 是否理解长期记忆和短期输出的分离设计 | cell state 是长期记忆库，更新慢、稳定。hidden state 是当前工作窗口，每一步都变，对外输出 | 认为两者是同一个东西 |
| LSTM 和 GRU 的区别？ | 是否了解简化版的设计动机 | GRU 合并了遗忘门和输入门为更新门，取消 cell state，参数更少。面试结论：效果接近，小数据/有限算力用 GRU | 说"GRU 比 LSTM 好"或"LSTM 比 GRU 好"，没有加条件 |
| 为什么 LLM 不用 LSTM？ | 是否理解架构选型的工程约束 | 核心原因是无法并行计算。LSTM 必须按时序串行，训练速度在 GPU 上远低于 Transformer。长序列时记忆仍有限 | 只说"Transformer 效果更好"，不提并行化这个工程决定因素 |
| 遗忘门偏置为什么初始化为 1？ | 是否了解训练技巧 | 初始状态先"记住一切"，防止训练初期丢失长期信息，帮助网络先学会长期模式 | 不知道这个技巧 |
| 双向 LSTM 能用于生成任务吗？ | 是否理解双向机制的限制 | 不能。生成任务只能看到已生成的词，双向 LSTM 需要看到未来信息。它只适合分类/理解任务 | 说"可以" |

## 一句话总结

LSTM 用一条加法传送带（cell state）绕过 RNN 的乘法梯度陷阱，用三道安检门（forget/input/output）控制信息流，让网络自己选择记住什么、忘记什么、输出什么。
