## 整体框架：四种方法在干什么

先有个全局视角，再往下看细节。

所有控制方法都在解同一个问题：**系统当前状态和目标状态有误差，怎么算出控制量来消掉它？**

| 方法 | 核心思路 | 需不需要系统模型 |
|------|---------|----------------|
| PID | 靠误差的大小、积累、变化速度来反应 | 不需要 |
| MPC | 用模型预测未来，找出最优的一段控制序列 | 需要 |
| 模糊控制 | 把人的经验写成规则，直接查规则表 | 不需要 |
| 数据驱动 | 从历史数据里学规律，预测未来 | 不需要（用数据替代） |

## 第一章：PID 控制

### 1.1 PID 在做什么

想象你在开车，目标是保持时速 100km/h，现在只有 80km/h。

- **P（比例）**：误差是 20，你用力踩油门，误差越大踩越深
- **I（积分）**：踩了很久还没到 100，说明有个持续阻力（比如上坡），I 项帮你补偿这个持续的"亏欠"
- **D（微分）**：速度正在快速上涨，D 项预感到快要超过目标，提前收油门

三个项各自负责一件事，合在一起：

```
u(t) = Kp·e(t) + Ki·∫e(t)dt + Kd·de(t)/dt
```

`u(t)` 是控制输出，`e(t)` 是当前误差 = 目标值 − 实际值。

### 1.2 三个环节各自的副作用

| 环节 | 它能干什么 | 它的问题 |
|------|-----------|---------|
| P | 快速减小误差 | 单独用会有稳态误差；太大会振荡 |
| I | 消除稳态误差 | 太大会超调、震荡；会有积分饱和 |
| D | 预测趋势、抑制超调 | 对噪声敏感，高频噪声会被放大 |

**稳态误差是什么**：系统达到稳定后，实际值和目标值之间还剩的那点差。P 项没法消掉它，因为误差一旦缩小，P 产生的力也跟着变小。I 项专门干这件事。

### 1.3 积分饱和（必考）

执行器（比如电机）有物理上限。如果长时间误差很大，I 项会一直累加，随后执行器到头了还不断累加，等到真的接近目标时，积分项已经超级大，导致严重超调。

**解决方法**（记住这几个）：
- **积分分离**：误差大时不积分，到了误差小的范围再开积分
- **Anti-windup**：执行器饱和时停止积分
- **积分限幅**：给积分项设置上下界

### 1.4 PID 参数怎么调

**Ziegler-Nichols 法**（面试最常提）：
1. 把 Ki、Kd 都设为 0
2. 慢慢增大 Kp，直到系统持续等幅振荡
3. 记录此时的临界增益 `Ku` 和振荡周期 `Tu`
4. 按经验公式设置：`Kp = 0.6Ku`，`Ti = 0.5Tu`，`Td = 0.125Tu`

**工程经验顺序**：先调 P → 加 I 消稳态误差 → 加 D 减超调

### 1.5 离散形式（嵌入式常用）

控制器跑在单片机上时，时间是离散的：

```
u(k) = Kp·e(k) + Ki·Σe(i)·dt + Kd·[e(k) - e(k-1)]/dt
```

## 第二章：MPC 模型预测控制（重点）

### 2.1 用一个比喻理解 MPC

PID 驾驶员：只看当前速度和目标速度的差，给当前时刻的油门。

MPC 驾驶员：提前观察前方 5 秒的路况（弯道、红绿灯），**规划未来 5 秒怎么踩**，但只执行当前这 1 秒，下一秒再重新看路规划。

这就是 MPC 的本质：**用模型预测未来，滚动求解最优控制序列，只执行当前一步**。

### 2.2 MPC 三要素

#### 要素 1：预测模型

用来算"如果我接下来这样控制，系统会怎么走"。

最常用的是线性离散状态空间模型：

```
x(k+1) = A·x(k) + B·u(k)     ← 状态方程
y(k)   = C·x(k)               ← 输出方程
```

- `x(k)`：系统状态（比如位置、速度）
- `u(k)`：控制输入（比如油门、转向角）
- `y(k)`：我们关心的输出

有了这个模型，从当前状态 `x(k)` 出发，给定未来一段控制序列 `u(k), u(k+1), ..., u(k+Nc-1)`，就能预测出未来所有状态。

#### 要素 2：滚动优化

在每个时刻，MPC 解一个优化问题，找出让代价函数最小的控制序列：

```
代价函数 J = (未来输出跟踪误差) + (控制能量) + (控制变化量)
```

数学形式：

```
J = Σ ||y(k+i|k) - r(k+i)||²_Q  +  Σ ||Δu(k+j)||²_R
     i=1 到 Np                        j=0 到 Nc-1
```

- `Np`：**预测时域**，往前看几步
- `Nc`：**控制时域**，未来几步控制量是自由变量（一般 Nc ≤ Np）
- `Q`：跟踪误差的权重，Q 大 → 更在意是否跟上目标
- `R`：控制量变化的权重，R 大 → 更在意控制平滑

把状态预测方程代入代价函数后，最终整理成**二次规划（QP）问题**：

```
min  ½·U^T·H·U + f^T·U
 U
满足约束：C·U ≤ d
```

用 OSQP、qpOASES 等求解器求解，得到最优控制序列 `U*`。

**滚动执行**：只取 `U*` 的第一个元素 `u(k)*` 施加，然后 k+1 时刻重新测量状态、重新求解。

#### 要素 3：反馈校正

每次都重新测量真实状态再优化，所以不怕模型有误差、不怕外部干扰。真实状态和预测状态之间的偏差会在下一轮优化里自动补偿。

**这就是为什么 MPC 鲁棒性好**：它不是事先算好一条路一直走，而是每步都重新看路、重新规划。

### 2.3 预测时域 vs 控制时域

容易混淆，记这个区别：

- **预测时域 Np**：往前预测几步（影响看得"多远"）
- **控制时域 Nc**：未来几步的控制量参与优化（一般 Nc ≤ Np）
- Nc 之后的控制量假设不变（保持最后一个控制量）

Np 越大：越能应对慢动态，但计算量越大。Nc 越大：控制自由度越高，但求解更慢。

### 2.4 MPC 完整执行流程

```
① 测量/估计当前状态 x(k)
② 给定未来参考轨迹 r(k+1), ..., r(k+Np)
③ 用预测模型展开：预测 X = F·x(k) + G·U
④ 构建代价函数 J，整理为 QP（H, f, 约束）
⑤ 调用 QP 求解器 → 得最优序列 U*
⑥ 只执行第一步 u(k)* → 施加到系统
⑦ k = k+1，回到 ①
```

### 2.5 MPC 的约束处理

这是 PID 做不到、MPC 能做到的事：

| 约束类型 | 例子 | 数学形式 |
|---------|------|---------|
| 控制输入约束 | 油门在 [0, 100%] | `u_min ≤ u ≤ u_max` |
| 控制增量约束 | 每步方向盘转动不超过 5° | `Δu_min ≤ Δu ≤ Δu_max` |
| 状态约束 | 车速不超过 120km/h | `x_min ≤ x ≤ x_max` |
| 输出约束 | 温度不超过 80°C | `y_min ≤ y ≤ y_max` |

这些约束加入 QP 问题后，求解器在满足所有约束的前提下找最优解。

### 2.6 面试高频对比

#### MPC vs PID

| 特性 | PID | MPC |
|------|-----|-----|
| 是否需要模型 | 否 | 是 |
| 能否处理约束 | 否 | 是 |
| 能否处理多变量 | 难（需多个 PID 协调） | 天然支持 |
| 计算量 | 极小 | 较大（需实时求解 QP） |
| 预测能力 | 无 | 有（Np 步） |
| 调参难度 | 相对简单（3个参数） | 较复杂（Np, Nc, Q, R） |

#### MPC vs LQR

- **LQR**：对无限时域优化，假设无约束，离线求解，得到增益矩阵 K，在线只做矩阵乘法
- **MPC**：有限时域滚动优化，支持硬约束，在线实时求解 QP
- 没有约束、系统是线性时，MPC（时域→∞）≈ LQR

**一句话区别**：LQR 是离线算好固定策略，MPC 是在线每步重新算。

### 2.7 MPC 是什么类型的优化问题

线性时不变系统 + 二次代价函数 → **QP（二次规划）**

非线性系统 → **NLP（非线性规划）**，运算更重，叫 NMPC（非线性 MPC）

## 第三章：模糊控制

### 3.1 模糊控制在解决什么问题

有些系统没有好的数学模型，但有丰富的人类操作经验。比如：

> "当温度偏高很多时，大幅降低加热功率；当温度稍微偏高时，小幅降低。"

这句话就是一条模糊规则。模糊控制把这类经验写成系统，然后计算机查规则来控制。

### 3.2 三步走

**第一步：模糊化（Fuzzification）**

把精确的测量值变成模糊量。比如误差 e = +15°C，你用"隶属度函数"算出：
- "偏高很多"：隶属度 0.7
- "偏高一些"：隶属度 0.3

**第二步：模糊推理（Fuzzy Inference）**

查 IF-THEN 规则库：
- IF 温度偏高很多 THEN 功率降低很多（权重 0.7）
- IF 温度偏高一些 THEN 功率降低一些（权重 0.3）

**第三步：去模糊化（Defuzzification）**

把推理结果变回精确数值。最常用：**重心法**——找到模糊输出集合的重心位置作为精确控制量。

### 3.3 模糊 PID

实际工程里常用"模糊 PID"：Kp、Ki、Kd 不是固定的，而是由模糊规则根据当前误差和误差变化率实时调整。

优点：自适应、鲁棒性好，比纯 PID 处理非线性系统更稳。

### 3.4 模糊控制的优缺点

| 优点 | 缺点 |
|------|------|
| 不需要精确数学模型 | 规则库设计依赖专家经验 |
| 能处理非线性 | 变量多时规则数量爆炸（维数灾难） |
| 鲁棒性好 | 理论分析（稳定性证明）困难 |
| 直观可解释 | 精度不如 MPC |

## 第四章：数据处理与预测

### 4.1 卡尔曼滤波

#### 它在解决什么问题

传感器测量有噪声，模型预测有误差。卡尔曼滤波通过**融合这两个信息源**，给出比单独用任何一个都更准的状态估计。

**举例**：GPS 信号有时跳变（测量噪声），IMU 积分会漂移（模型误差）。卡尔曼滤波把两者融合，得到更准的位置估计。

#### 两步走（预测 + 更新）

**预测步**（用模型往前推）：
```
x̂(k|k-1) = A·x̂(k-1|k-1) + B·u(k-1)   ← 预测状态
P(k|k-1)  = A·P(k-1|k-1)·A^T + Q        ← 预测协方差
```

**更新步**（用测量值修正）：
```
K(k) = P(k|k-1)·H^T · (H·P(k|k-1)·H^T + R)^(-1)   ← 卡尔曼增益
x̂(k|k) = x̂(k|k-1) + K(k)·(y(k) - H·x̂(k|k-1))   ← 更新状态
P(k|k)  = (I - K(k)·H)·P(k|k-1)                    ← 更新协方差
```

- `Q`：过程噪声协方差（模型有多不准）
- `R`：测量噪声协方差（传感器有多不准）
- `K`：卡尔曼增益，决定"更信模型还是更信测量"

**直觉**：K 大时更信测量，K 小时更信模型预测。

#### 扩展卡尔曼滤波（EKF）

当系统是非线性时，对非线性函数在当前估计点做泰勒展开（一阶线性化），然后用标准卡尔曼滤波流程。代价是线性化带来的误差。

**无迹卡尔曼滤波（UKF）**：用一组 Sigma 点来近似非线性变换，比 EKF 更准，不用手推雅可比矩阵。

#### MPC 中的角色

MPC 要求知道当前状态 x(k)，但往往只有传感器输出 y(k)。卡尔曼滤波就是那个从 y(k) 反推 x(k) 的模块，常见组合：MPC + Kalman Filter（或 Luenberger 观测器）。

### 4.2 系统辨识

#### 它在解决什么问题

MPC 需要模型（矩阵 A、B、C），但很多实际系统的精确物理模型不好建。系统辨识就是**从输入输出数据里把模型学出来**。

#### 主要方法

**最小二乘辨识（Least Squares）**

对于线性时不变系统，给系统打一段输入信号（比如伪随机二进制序列 PRBS），采集输出，然后：

```
θ* = argmin Σ ||y(k) - ŷ(k; θ)||²
```

用矩阵形式求解：`θ = (Φ^T Φ)^{-1} Φ^T Y`

**子空间辨识（Subspace Methods）**

直接从输入输出数据中提取状态空间模型（A, B, C, D），不需要先假设模型结构。代表算法：N4SID。适合 MIMO（多输入多输出）系统。

**ARX / ARMAX 模型**

用过去的输入输出来预测下一步输出：

```
y(k) = a1·y(k-1) + ... + an·y(k-n) + b1·u(k-1) + ... + bm·u(k-m) + e(k)
```

ARMAX 在 ARX 基础上加了噪声模型。

#### 辨识好模型的关键

- 激励信号要"足够丰富"（覆盖系统工作频段）
- 选合适的采样频率（香农采样定理：采样频率 > 2倍信号最高频率）
- 验证集评估：用没参与训练的数据验证模型预测精度

### 4.3 机器学习预测模型

#### 什么时候用 ML 替代物理模型

- 系统过于复杂，物理建模代价太高
- 有大量历史数据
- 模型需要从在线数据持续更新

#### 常用方法

**高斯过程（Gaussian Process, GP）**

输出不只是一个预测值，而是一个**概率分布**（均值 + 方差）。方差告诉你预测有多可信。常用在数据稀少时，或需要知道预测不确定性的场合（比如 Safe MPC）。

**神经网络（尤其是 LSTM）**

处理时序数据预测。LSTM 有记忆机制，能捕捉长时依赖，适合预测动态系统的状态序列。用在数据驱动 MPC 里（用神经网络替代物理模型做预测）。

**随机森林 / XGBoost**

适合特征工程后的预测任务（比如预测设备故障、预测能耗）。不适合做动态系统内部的滚动预测，但适合做一步预测。

#### 数据驱动 MPC（关键趋势）

用机器学习模型替代 MPC 中的物理预测模型：

```
传统 MPC：x(k+1) = A·x(k) + B·u(k)      ← 物理模型
数据驱动 MPC：x(k+1) = f_NN(x(k), u(k))  ← 神经网络
```

代价函数和滚动优化框架不变，只换掉预测模型。面试里这是热门方向。

### 4.4 状态观测器（Luenberger Observer）

当状态 x(k) 不能直接测量时，可以用观测器从输出 y(k) 来估计状态：

```
x̂(k+1) = A·x̂(k) + B·u(k) + L·(y(k) - C·x̂(k))
```

`L` 是观测器增益，通过极点配置设计，让估计误差指数收敛到 0。

和卡尔曼滤波的区别：卡尔曼滤波考虑了噪声的统计特性，Luenberger 观测器是确定性的。

### 一句话总结四种方法

| 方法 | 一句话 |
|------|-------|
| PID | 看误差的大小、历史、变化方向，实时调整输出 |
| MPC | 用模型预测未来，在线求解最优控制序列，每步滚动 |
| 模糊控制 | 把专家经验写成 IF-THEN 规则，不需要精确模型 |
| 卡尔曼滤波 | 融合模型预测和传感器测量，给出最优状态估计 |
