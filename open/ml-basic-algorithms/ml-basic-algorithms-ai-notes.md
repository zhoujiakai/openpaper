## 锚点

**同一个问题，三种完全不同的破法。**

你有一堆数据点，分两类，要找一条"分界线"把它们分开——这是机器学习最基本的分类问题。

- **决策树**：连续问问题。"这个点的特征A大于5吗？是的话继续问B，不是的话看C……"像侦探审案，顺着线索一路往下排查。
- **随机森林**：请500个侦探各自审，然后投票。没有哪个侦探看到完整的案卷，但集体判断比一个人准得多。
- **SVM**：不关心内部，只盯着边界。找到那条线，让它离两边最危险的点（最近的点）尽可能远——间隔最大=边界最安全。

后面所有概念，回到这三个场景里去理解。

## 核心问题

**为什么"画一条直线"不够用？**

最简单的分类器——逻辑回归——本质上是在数据空间里画一条直线（或平面）。两类数据分别在直线两侧，搞定。

但现实数据很少这么配合。两种常见的"难搞"场景：

1. **非线性边界**：数据分布像两个嵌套的同心圆——内圈是A类，外圈是B类。任何直线都分不开它们。
2. **复杂特征关系**：特征A和B的组合决定类别，不是某个单一特征。

三件套给了三条出路：
- 决策树：把空间"切碎"，每次用一个特征做直刀切割，多刀下去，复杂边界就出来了
- 随机森林：很多棵决策树投票，单棵树的错误被平均掉了
- SVM：把数据"扔到"更高维空间，在那个空间里，原来纠缠在一起的点就能被一个平面分开

## 核心概念

### 决策树：侦探怎么排查嫌疑人

想象你在排查一批包裹，要判断哪些是危险品。你会怎么做？

第一步，问"重量超过5kg吗"——超过的一堆，不超过的一堆。然后继续问，每问一次，堆变得更"纯"——每堆里危险品和安全品的比例越来越极端（全是危险品，或者全是安全品）。

决策树做的就是这个：**每次选一个特征+一个阈值，把样本分成两堆，让两堆尽可能"纯"。** 一直分下去，直到每堆里的样本都是同一类——或者你设定了停止条件。

**怎么量"纯度"？** 两种方式：

**基尼指数（Gini，sklearn 默认）**：
- 纯的时候 = 0（全是同一类）
- 最乱的时候 = 0.5（两类各一半）
- 公式：`1 - Σ pᵢ²`
- 不用算 log，速度快

**信息熵（Entropy）**：
- 纯的时候 = 0，最乱的时候 = 1
- 公式：`-Σ pᵢ log₂(pᵢ)`
- 需要算 log，稍慢，但对类别不均衡时更敏感

每次分裂，树选"分完之后纯度提升最大"的那个特征和阈值。这个提升量叫**信息增益**。

**树长多深才够？** 这是过拟合的问题。树越深，越能记住训练数据里的每个细节，但对新数据预测越差。控制方式：

- **预剪枝**（边长边控制）：`max_depth=3`、`min_samples_leaf=5`——达到限制就停
- **后剪枝**（长完再砍）：`ccp_alpha` 参数，值越大砍得越多

实战建议：先不剪枝看基准，再用交叉验证调 `max_depth`。一般从3-5开始试。

### 随机森林：为什么500个"半知道"比1个"全知道"更准

一棵决策树有个大毛病——**对训练数据太敏感**。你稍微换几个训练样本，树的结构可能面目全非。这叫**高方差**。

解法很暴力：造很多棵树，让它们各自判断，投票取多数。

但如果每棵树用同样的数据，学出来的结果几乎一样，投票也没意义。所以随机森林引入**两重随机性**，让每棵树都不一样：

**随机性 1 — Bootstrap 采样**：
每棵树的训练数据从原始数据集中**有放回地随机抽取**，大约63.2%的样本会被抽到（剩下的~36.8%叫"袋外样本"，Out-of-Bag，OOB）。每棵树见到的数据集不一样。

**随机性 2 — 特征随机**：
每次节点分裂时，不看全部特征，而是**随机选一个子集**来找最佳分裂点。分类问题默认用 `sqrt(总特征数)` 个特征，回归默认用全部。

两重随机让树之间"相关性低"——不同棵树的错误是随机分散的，一平均，错误就消失了。这就是随机森林比单棵决策树强的原因：**平均降方差，偏差基本不变**。

**OOB 误差**：每棵树训练时没见过的那36.8%样本，正好可以用来评估这棵树。所有树的OOB评估平均起来，就是不需要单独拿出验证集的泛化误差估计。开启方式：`oob_score=True`。

**特征重要性（Feature Importances）**：
随机森林能输出每个特征"贡献了多少"。计算方式是统计每个特征在所有树的所有分裂中平均降低了多少不纯度（MDI，Mean Decrease in Impurity）。

但有一个坑：**MDI 对取值多的特征（高基数特征）有偏向**，会把它们的重要性算得偏高，即使它们实际上没那么有用。更可靠的方式是用 Permutation Importance（打乱某个特征的值，看性能下降多少）。

### 支持向量机（SVM）：只盯着最危险的边界点

SVM 从一个问题出发：**找一条分类线，什么样的线最"安全"？**

安全 = 不容易被边界附近的新样本打穿。所以最安全的线，是离两侧最近的点尽可能远的线。这个"最近距离"叫**间隔（Margin）**，SVM 的目标是**最大化间隔**。

离这条线最近的那些点，就叫**支持向量**。整个模型的决策边界，完全由这几个支持向量决定。其他点不影响边界——去掉它们，模型不变。

**三种场景，三种应对：**

**场景1 — 完美可分，硬间隔（Hard Margin）**：
两类数据泾渭分明，能找到一条线完全分开，没有任何点在线的错误一侧。现实里几乎不存在这种场景。

**场景2 — 大部分可分，软间隔（Soft Margin）**：
允许少数点"越线"，但对越线行为收惩罚。参数 `C` 控制惩罚力度：
- `C` 大 → 惩罚大 → 尽量不让点越线 → 间隔窄 → 容易过拟合
- `C` 小 → 惩罚小 → 容忍更多越线 → 间隔宽 → 容易欠拟合

现实里基本都用软间隔。调参时 `C` 是第一个要动的。

**场景3 — 根本分不开，核技巧（Kernel Trick）**：
二维里嵌套的同心圆，怎么也找不到一条直线。但如果把数据点"扔到"三维空间，加一个新维度——比如`z = x² + y²`——内圈点集中在z小的地方，外圈点在z大的地方，这时候一个水平面就能分开了。

核函数做的就是这件事，而且**不需要真的计算高维坐标**。它通过计算两个点在高维空间的"内积"来代替显式映射——这就是"核技巧"，大幅节省计算量。

常用核函数：

| 核函数 | 适用场景 | 特点 |
|--------|----------|------|
| `linear` | 特征数远多于样本数；文本分类 | 最快，无额外参数 |
| `rbf`（默认） | 大多数非线性问题 | 万能，但要调 `gamma` |
| `poly` | 特征间有多项式交互 | 参数多，不好调 |

**gamma 参数**（RBF核专属）：
- `gamma` 大 → 每个点只影响周围很小范围 → 边界扭曲、复杂 → 过拟合
- `gamma` 小 → 每个点影响范围大 → 边界平滑 → 欠拟合

调参时 `C` 和 `gamma` 要一起用网格搜索（都在指数尺度上试：0.001, 0.01, 0.1, 1, 10, 100）。

**一个坑**：SVM 对特征的量纲极度敏感。一个特征范围0-1，另一个范围0-1000，后者会主导模型。**用 SVM 之前必须先做特征缩放**（`StandardScaler`）。

## sklearn 代码速查

```python
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import RandomForestClassifier
from sklearn.svm import SVC
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import cross_val_score, GridSearchCV
from sklearn.pipeline import Pipeline

# --- 决策树 ---
dt = DecisionTreeClassifier(
    criterion='gini',      # 'gini' 或 'entropy'
    max_depth=5,           # None = 不限制，建议先设3-5试试
    min_samples_leaf=5,    # 叶节点最少样本数，防过拟合
    ccp_alpha=0.01,        # 后剪枝，越大剪得越多
    random_state=42
)

# --- 随机森林 ---
rf = RandomForestClassifier(
    n_estimators=100,      # 树的数量，多多益善但边际递减
    max_features='sqrt',   # 每次分裂看多少特征，分类默认 sqrt
    oob_score=True,        # 开启袋外误差估计
    n_jobs=-1,             # 并行用全部 CPU 核
    random_state=42
)

# --- SVM（必须先缩放）---
svm_pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('svm', SVC(
        kernel='rbf',      # 默认，大多数情况先试这个
        C=1.0,             # 惩罚强度，越大越容易过拟合
        gamma='scale',     # 'scale' 比 'auto' 通常好
        probability=True   # 开启才能 predict_proba
    ))
])

# --- 调参 (C 和 gamma 一起搜) ---
param_grid = {
    'svm__C': [0.1, 1, 10, 100],
    'svm__gamma': [0.001, 0.01, 0.1, 1]
}
grid = GridSearchCV(svm_pipe, param_grid, cv=5, n_jobs=-1)
grid.fit(X_train, y_train)
```

## 与其他方法的对比

三件套解决的都是分类问题，但它们的"思路根源"完全不同：

决策树 → 加集成 → 随机森林。这是同一条路的两个版本：随机森林就是"很多棵随机化的决策树"。

随机森林（Bagging）和 XGBoost/LightGBM（Boosting）是集成学习的两条路：

| 维度 | 随机森林（Bagging） | XGBoost/LightGBM（Boosting） |
|------|---------------------|------------------------------|
| 目标 | 降低**方差** | 降低**偏差** |
| 训练方式 | 并行（树之间独立） | 串行（后一棵修前一棵的错） |
| 基学习器 | 深树（强学习器） | 浅树/树桩（弱学习器） |
| 对噪声 | 更鲁棒 | 更容易被噪声带偏 |
| 速度 | 快（可并行） | 慢（串行），但 LightGBM 做了优化 |

SVM 和树类方法从本质上就不同：
- 树类：**空间划分**，轴对齐地切割特征空间
- SVM：**边界最大化**，找间隔最大的分离超平面

| 维度 | 决策树 | 随机森林 | SVM |
|------|--------|----------|-----|
| 可解释性 | 最高（可画图） | 低（黑箱） | 低 |
| 小数据集 | 容易过拟合 | 好 | 很好 |
| 大数据集（>10W） | 快 | 慢（但可并行） | 很慢 |
| 高维稀疏（文本） | 差 | 一般 | 线性 SVM 很好 |
| 特征缩放 | 不需要 | 不需要 | 必须 |
| 概率输出 | 支持 | 支持 | 需要 `probability=True`，慢 |

**选算法的粗略原则**：
1. 需要解释给业务方 → 决策树（能画图说清楚）
2. 快速跑一个好用的基准模型 → 随机森林
3. 竞赛或追求最佳性能 → XGBoost/LightGBM（Boosting 系列通常比随机森林强）
4. 数据少、高维、分类边界复杂 → SVM + RBF 核
5. 文本分类或特征维度远大于样本量 → SVM + 线性核 或 逻辑回归

## 面试高频考点

### 决策树部分

| 考点 | 他真正想知道的 | 回答要点 | 常见错误 |
|------|----------------|----------|----------|
| Gini vs 熵，有什么区别？ | 你理解不纯度量化的本质，知道为什么有两种 | 都量"乱度"；Gini 不用算 log 所以快，熵对不均衡更敏感；sklearn 默认 Gini | 只说"计算方式不同"，没说哪种情况用哪种 |
| 为什么决策树容易过拟合？ | 你理解偏差-方差权衡 | 树越深，越能精确记住每个训练样本，泛化就差；高方差问题 | 答"因为训练了太久" |
| 怎么防止决策树过拟合？ | 你知道具体操作，不只说"正则化" | max_depth、min_samples_leaf 限制复杂度（预剪枝）；ccp_alpha 后剪枝；或者用随机森林替代 | 只答"正则化"，没说具体参数 |

### 随机森林部分

| 考点 | 他真正想知道的 | 回答要点 | 常见错误 |
|------|----------------|----------|----------|
| 随机森林为什么比单棵决策树好？ | 你理解 Bagging 降方差的机制 | 两重随机（Bootstrap 采样 + 特征随机）让树之间相关性低；平均后方差下降，偏差基本不变 | 只说"集成了很多树"，没讲随机性为什么重要 |
| Bagging 和 Boosting 的区别？ | 你知道两类集成方法的本质差异 | Bagging 降方差（并行、Bootstrap、平均）；Boosting 降偏差（串行、后树修前树的错误、加权）| 混淆了降方差和降偏差 |
| 特征重要性 MDI 有什么缺陷？ | 你知道这个工具的局限性，不会被结果盲信 | 在训练集上算的，可能不代表泛化；偏向高基数特征；建议用 Permutation Importance 交叉验证 | 直接说"MDI 就是重要性，越大越重要"，不提缺陷 |
| OOB 误差是什么？ | 你理解为什么 OOB 是有效的评估方式 | Bootstrap 抽样约 36.8% 样本未被某棵树见到；用这些样本评估该树；整体 OOB 是不需要验证集的泛化估计 | 说"OOB 就是测试误差"（不是，是估计值，通常偏悲观）|

### SVM 部分

| 考点 | 他真正想知道的 | 回答要点 | 常见错误 |
|------|----------------|----------|----------|
| 什么是支持向量？ | 你理解 SVM 的核心机制，不只会背定义 | 离决策边界最近的那些训练点；只有它们决定边界位置；其他点可以删掉模型不变 | "就是训练样本中比较重要的点"——太模糊 |
| SVM 的参数 C 怎么理解？ | 你能解释 C 对模型行为的影响，知道怎么调 | C 是惩罚系数：大 → 不允许越线 → 间隔窄 → 过拟合；小 → 宽容越线 → 间隔宽 → 欠拟合。用网格搜索调 | 说"C 是学习率" |
| 核函数是什么，为什么需要它？ | 你理解非线性问题的处理思路 | 线性不可分时，把数据映射到高维空间，在那里变成线性可分；核函数用内积避免显式计算高维坐标，节省计算量 | 只说"核函数处理非线性"，没讲为什么不直接计算高维坐标 |
| SVM 为什么必须特征缩放？ | 你知道哪些算法对尺度敏感 | SVM 的间隔计算依赖欧氏距离，特征尺度不同会让某些特征主导距离计算，导致结果偏差。决策树不需要缩放（用的是阈值比较，不是距离）| "所有机器学习算法都需要特征缩放" |
| Hard Margin vs Soft Margin？ | 你理解 SVM 从理想到实用的演化 | Hard 要求完全可分，现实里几乎用不上；Soft 加了松弛变量 ξᵢ，允许部分错误，参数 C 控制惩罚力度 | 分不清松弛变量和核函数的作用 |

## 知识全景：三件套之外，你还应该知道什么

当前这套笔记专注三件套。但机器学习基础算法的完整图谱更大：

```
监督学习（分类/回归）
├── 线性方法
│   ├── 线性回归 / 逻辑回归   ← 最基础，是理解其他算法的起点
│   └── 朴素贝叶斯            ← 文本分类的老将
├── 基于距离
│   └── KNN                  ← 直觉最强，但预测慢，大数据不适用
├── 树类（本笔记核心）
│   ├── 决策树               ← 可解释性最强
│   ├── 随机森林（Bagging）   ← 实用基准
│   └── Boosting 系列
│       ├── AdaBoost         ← Boosting 的起点，理解原理用
│       ├── GBDT             ← 梯度提升的标准形式
│       ├── XGBoost          ← 竞赛标配，工程优化极强
│       └── LightGBM         ← 大数据场景比 XGBoost 更快
└── 支持向量机（本笔记核心）
    └── SVM                  ← 小数据高维的强力选手

无监督学习
├── 聚类：K-Means、DBSCAN、层次聚类
└── 降维：PCA、t-SNE、UMAP
```

**下一步建议**：掌握三件套后，Boosting 系列（XGBoost/LightGBM）的优先级最高——竞赛和实际项目里它们比随机森林更常见，而且理解了随机森林之后，学 Boosting 只需要搞清楚"串行修错误"这一个核心差异。
