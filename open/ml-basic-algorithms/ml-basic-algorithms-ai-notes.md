## 锚点

**三把刀，切同一块蛋糕，下刀方式完全不同。**

决策树是一把直刀——每次选一个特征，一刀切到底，切出越来越小的块。随机森林是请一百个刀工各自切，最后投票哪个块最大。SVM 则不关心块，它找的是刀与蛋糕之间最大的空隙——不是在切，是在画线，而且这条线只关心离它最近的几块蛋糕屑。

后面所有概念，回到这三把刀的画面去想。

---

## 核心问题

**为什么逻辑回归不够用？**

因为现实世界的数据很少是一条直线能分开的。一个二分类问题，逻辑回归划一条直线——要么斜着划，要么加点多项式特征弯曲一下。但遇到这种场景：样本A嵌在样本B中间（同心圆），或者特征之间有复杂的非线性关系——逻辑回归直接报废。

这时候你需要决策树（递归地切空间）、随机森林（多棵树投票防过拟合）、SVM（核函数把数据拍到高维，在那边划平面）。

三者解决了同一个问题——**非线性分类**——但解决路径完全不同。

## 核心概念

### 决策树：先分什么，怎么判断分得好不好？

想象你在整理一堆乱七八糟的快递。你拿起一个包裹，先看"大小"还是先看"重量"？你的目标是：每分一次，剩下的两堆各堆尽可能"纯"——一堆全是书，一堆全是衣服，别混着。

决策树做的就是这件事。它用一个数字来衡量"纯度"：

- **熵**：越乱值越大。全是一种东西 → 熵=0。一半一半 → 熵=1。公式：`-Σ pᵢ log pᵢ`
- **基尼指数**：另一个量乱的方式。`1 - Σ pᵢ²`。纯的时候=0，最乱=0.5。Gini 比熵算得快，因为它不用算 log。

两种量法都行，但 Gini 更常用——省计算。面试官问"Gini vs 熵"，记住：熵用 log，更偏好离散属性；Gini 没 log，更偏好连续属性，计算更快。

每次分裂，树选那个让"纯度提升最大"的特征。这个提升量就是**信息增益**（分之前纯度 − 分之后纯度的加权和）。

信息增益有个毛病：它天然偏爱取值多的特征。比如"身份证号"这个特征，每个值就一个样本，按它分纯度瞬间拉满——但这对预测没用。C4.5 算法的修补方案是除以特征本身的熵，得到**信息增益比**，相当于加了罚项。

实际 sklearn 里的决策树用的是 CART 算法：必须二叉分裂，分类用 Gini，回归用 MSE。你设 `criterion='gini'` 或 `criterion='entropy'`。

树越大越容易背数据（过拟合）。解决方式两种：预剪枝（`max_depth=3`，长了就砍）和后剪枝（先长完整，再从底下往上砍掉没用的枝）。sklearn 用 `ccp_alpha` 做后剪枝：`ccp_alpha` 越大，砍掉的枝越多。

### 随机森林：一棵树不够稳，一百棵怎么样？

一棵决策树很容易过拟合——你多给几个样本，树的结构可能大换血。这就是**高方差**。

森林的思路简单到粗暴：造一百棵树，每棵树用的训练数据都不一样（Bootstrap 采样——有放回地抽，大概 63.2% 的样本被抽到），每棵树分裂时也只能看随机选的一部分特征。最后投票。

两重随机性——随机样本 + 随机特征——让森林里每棵树都不同。平均下来，方差降下来了，但偏差基本不变。

面试官最爱问：**Bagging 降方差还是降偏差？** Bagging 降方差。它是通过平均多个模型的预测来减少波动。Boosting（XGBoost、LightGBM）降偏差——串行训练，后一个模型专门修正前一个模型的错误。

sklearn 里森林的关键参数：
- `n_estimators`：树的数量。默认 100。越多越好，但边际收益递减。上万棵对大多数数据没必要。
- `max_features`：每次分裂看多少特征。分类默认 `sqrt`，回归默认全部。这是最重要的调参项——太小，树太弱；太大，树之间太像。
- `bootstrap=True`：开启才有随机样本。

森林输出两个信息：`feature_importances_` 和 `oob_score_`（如果开了）。特征重要性基于杂质减少量（MDI），但它有坑——偏好高基数特征（比如有很多不同值的特征）。面试时如果被追问"特征重要性可靠吗"，答：MDI 在训练集上算的，不代表泛化能力；permutation importance 更可靠。

### 支持向量机：只关心边界上的点

回到刀和蛋糕的类比。SVM 不关心蛋糕堆得多密，它只关心离刀最近的那几块蛋糕屑——它们叫**支持向量**。

SVM 做的事：找到一条线（或面），让它离两边最近的那些点尽可能远。这个距离叫**间隔**。最大化间隔 = 最稳的分类边界。

但世界不是总是线性的。三种情况：

1. **完美可分** → 硬间隔：严格要求所有点都在线之外
2. **差不多可分** → 软间隔：允许几个点跨过去，但加惩罚（松弛变量 ξᵢ，参数 C 控制惩罚力度）
3. **根本分不开** → 核技巧：把数据投射到高维。二维空间里缠在一起的圈，到三维可能就像一张饼可以轻松分开

**核函数**是 SVM 的灵魂。最常用 RBF 核：`exp(-γ‖x − x'‖²)`。理解为：两个点越近，相似度越高（接近 1）；越远，相似度越低（接近 0）。γ 控制"多近算近"——γ 大，影响半径小，决策边界复杂，容易过拟合；γ 小，影响半径大，边界光滑。

两个核心参数 C 和 γ 一起调：用 `GridSearchCV`，指数级间距（0.1, 1, 10, 100）。

SVM 的软肋：数据大了跑不动，复杂度 O(n_features × n²) 到 O(n_features × n³)。千万级样本别想了。另外特征必须先标准化——SVM 是基于距离的，量纲不一致直接废掉。

## 关键公式 / 代码模式

### 决策树分裂准则

**Gini**（sklearn 分类默认）：
```
Gini = 1 - Σ pₖ²
```
pₖ = 第 k 类样本在当前节点的占比。全是同类=0，均匀分布=最乱。计算没 log，快。

**信息增益**：
```
IG = 父节点熵 - (左子熵×左占比 + 右子熵×右占比)
```
选 IG 最大的特征做分裂。

### SVM 决策函数

```
f(x) = sign( Σ αᵢ yᵢ K(xᵢ, x) + b )
```
∑ 只对支持向量求和（αᵢ > 0 的那些点）。这意味着预测时只用少数点——model 很紧凑。

### 核心代码片段

```python
# --- 决策树 ---
from sklearn.tree import DecisionTreeClassifier, plot_tree

tree = DecisionTreeClassifier(
    criterion='gini',      # 或 'entropy'
    max_depth=5,           # 先设小，再放宽
    min_samples_split=10,  # 节点至少 10 个样本才分
    min_samples_leaf=5,    # 叶子至少 5 个样本
    ccp_alpha=0.01,        # 后剪枝强度
    random_state=42
)
tree.fit(X_train, y_train)

# 看树长什么样
from sklearn.tree import plot_tree
plot_tree(tree, filled=True, feature_names=feature_names)

# --- 随机森林 ---
from sklearn.ensemble import RandomForestClassifier

rf = RandomForestClassifier(
    n_estimators=100,
    max_features='sqrt',   # 或 0.5, 或整数
    oob_score=True,        # 开 OOB 验证
    n_jobs=-1,             # 全核并行
    random_state=42
)
rf.fit(X_train, y_train)
# OOB 分数（不占测试集）
print(rf.oob_score_)
# 特征重要性
print(rf.feature_importances_)

# --- SVM ---
from sklearn.svm import SVC
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import make_pipeline

# 标准化 + SVM 一定要放一起
svm = make_pipeline(
    StandardScaler(),
    SVC(
        kernel='rbf',
        C=1.0,        # 正则化，小 → 光滑
        gamma='scale', # 默认，自动算
        probability=True,  # 开概率（会变慢）
        random_state=42
    )
)
svm.fit(X_train, y_train)
```

## 逐节详解

### Section 1: 决策树怎么长出来

你有一堆数据，每行有特征 X 和标签 y。树要从根开始长。

**问题：先拿哪个特征分？**

树的策略：试遍所有特征的所有切点，算每种切法的纯度提升，选提升最大的。然后对左右两个子节点递归做同样的事。

这就是 CART 的贪心策略——每一步都选当前最优，不管后面。所以 CART 不保证全局最优（要全局最优是个 NP 难问题）。

**什么时候停下来？**

1. 节点里的样本全是一类了 → 不用分了，这就是答案
2. 到了 `max_depth` → 强制打住
3. 节点里样本太少，不到 `min_samples_split` → 不分了
4. 分了之后纯度提升太小（`min_impurity_decrease`）→ 分也没意义

树太深 → 背数据，测试集拉胯。树太浅 → 学不到东西，连训练集都拉胯。找平衡就是调参。

回归树和分类树的区别只在于用什么量纯度：分类用 Gini/熵，回归用 MSE/MAE。

### Section 2: 怎么防过拟合——预剪枝 vs 后剪枝

预剪枝就是在长的过程中设限：`max_depth`、`min_samples_split`、`min_samples_leaf`。优点是快，缺点是可能砍得太早——某个分支现在看起来不怎么样，但再分一层可能突然变好。

后剪枝先让树长满，再从底下往上砍。sklearn 用的是**最小代价复杂度剪枝**（`ccp_alpha`）。公式：

```
R_α(T) = R(T) + α × (叶子数量)
```

R(T) 是错误率，α 是惩罚系数。α=0 时不剪，α 越大剪得越多。sklearn 提供 `cost_complexity_pruning_path()` 帮你找出最优 α。

调参建议：先用 `ccp_alpha=0` 长满，看剪枝路径图，选拐点处的 α。

### Section 3: 随机森林为什么有效——方差是怎么降下来的

一棵树容易过拟合，本质是方差大：换个训练集，树的结构天差地别。

随机森林通过两重随机性把方差打下来：

1. **样本随机**：Bootstrap。N 个样本，有放回抽 N 次。每个样本被抽到的概率约 63.2%，剩下的叫 OOB（袋外样本）——可以用来不占测试集地估泛化性能。
2. **特征随机**：每个节点分裂时只看 k 个随机挑选的特征。k 默认是 `sqrt(n_features)`（分类）或全部（回归）。

这两个随机性让树与树之间"长得不一样"。它们的错误不相关——树A在这个样本上错，树B在那个样本上错，投票时互相抵消。这就是**集成学习中降低方差的核心机制**：独立误差互消。

但注意：RF 在大噪声数据上也会过拟合。如果噪声很强，每棵树都在追着噪声跑，投票也救不回来。

**OOB 分数黑话解释**：每棵树只用 63.2% 的样本训练。剩下的 36.8% 没被它见过。用这些"没见过"的样本测每棵树，投完票算准确率，就是 OOB 分数。它相当于免费的交叉验证，但 OOB 通常偏悲观（比 CV 估得低），sklearn 文档建议能用 CV 就别用 OOB。

### Section 4: SVM 的数学直觉——不推导公式，说清楚它在算什么

SVM 找的是最大间隔超平面。"间隔"是什么？就是最近的那个点到平面的距离。SVM 要最大化这个最小距离——听起来矛盾的"最大最小"，所以叫 max-margin。

SVM 的优化目标精简成大白话：

> 找一个方向 w（法向量），让两类样本到平面的最小距离尽可能大。同时，允许个别点犯错（松弛变量），但犯错要罚（参数 C）。

当数据线性不可分时，SWM 用核函数把数据"拍"到高维。核函数算的是两个样本在高维空间里的内积，但不需要真的把数据升维再算——这就是**核技巧**——直接在原空间算核函数就等于在高维做了内积。

**C 和 γ 的口诀**：
- C：容忍度。越小越"佛系"——宁可画错几个点，也要线光滑。越大越"完美主义"——必须分类全对。
- γ：影响力半径。越小 → 一个点能影响到很远 → 边界平滑。越大 → 一个点只影响近邻 → 边界崎岖（过拟合风险）。

调参时 C 和 γ 一起涨容易过拟合，一起降容易欠拟合。GridSearchCV 稳。

### Section 5: sklearn API 的正确打开方式

**SVM 必须配 StandardScaler**。SVM 是基于距离的，特征量纲不一致直接导致某些维度主导决策。用 Pipeline 打包：

```python
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.svm import SVC

clf = make_pipeline(StandardScaler(), SVC())
```

**决策树和随机森林不需要标准化**。它们是基于阈值分割的，不是基于距离的。你统一把特征乘 2，树的分裂点也跟着乘 2，结果不变。

**多分类策略**：SVC 默认一对一（ovo）——如果有 10 个类，建 45 个二分类器。LinearSVC 是一对多（ovr）。ovo 更多分类器但每个只在小数据集上跑，ovr 更少分类器但每个要在全量数据上跑。类别多时 ovo 可能更快。

**概率输出**：SVC 设 `probability=True` 才能用 `predict_proba()`。代价是内部做 5 折交叉验证来校准 Platt Scaling，显著变慢。不需要概率就别开。

## 与其他方法的对比

三者解决了同一个问题（分类/回归），但问题场景和约束不同，选择逻辑也不同。

| 维度 | 决策树 | 随机森林 | SVM |
|------|--------|---------|-----|
| 擅长的数据量 | 中小 | 中大型 | 中小（大了跑不动） |
| 特征维度 | 低维可解释 | 高维也行 | 高维甚至超过样本数也行 |
| 是否需要标准化 | 不需要 | 不需要 | **必须** |
| 可解释性 | 白盒，可画图 | 黑盒 | 黑盒（但支持向量有迹可循） |
| 过拟合风险 | 高（不剪枝必过拟合） | 低（随机性天然防过拟合） | 中（靠 C 和 γ 控制） |
| 训练速度 | 快 | 中（并行可加速） | 慢 O(n²) ~ O(n³) |
| 缺失值 | 原生支持 | 原生支持 | 需要预处理 |
| 特征重要性 | 内置 | 内置（MDI） | 无（系数只对线性核有意义） |

**选择决策树的唯一理由**：你需要给别人解释为什么这个预测是这个结果。银行信贷审批、医疗诊断——可解释性是刚需。

**选择随机森林的典型场景**：表格数据，不想花太多时间调参，需要基线模型快速出结果。RF 是"开箱即用"最好的算法之一。

**选择 SVM 的典型场景**：小样本、高维度。比如基因表达数据（几百个基因，几十个样本）。或者你对核函数的选择很有把握。

| Bagging 降方差还是降偏差？ | 你理解集成学习的核心原理 | Bagging 降方差。并行训练多个模型，独立误差投票抵消。Boosting 降偏差——串行矫正。 | 搞反成 BOOSTING 降方差 |

| C 太大/太小会怎样？ | 你懂正则化的含义 | C 大 → 惩罚重 → 几乎不许犯错 → 边界复杂 → 容易过拟合。C 小 → 惩罚轻 → 允许更多点跨边界 → 间隔大但可能欠拟合 | 把 C 和 γ 的作用说反 |

| 什么时候用决策树不用森林？ | 你在做工程权衡 | 需要可解释性时。给老板看一棵树能画出来，看一百棵树没法说。还有数据极小时，RF 的 Bootstrap 就是在浪费数据 | 说"小数据集"但不提可解释性 |
| 多分类时 SVC 怎么处理？ | 你知道 ovo 和 ovr 的区别 | SVC 用 ovo：3 类建 3 个分类器，4 类建 6 个。LinearSVC 用 ovr。ovo 训练快（每个分类器数据少）但分类器多；ovr 反之。 | 不知道这两个策略的区别 |
| 怎么判断特征重要性是否可靠？ | 你知道 MDI 的坑 | RF 的 feature_importances_ 基于 MDI（杂质的平均减少量），有两个坑：①只在训练集上算，不代表泛化能力 ②偏好高基数特征。permutation importance 更可靠——打乱某个特征看准确率掉多少 | 直接答"重要就是重要"，没质疑评估方式 |
