## 容易混淆的点

### 1. Bagging 降的是方差，不是偏差

我当时答的是"降偏差"，理由是"错误相互抵消"——方向对了，但对应的概念搞反了。

- **方差**：换一批训练数据，预测结果变化有多大（不稳定性）
- **偏差**：模型预测的平均值和真实答案的差距（系统性偏差）

单棵决策树：偏差低，但**方差高**（换数据树结构大变）。随机森林把很多棵树的随机误差平均掉，消的是不稳定性，也就是**方差**。偏差基本不变。

口诀：
> **Bagging（随机森林）= 降方差**，并行，平均，随机误差抵消
> **Boosting（XGBoost）= 降偏差**，串行，后一棵修前一棵的错

---

### 2. n_estimators 的上限不是特征数量，而是收益递减

我说"上限可能是特征数量"——这是错的，树的数量和特征数没有关系。

真正的限制是**收益递减**：从1棵→100棵提升明显，从1000棵→2000棵几乎没变化。误差在某个点之后趋于稳定。实践里 100~500 棵够用，超过之后只是烧算力。

---

### 3. SVM 不会自动忽视异常值——取决于 C 参数

我说"SVM 会把异常值当作异常值处理，对模型没影响"——不准确。

- **Hard Margin**：零容忍，一个离群点就可能导致无解或间隔被压垮
- **Soft Margin**：允许部分点越线，由参数 C 控制惩罚力度
  - C 小 → 惩罚轻 → 容忍越线 → 对异常值不敏感（接近"忽视"）
  - C 大 → 惩罚重 → 尽量不越线 → 对异常值敏感

所以不是自动忽视，而是 C 小的时候才相对宽容。

---

### 4. SVM 能处理非线性问题——核技巧

我当时说"SVM 无法处理同心圆这种非线性分布"——完全错了。

SVM 通过**核技巧**处理非线性：把数据映射到高维空间，在那里变成线性可分，再用超平面分开。

例子：同心圆数据加一个维度 `z = x² + y²`，内圈 z 小，外圈 z 大，三维里一个水平面就分开了。

精妙之处：不需要真的算出高维坐标，核函数直接计算高维内积，节省计算量。

### 5. gamma 参数——大了过拟合，小了欠拟合

之前完全不知道 gamma 是什么，补充如下：

gamma 控制每个训练样本对决策边界的"影响范围"：
- gamma **大** → 影响范围小 → 边界贴着每个点弯来弯去 → **过拟合**
- gamma **小** → 影响范围大 → 边界平滑 → **欠拟合**

C 和 gamma 要一起调（网格搜索，指数尺度）：

| | C 小 | C 大 |
|--|------|------|
| **gamma 小** | 欠拟合 | 还行 |
| **gamma 大** | 还行 | 过拟合 |

### 6. Bootstrap 采样：约 36.8% 的样本不会被选到

之前不会算。推导：

- 单次没被抽到的概率：`1 - 1/n`
- n 次都没被抽到：`(1 - 1/n)^n ≈ 1/e ≈ 0.368`

结论：Bootstrap 采样，约 **63.2% 被抽到，36.8% 没被抽到（OOB 样本）**，跟数据集大小无关。
