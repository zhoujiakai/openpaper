## 一、PID 控制

### 稳态误差 - 我一开始没懂的地方

我的困惑："P 会让水龙头越开越小到刚好抵消蒸发，那再拧大一点不就行了？"

**实际答案**：P 控制是个死规矩——`水龙头开度 = Kp × 误差`。误差 0.5，就是开度 Kp×0.5，不能自己"再拧大一点"。它不是人能灵活判断的，是用公式算死的。公式算出来多大就是多大。

### P 震荡调参 - 我没答出来

我的困惑：一直震荡不知道怎么办。

答案：**把 Kp 减小**。
- 震荡的原因：Kp 太大，每次调整幅度太大，冲过头
- 减 Kp → 每次调整幅度变小 → 震荡减弱/消失
- 代价是响应变慢

附：面试通用的调参口诀
- 响应太慢 → 加大 Kp
- 震荡 → 减小 Kp
- 有稳态误差到不了目标 → 加 I（Ki）
- 超调严重（冲过头） → 加 D（Kd）

### 积分饱和 - 我听懂了

当水龙头已经开到 100% 封顶，但 I 还在累积"欠账"——等水到了目标水位，I 的累积还在起作用，导致严重超调。

解法三个：
1. 积分分离：误差大时关 I，误差小再开
2. Anti-windup：执行器饱和时停 I
3. 积分限幅：给 I 设天花板

## 二、MPC 模型预测控制

### 核心理念 - 我理解到位的地方

PID = 被动反应（出错了再纠偏）
MPC = 主动规划（提前算好怎么走，每步重新算）

这两个对比让我明白了 MPC 的本质。

### 为什么 MPC 能处理约束，而 PID 不能 - 我没答出来

**直接原因**：

PID 是 **直接算**：`u = Kp·e + Ki·∫e + Kd·ė`，从误差到控制量是一条公式算出来的，没经过任何"检查"。算出来 150% 就直接输出 150%，超过实际限制了也得执行。

MPC 是 **在框框里挑最优**：先列出一堆"被允许的控制方案"，再在里面找最好的。约束条件是求解 QP 之前就写进去的，求解器自然会在范围内找解。

**一句话**：PID 不管约束，只管算；MPC 先划定范围，再在范围内找最优。

### MPC vs LQR - 我不知道 LQR 是什么

**LQR 的核心**：
- 离线一次性算出一个固定的增益矩阵 K
- 在线只需要做 `u = -K·x`（简单的矩阵乘法）
- 不能处理约束（没设计这个功能）

**对比**：

| | LQR | MPC |
|--|-----|-----|
| 怎么算 | 离线算好 K | 在线每步重新求解 QP |
| 约束 | ❌ 不能 | ✅ 能 |
| 计算量 | 极小（一个矩阵乘法） | 大（要解 QP） |
| 有趣关系 | - | 线性+无约束+无限时域时≈LQR |

## 总结：我目前能用自己的话讲清楚什么

- P：管现在，误差×Kp，但有稳态误差
- I：管历史，累积误差，清稳态误差，但有积分饱和风险
- D：管趋势，提前刹车，抑制超调，但怕噪声
- MPC 三要素：预测模型、滚动优化、反馈校正
- MPC 流程：测状态→预测→设代价函数→加约束→解 QP→执行第一步→滚动
- 约束处理：MPC 在框框里挑最优（和 PID 的根本区别）
- LQR：离线算好固定策略，不能处理约束
