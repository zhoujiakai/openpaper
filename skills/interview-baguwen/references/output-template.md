# 输出模板

以下是生成的八股文 Markdown 文件的目标结构和格式标准。

## 文件命名

`{方向}_{模块}_八股文.md`

- `方向`：从简历文件名推导（如"AI开发方向"）
- `模块`：提取的特定技能模块（如"开发技术"、"AI应用开发"）

## 文档结构

```markdown
# {方向} - {模块}面试八股文

> 基于简历「{简历文件名}」中「专业技能 → {模块}」模块整理
> 涵盖：{列举所有涉及的技术关键词，逗号分隔}

---

## 目录

- [一、{技术1名称}](#一技术1名称)
- [二、{技术2名称}](#二技术2名称)
- ...

---

## 一、{技术名称}

### 1.1 {子主题}

**Q: {面试问题}？**

{答案内容，包含：}

- 文字说明（简洁准确）
- 代码示例（如适用，用 ```代码块```）
- 对比表格（如适用，用 markdown table）
- 实际应用场景

**Q: {追问/相关面试问题}？**

{继续 Q&A ...}

---
```

## 问答写作规范

每条问答必须遵循以下规则：

1. **问题格式**：`**Q: {问题}？**` — 使用清晰、直接的面试风格提问
2. **答案格式**：
   - 以简洁的1-2句定义或结论开头
   - 接着展开详细说明
   - 在适用的地方包含带语法高亮的代码示例
   - 对相关概念使用对比表格（如 MySQL vs PostgreSQL、V1 vs V2）
3. 每项技能的**深度层次**：
   - 基础：定义、用途、核心概念（2-3题）
   - 中级：内部工作机制、常见使用模式（3-5题）
   - 高级：边缘情况、优化、生产实践（2-3题）
4. **代码示例**：必须是语法正确的 Python/SQL/YAML/Nginx 配置
5. **表格**：使用带清晰表头的 markdown 表格进行对比
6. **杜绝废话**：每一句话都必须有信息价值

## 章节覆盖要求

针对简历中发现的每项技术/技能，覆盖：

| 覆盖领域 | 是否必须 | 示例 |
|----------|----------|------|
| 核心定义 | 是 | 什么是 X？ |
| 关键特性/机制 | 是 | X 的内部工作原理是什么？ |
| 与替代方案的对比 | 如适用 | X 和 Y 的区别是什么？ |
| 常见面试题 | 是 | 每项技术 3-8 题 |
| 代码示例 | 如为代码相关技术 | Python/SQL/配置片段 |
| 生产环境最佳实践 | 是 | 在生产中如何使用 X？ |
| 故障排查/优化 | 如适用 | 常见陷阱和解决方案 |

## 附录部分

文档末尾附上附录，涵盖：

```markdown
## 附录：面试高频综合题

**Q: 你的项目中有哪些技术难点？怎么解决的？**

{结合简历项目经历给出建议}

**Q: 如何设计一个高可用、高性能的{相关}系统？**

{架构设计思路，分点阐述}
```
