# 面试八股文生成器

基于从用户简历中提取的技术技能，生成全面、准确的面试问答准备指南（"八股文"）。

## 工作流程

### 第一步：读取并解析简历

使用 `docx` 技能（针对 .docx 文件）或 `pdf` 技能（针对 .pdf 文件）读取用户的简历文件，提取全文内容。

识别"专业技能"部分及其所有子模块（如"开发技术"、"AI应用开发"、"算法能力"）。提取其中提到的每一项技术、框架、工具和概念。

如果用户指定了要聚焦的模块，则仅筛选该模块。否则，覆盖所有模块。

### 第二步：识别并整理技能

针对发现的每项技能/技术，按逻辑分组整理：

- **语言与核心**：Python、async、装饰器、生成器等
- **框架**：FastAPI、Pydantic、SQLAlchemy、Celery等
- **数据库**：MySQL、PostgreSQL、Redis等
- **基础设施**：Docker、Nginx、Prometheus、Grafana等
- **领域专精**：SEO、GEO等

在继续之前，将所有已识别的技能列出来供用户确认。

### 第三步：调研（网络搜索）

使用 WebSearch 验证并收集每项技术的最新信息。尽可能并行搜索。关键搜索模式：

```
"{technology} 面试题 interview questions {current_year}"
"{technology} V2 new features changes"
"{technology} vs {alternative} differences"
"{technology} optimization best practices production"
```

### 第四步：生成面试问答

针对每项技术，生成覆盖三个深度层次的问答：

1. **基础**（2-3题）：定义、用途、核心概念
2. **中级**（3-5题）：内部机制、常见模式、与替代方案的对比
3. **高级**（2-3题）：生产实践、优化、边缘情况

写作规则：
- 每个答案必须事实准确——不得编造
- 包含语法正确的代码示例（Python、SQL、YAML、Nginx等）
- 对相关概念使用对比表格（markdown表格）
- 问题必须是求职者在面试中真实会遇到的问题
- 答案简洁但完整——注重清晰而非冗长

### 第五步：格式化与保存

按照 `references/output-template.md` 中的输出模板设定文档结构和格式标准。

文件命名：`{方向}_{模块}_八股文.md`

- `方向`：从简历文件名推导（如"AI开发方向"）
- `模块`：所覆盖的技能模块（如"开发技术"）

将文件保存在源简历的**同一目录**下。

## 质量检查清单

在最终完成前，验证：

- [ ] 简历技能部分的每项技能均已覆盖
- [ ] 没有编造的API、方法或版本特性
- [ ] 代码示例语法正确
- [ ] 对比表格信息准确
- [ ] 反映了最新版本变化（如 Pydantic V2、SQLAlchemy 2.0）
- [ ] 附录包含与用户项目经验相关的跨领域面试题
